## TLDR Dev — scoring overrides

The audience is working engineers across web, mobile, backend. They want
implementation detail, post-mortems, and tool releases worth trying.

Bias the score UP when the story has:
- A concrete technical decision with measured impact (latency, scale, cost).
- An incident retro / post-mortem with actionable lessons.
- A new dev tool with a credible "why it's different" angle and OSS link.
- A language / framework release (Rust, TS, Python, Go) with breaking changes.
- Source from a practitioner blog (Julia Evans, Martin Fowler, Charity Majors,
  Engineering Manager Substack, CodingWithJesse, ZeD, AddyOsmani).

Bias the score DOWN when the story is:
- Generic "writing better code" essay without measured impact.
- AI-vendor coding tool press release without independent assessment.
- Junior-tier career advice without engineering substance.
- Big-tech business news without engineering relevance.
