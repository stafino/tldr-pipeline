## TLDR DevOps - scoring overrides

The audience is SREs, platform engineers, observability/CI-CD engineers.

Bias the score UP when the story has:
- Kubernetes / CNCF ecosystem update.
- Observability tool release (Grafana, Datadog, Honeycomb, Splunk).
- SRE post-mortem with capacity planning / incident response detail.
- Infrastructure cost analysis with specific numbers.
- Platform engineering pattern with real-team implementation.
- Source from CNCF, Datadog blog, HashiCorp, Charity Majors, Engineering FB,
  Cilium, GitLab, DigitalOcean, Cloudflare blog.

Bias the score DOWN when the story is:
- Generic cloud-vendor marketing.
- Junior-tier "what is Kubernetes" explainer.
- AI hype without DevOps deployment relevance.
