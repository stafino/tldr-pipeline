# tldr-pipeline

AI-assisted curation pipeline for the TLDR family of newsletters. Two parts:
the Python pipeline (scrape → score → blurb → format) and the Next.js review
viewer that ships to https://www.trylede.com.

## Layout

```
ingestion/   RSS scrapers              → data/raw/<date>.jsonl
dedup/       URL canonicalization      → data/deduped/<date>.jsonl
ranking/     Multi-newsletter scoring  → data/scored/<date>.jsonl
blurbs/      Blurb generation (LLM)    → data/blurbs/<date>.jsonl
formatters/  Final issue rendering     → data/issues/<nl>-<date>.txt
common/      Shared models + LLM client
config/      newsletters.yaml (sections, target counts, edition size)
web/         Next.js App Router review UI (deployed to Vercel)
scripts/     Backtest + cron orchestration
tldr         Bash CLI driver (entry point for every stage)
```

## Run the pipeline

```bash
./tldr refresh              # full pipeline for today (DATE env to override)
./tldr ingest|dedup|rank    # one stage
./tldr blurbs all           # blurb every assigned story across all newsletters
./tldr backfill-blurbs      # idempotent: re-run blurbs over every scored date
./tldr format all           # render text issues
./tldr backtest 2026-06-10 2026-06-17   # recall@N vs real TLDR archives
./tldr status               # counts for today's pipeline
```

Cron: `scripts/cron-refresh.sh` runs every 6h via launchd. It chains
ingest → dedup → rank → blurbs → format → backtest_cache → learn_weights
and commits the results to main.

Env overrides:
- `DATE=2026-06-14` — run for a specific date
- `LLM_BACKEND=cli|api` — switch between the `claude` CLI and the Anthropic API
- `BLURB_MODEL=claude-opus-4-7` — override the blurb model
- `MIN_ASSIGNMENT_SCORE=55` — minimum per-newsletter score to keep an assignment

## Web (`web/`)

Next.js 14 App Router + Tailwind, TypeScript. Deployed from GitHub `main`
to Vercel project `tldr-pipeline` (URL: https://www.trylede.com).

```
web/app/
  page.tsx          Curate tab (newsletter rail + section view + detail pane)
  edition/page.tsx  Edition tab (one finalized issue per day)
  backtest/page.tsx Backtest tab (recall@N dashboard)
  icon.svg, apple-icon.png, opengraph-image.png  brand assets
  manifest.webmanifest
web/components/
  Nav, DatePicker, NewsletterPicker, StoryRow,
  DetailPane, EditionStories, CurateNewsletterView, useDecisions
web/lib/
  data.ts   File-system data loader (reads ../data/*)
  types.ts  ScoredStory, Blurb, Decision, etc.
  utils.ts  shortDate, canonicalDomain
```

Build / dev:
```bash
cd web && npm run build      # production build
cd web && npm run dev        # local on :3000
```

The viewer reads JSONL files directly off disk (`web/lib/data.ts findRoot`),
so a deploy needs `_embedded/` populated by `scripts/embed.mjs` OR the whole
repo checked out next to `web/`.

## Conventions

- Dates are UTC. The Curate "Filter by date" means the story's UTC
  `published_at` day, not the scraper's fetch day. `shortDate()` renders in
  UTC for the same reason.
- Decisions (approve/reject/edit) live in `localStorage` under
  `lede.decisions.v1`. `useDecisions()` is a module-level shared store via
  `useSyncExternalStore`, so an upsert in one component instantly updates
  every other component.
- The blurb generator skips stories already blurbed for the same newsletter
  in the last 14 days (`common/past_blurbs.py`). Re-running blurbs is
  cheap and idempotent.
- Every story assigned to a newsletter must end up with a blurb. The
  generator now blurbs ALL assignments (not just top-N per section).
- Edition export buttons live in `web/components/EditionStories.tsx`.
  Four outputs are generated client-side from the approved-set: rich
  email HTML (heavy inline styles for direct rendering), semantic
  editor HTML (clean h1/h2/p for Substack/Beehiiv paste), `.eml` draft
  (multipart/alternative, opens as Apple Mail draft), and a plain-text
  fallback. No backend mail sending.

## Don't do unprompted

- Mock the LLM in tests — calls go through `common/llm.py` and route to
  either the API or the CLI based on `LLM_BACKEND`. Use the cache instead.
- Modify `data/backtest/*.json` by hand — they're regenerated by
  `scripts/run_backtest_cache.py`.
- Touch `data/raw/`, `data/deduped/`, `data/embeddings_cache/` — gitignored
  intermediate artifacts that the pipeline regenerates.
- Bump `next`/`react`/`tailwindcss` versions.

## Anti-hallucination

- The repo has memory under `~/.claude/projects/.../memory/`. Treat it as a
  snapshot — when a memory entry names a file or flag, grep to confirm it
  still exists before acting on it.
- Blurb model defaults to `claude-opus-4-7`. Don't assume cheaper Haiku
  unless I switch it.
