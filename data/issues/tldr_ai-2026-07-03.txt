Memorizing session transcripts isn't useful (1 minute read)

The author argues that giving coding agents search access to their own past session transcripts yields zero measurable benefit on SWE tasks, and may even degrade performance. Their explanation is that well-maintained artifacts already capture the useful signal: agents are instructed to write thorough commit messages, PR descriptions, and docs, so a transcript search server just spends tokens re-reading distilled context the agent can already access. Track artifacts, not scratch.

60% Fable cost cut by converting code to images and having the model OCR it (1 minute read)

pxpipe is a local proxy that cuts Claude Code's input tokens by rendering bulky context (system prompt, tool docs, older history) as compact PNGs before the request leaves your machine. Because an image's token cost is fixed by its pixel dimensions, dense code and JSON pack roughly 3.1 characters per image-token versus 1 per text-token, turning a ~25k-token prompt into ~2.7k image tokens that the model still reads at 100/100 on a clean eval.
