AI coding tool hole illustrates a big problem with human in the loop (3 minute read)

Wiz disclosed GhostApproval, a vulnerability pattern in six top AI coding assistants including Amazon Q Developer, Claude Code, Cursor, and Google Antigravity, where a malicious repository uses symlinks to trick the agent into writing outside its sandbox, potentially achieving remote code execution. The deeper problem is CWE-451: the agent's reasoning recognizes the dangerous target, but the confirmation prompt hides it, so the human approves what looks like a harmless local edit. It shows that human-in-the-loop safeguards fail when the tool controls what the human sees.

Enterprises using multiple AI models are underestimating failure rates by 2.25x (3 minute read)

A study evaluating 67 frontier models from 21 providers argues that enterprises routing queries across multiple models underestimate their failure rates by 2.25x. The reason is the "co-failure ceiling": orchestration is limited not by how often models disagree, but by the share of prompts where every model in the pool is wrong at once, a floor that diverse routing cannot cross. The same math yields a cost-free test for when multi-model orchestration actually pays off.

3100 Opinions on Code Review in an AI World: Building Causal Theory from Practitioner Discourse (1 minute read)

This paper synthesizes practitioner discourse into a causal theory of how coding agents affect code review. The authors mined 38,709 engineering blogs and Reddit threads, coded a stratified sample of 3,100 with an LLM-assisted pipeline, and built a model of 26 constructs and 67 relationships. Its central claim is that review is the control point that decides an agent's effect on software, and AI doesn't fix the sign of that effect: the team sets it through its expertise and how it structures review.

An off switch for dual use knowledge in AI models (5 minute read)

Anthropic and AE Studio introduced GRAM (Gradient-Routed Auxiliary Modules), a method that isolates dual-use knowledge like virology and cybersecurity into a removable module so a single training run can produce both restricted and unrestricted model versions. Unlike pretraining data filtering, which requires training a separate model for each capability set, GRAM lets developers grant vetted users access to sensitive capabilities without retraining, though results are preliminary and not yet applied to production models.

SMetric: Rethink LLM Scheduling for Serving Agents with Balanced Session-centric Scheduling (1 minute read)

SMetric is a session-centric LLM scheduler for agentic serving, where requests come from agents rather than humans and KV-cache reuse can exceed 80% of request tokens. It routes each agent session's first request purely for load balance and follow-up requests cache-aware, improving cluster TPS by 10-16% under prefill-decode colocation and prefill TPS by 2-34% under disaggregation over state-of-the-art schedulers.

It Takes a MAESTRO To Prune Bad Experts (1 minute read)

MAESTRO is a structured pruning framework for Mixture-of-Experts models that models autoregressive expert activation as ergodic Markov chains, using their stationary distributions to score expert importance with awareness of cross-layer routing dependencies. Across five domains including safety, bias, and ethics, it beats state-of-the-art baselines by up to 10.61% in performance retention under 50% compression while showing lower cross-task variance.

TTHE: Test-Time Harness Evolution (1 minute read)

This paper introduces Test-Time Harness Evolution (TTHE), a method that optimizes an LLM agent's harness during evaluation using only unlabeled execution traces, without updating model weights or requiring gold labels. TTHE maintains a population of candidate harnesses that an agentic proposer refines and a judge commits based on execution-derived proxy signals, improving fixed ReAct-style baselines across text-to-SQL, competitive programming, software engineering, and agentic tool-use tasks.

The Illusion of Equivalency: Statistical Characterization of Quantization Effects in LLMs (1 minute read)

This paper shows that accuracy and perplexity miss the behavioral changes post-training quantization introduces in LLMs, proposing correctness agreement, a decision-level metric measuring overlap in correct predictions between base and quantized models. Testing 8-bit to 2-bit schemes, the authors find divergence emerges under moderate quantization and that query and key projections are consistently more sensitive than value and output projections.

Show HN: Getting GLM 5.2 running on my slow computer (1 minute read)

Colibrì is a single-file C engine that runs GLM-5.2, a 744B-parameter MoE model, on a consumer machine with about 25GB of RAM by keeping the dense layers resident at int4 and streaming the routed experts from disk on demand. It uses MLA attention with a 57x smaller KV-cache and GLM-5.2's native multi-token-prediction head for speculative decoding, with no BLAS, no Python at runtime, and no GPU required.

GitHub AI agent leaks private repositories via prompt injection attack (2 minute read)

Noma Security researchers disclosed GitLost, a prompt injection attack that tricks GitHub's preview Agentic Workflows into pulling content from private repositories and posting it in public comments. An unauthenticated attacker only needs to file a crafted GitHub issue on a public repo, and if the agent has read access to private repos in the same organization, it treats the hidden instructions as commands and leaks the data with no stolen credentials or malware required.

‘HalluSquatting’ Turns AI Hallucinations Into Botnet Delivery Mechanism (1 minute read)

Researchers from Tel Aviv University, Technion, and Intuit detailed HalluSquatting, an attack that pre-registers the fake repository and package names LLMs commonly invent, then plants malicious instructions inside them for coding assistants like Cursor, GitHub Copilot, and Gemini CLI to clone and execute. Hallucination rates reached 85% for repo-cloning prompts and 100% for skill installations, and the same invented names recurred across foundation models, letting attackers build agentic botnets that spread via prompt injection rather than traditional vulnerabilities.
