The Pulse: Interesting AI coding stats from Cursor (5 minute read)

Cursor released a report on two years of aggregated usage data, and the distribution is heavily skewed: the median developer generates about 700 lines of code per week, while the 90th percentile hits 9,000 and the top 1% produces 30-40K. Token usage runs 10:1 input to output, meaning most spend goes to reading the existing codebase rather than generating new code, and nearly half of AI changes are accepted without manual review.

Ways to think about token pricing (5 minute read)

Benedict Evans argues that foundation model providers are heading toward becoming low-margin commodity infrastructure rather than capturing durable pricing power. He notes today's supply crunch is transitory and driven almost entirely by one use case, software development, while over $1T in data center capex and rapidly improving inference efficiency come online. Inference currently runs at 40-50% gross margins, but that excludes training costs that still far exceed revenue, leaving long-term pricing unresolved.

Auto mode knows it's dangerous (and does it anyway) (1 minute read)

Coding agents will run `curl | sh` installers even when they correctly identify the risk. When asked to install air via a piped remote script, both Claude Code and Codex flagged the danger, then executed anyway: Claude Code disabled its sandbox because the user had explicitly requested it, and Codex auto-approved after rating the command medium-risk but high-authorization. The author's point is that recognizing a threat and deferring to explicit user intent are separate decisions, and auto mode collapses them.

What's really slowing down the AI buildout (5 minute read)

The AI buildout's real bottleneck isn't a shortage of electricity but the grid's inability to deliver it where new data centers are being built. The interconnection queue, which uses a rigid first-come, first-served process, has stretched from under 20 months for the median 2005 power plant to 55 months by 2023. The author argues that grids need flexible interconnection rules to keep pace with projects like OpenAI and SoftBank's $40B, 1.2-gigawatt Stargate campus.

Google's TabFM skips per-dataset training and still predicts on tables it's never seen (2 minute read)

Google Research introduced TabFM, a foundation model that treats tabular prediction as in-context learning and generates predictions for an unseen table in a single forward pass, skipping the per-dataset training that gradient-boosted models require. It sidesteps the LLM approach, which chokes on tables due to context limits, tokenization that splits numerical values, and loss of row-column structure when a 2D table is serialized to text.

Z.ai's Stable Asynchronous RL (5 minute read)

This paper introduces Single-rollout Asynchronous Optimization (SAO), a reinforcement learning method that replaces GRPO's group-wise sampling with one rollout per prompt and adds strict double-side token-level clipping to stabilize off-policy training. SAO trains stably for 1,000 steps and beats GRPO variants on SWE-Bench Verified, BeyondAIME, and IMOAnswerBench, and was used in Z.ai's pipeline for the open GLM-5.2 model (750B-A40B).

Flexible Video Diffusion (5 minute read)

NVIDIA's Flex-Forcing is a video diffusion method that lets a single model handle text-to-video, image-to-video, and video extension without task-specific retraining. It works by varying the noise schedule across frames during training, so the same model can condition on clean past frames or generate sequences from scratch at inference.
