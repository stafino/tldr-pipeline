The Economics of Recursive Self-Improvement [pdf] (1 minute read)

This paper examines recursive self-improvement, the scenario where an AI system iteratively upgrades its own capabilities, through an economic rather than purely technical lens. The authors argue that the pace of any self-improvement loop is bounded by the cost of compute, data, and experimentation at each step, so returns can diminish even when the underlying algorithms keep working. The framing reframes "intelligence explosion" debates as a question about input costs and marginal gains.

Playful AI in Professional Email: A Field Experiment on Tone and Recipient Engagement (1 minute read)

A randomized crossover field experiment had 121 employees across six companies send 16,880 work emails under three conditions: unaided, GPT-5 rewriting in a playful tone, and GPT-5 rewriting in a professional tone. Playful editing raised emotional positivity and professional editing lowered it, but neither directly changed open rates, reply rates, or response times. Instead, sender positivity predicted both opening (OR=2.05) and replying (OR=3.32), meaning AI shaped engagement only through the tone of the language it produced.

Distributed Denial of Science: How Indirect Data Poisoning of AI Systems Can Industrialize Scientific Fraud (1 minute read)

This paper describes indirect data poisoning, an attack in which an adversary corrupts an open dataset with misleading metadata and uploads it to a public repository, where autonomous research agents retrieve and process it, turning honest scientists into unwitting distributors of fraud. Across 450 runs on Claude Opus 4.7, GPT-5.5, and Gemini 3.1 Pro, poisoning succeeded 49.56% of the time while detection reached only 6%. The authors show that a data provenance audit with five checks cuts the attack success rate to zero.

Understanding the Impact of AI Code Assistants on Security API Usage: An Empirical Study (1 minute read)

A study of 44 professional developers completing security API programming tasks with and without GitHub Copilot found that the assistant improves functional correctness but does not significantly improve secure API usage. Developers rarely raised security concerns while using Copilot, and many failed to recognize that their final implementations remained insecure. It is the first empirical, developer-centered investigation of how AI assistants affect security API use.

Inside the Unfair Judge: A Mechanistic Interpretability Account of LLM-as-Judge Bias (1 minute read)

This paper argues that LLM-as-judge scoring bias can be read directly in a judge's hidden states, not just at the input-output level, testing across seven judges, seven bias types, and nine benchmarks. Extreme Event Likelihoods with Guided Generative Models (1 minute read)

NVIDIA researchers introduced a method for estimating extreme-event likelihoods by guiding a diffusion-based climate emulator toward rare states and computing odds ratios between guided and unguided distributions. Applied to tropical cyclones using NVIDIA cBottle, the approach corrects for the oversampling that guidance introduces, enabling importance sampling of rare events at far lower cost than brute-force Monte Carlo.

Agent Hacks Agent: Autoresearch for Production-Agent Red-Teaming (1 minute read)

This paper introduces AHA, a falsifiable discovery loop that automatically red-teams production LLM agents like Claude Code and Codex by proposing a vulnerability hypothesis, building a falsifier, running the attack in a sandbox, and promoting confirmed findings into a Vulnerability Concept Graph (VCG). A frozen VCG requires no further search and beats the strongest frozen discovery baseline by 14.2 percentage points under the same single-shot protocol, transferring across models, scenarios, and attack channels.

An Exact Instrument for State Usage in Selective State-Space Models, and the Input-Driven Migration It Reveals (1 minute read)

This paper introduces an exact instrument for measuring how trained selective state-space models like Mamba use their internal modes, decomposing each channel's output into per-mode contributions and predicting pruning error to a median relative deviation of 5×10⁻⁷ across 4,464 configurations. Across the Mamba-1 family, Falcon-Mamba 7B, and Mamba-2, the authors find models re-allocate state with the input, and input-scheduled pruning matches the unpruned model at half the state budget.

When Local Monitors Miss Compositional Harm: Diagnosing Distributed Backdoors in Multi-Agent Systems (1 minute read)

This paper shows that runtime monitors checking each agent message or tool call in isolation cannot catch distributed backdoors, where a harmful payload is split across agents so every local check passes while the assembled object is the attack. The authors prove an observability boundary: once fragments look benign in the monitored view, no detector on that view can catch them, and detection returns only when the monitor sees the reassembled payload.

AdvancedMathBench: A Benchmark Suite for Advanced Mathematical Proof Generation and Verification (1 minute read)

AdvancedMathBench is a benchmark suite for evaluating LLMs on advanced mathematical proof generation and verification, spanning undergraduate to doctoral qualifying-exam levels. Its ProverBench set of 296 problems remains hard for frontier models, with GPT-5.5-xhigh scoring only 75.8 and 66.1 on the two splits, while the best verifier reaches a Balanced F1 of just 65.1 and struggles to catch critical proof errors.

Proactive Memory for Long-Horizon Agents (5 minute read)

This paper introduces a proactive memory agent that runs alongside an unmodified action agent, updating a structured memory bank from the recent trajectory and injecting grounded reminders only when needed to counter "behavioral state decay" on long-horizon tasks. The plug-and-play module improved pass@1 by 8.3 points on Terminal-Bench 2.0 and 6.8 points on τ²-Bench, with ablations showing selective intervention beats always-on injection and general retrieval.
