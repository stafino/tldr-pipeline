How Claude's values vary by model and language (5 minute read)

Anthropic studied how the values Claude expresses shift across its models and the languages people use, compressing over 3,000 previously identified values into a small set of interpretable axes. Four axes, including Deference vs. Caution and Warmth vs. Rigor, capture 15% of the variation across the top 20 languages on Claude.ai. The approach could eventually link value differences back to specific character-training and fine-tuning decisions.

How I Turned AI to the Dark Side (3 minute read)

Security researcher Dave Kuszmar reports that a handful of simple prompting techniques let him bypass safety guardrails on nearly every major LLM, extracting instructions for making napalm, cooking methamphetamine, and bootstrapping uranium enrichment. He argues the restrictions meant to secure these models are the same levers attackers exploit, and that labs have been largely unresponsive to disclosures. He calls for slower deployment, more transparency, and large-scale safety research before further integration.

The Tower Keeps Rising (1 minute read)

The author argues that AI-assisted coding removes the wrong bottleneck. Large software projects have never been limited by how fast an individual can produce code, but by how well people coordinate their shared understanding of a system: its concepts, boundaries, invariants, and ownership. Much of that understanding was maintained by the friction of code review and cross-team conversation, and agents that let developers change code faster erode the very process that kept everyone speaking the same language.

Open-Weight Models Reached 29% of AI Gateway Usage (5 minute read)

Open-weight models ran 29% of AI Gateway tokens in June, up from 11% in April, while accounting for under 4% of spend. DeepSeek hit 22.6% of volume, less than two points behind Google, and newly released GLM 5.2 broke into the top models within two weeks. Anthropic still took 61% of spend on 32% of tokens, capturing 72% or more in every high-stakes use case. The pattern is routing discipline: cheap volume goes to open models, high-risk work stays on the frontier.

Control the ideas, not the code (5 minute read)

antirez argues that programmers now have less impact than they could because they still fixate on reading code rather than controlling their software's ideas. His case: LLMs generate more code than anyone can meaningfully review line by line, and they excel at locally optimal code while remaining weaker at the big design decisions. The better workflow is to prompt and interrogate the design you hold in your head, not to scan function by function. He stresses this is not vibe coding toward a final product.

Guardian Angels: LLM Personalization for Productivity and Security (1 minute read)

Gwern proposes "Guardian Angels": highly personalized LLMs that emulate a single user's values, personality, and preferences rather than acting as generic assistant chatbots, unifying principal and agent to weaken the principal-agent problem. The Agentic Loop: Three loops in a trench coat (1 minute read)

This post argues that the "agent loop" is really three nested loops working together: an inference loop that makes chat completion calls and manages history, a tool loop that executes tool-use requests, and an outer agentic loop that ties them into a coherent experience. It walks through pseudo-code for each, showing that LLM providers' stateless APIs force developers to persist chat history themselves and orchestrate the handoffs between inference and tool execution.

Towards demystifying the creativity of diffusion models (1 minute read)

Google Research published an ICLR 2026 paper arguing that a diffusion model's ability to generate novel data rather than memorize its training set follows mathematically from neural networks learning a smoothed version of the score function. That smoothing pushes generation to interpolate between training points along the hidden data manifold, which reframes creativity as a predictable side effect of how denoising is learned rather than an emergent property.

What building Shippy taught us about building agents (5 minute read)

The Allen Institute's Skylight team shared how they built Shippy, a maritime domain awareness agent, around a three-part structure: a soul (the system prompt and behavioral boundaries), skills (markdown files following the same agent-skills spec as Claude Code and Codex), and config. The soul and skills are baked into a versioned Docker image, while the harness (OpenClaw) and model (Claude Opus 4.6) stay in config, so swapping either is a config change rather than a rebuild.

Blocking Slow-Burn Attacks: Contextual Policies in Omnigent (1 minute read)

Databricks demonstrates contextual policies in Omnigent, a stateful guardrail that tracks risk across an entire agent session to block slow-burn prompt injection attacks that split data theft into individually harmless steps. The single policy stops an agent from sending sensitive data once it has read too much, and the agent cannot disable or weaken the guard, since removing a policy requires human approval and any denial wins when policies combine.

Form, Not Content? A Preregistered, Placebo-Controlled Evaluation of Learned Error-Conditioned Self-Repair Through Prompts and Weights in Frozen Small Code Models (1 minute read)

This paper introduces PoPE (Popperian Placebo-controlled Evaluation), a preregistered methodology that tests whether frozen small code models (0.5-1.5B parameters) actually use execution error content during self-repair by pairing real error feedback against content-ablated placebos. Across both a prompt channel and an adapter-training weight channel, placebo arms matched or beat live error feedback, so content-attributable improvement in self-repair was not confirmed.
