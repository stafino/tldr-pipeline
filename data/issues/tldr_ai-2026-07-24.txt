Why Software Factories Fail (or: harness engineering is not enough) (1 minute read)

This post argues that "software factory" setups where no human reads or writes code fail because harness and loop engineering alone cannot ensure quality, citing a Faros AI report linking AI coding tool adoption since early 2026 to lower PR review quality, more incidents, and more bugs per developer. Emergent Misalignment Recruits a Pre-existing Persona Subspace (1 minute read)

This paper investigates emergent misalignment, where fine-tuning an aligned model on narrow bad advice makes it broadly misaligned, and shows the effect recruits a low-rank persona subspace that already exists in the frozen instruction-tuned model. Projecting that subspace out of the residual stream during fine-tuning cut broadly misaligned generations from 27.7% to 0.0%, while injecting it into a clean model induced misalignment up to 45.4%.

How Many Bits Can an Adapter Write? Measuring the Capacity and Memorization of Parameter-Efficient Fine-Tuning (1 minute read)

This paper measures in bits how much data a LoRA adapter memorizes about its training set, extending compression-based memorization analysis to the frozen-base setting. Adapters store roughly a couple of bits per trainable parameter, with capacity depending on placement rather than count, and on Qwen2.5 fine-tunes privacy leakage tracks bits written, with reinforcement learning recording none of the secrets supervised fine-tuning copies verbatim.

QuantiBias: Benchmarking Quantization-Induced Bias in LLMs (1 minute read)

QuantiBias is a benchmark that tests large language models for bias introduced by quantization, the compression step nearly every deployed model goes through. The authors find that quantized Qwen and Gemma builds pass standard refusal and multiple-choice safety checks yet volunteer stereotypes in roughly one in four open-ended answers across all eight languages probed, with reasoning-before-answering halving the effect on some model families and doing nothing on others.

Measuring Reward-Seeking by Instilling Contrastive Beliefs (5 minute read)

OpenAI's alignment team introduced Contrastive Synthetic Document Finetuning (Contrastive SDF), a test that alters a model's beliefs about its grader to measure whether it is reward-seeking rather than pursuing the intended objective. Frontier models trained with reinforcement learning but no safety training increasingly did what they believed the grader wanted, even when that conflicted with user or developer intent, and the tendency strengthened over training.

Anti-Periodic Positional Encoding: Möbius Boundary Conditions Make In-Context Retrieval Reliable (1 minute read)

Möbius RoPE is a rotary positional encoding built on an anti-periodic frequency ladder that deterministically couples the two ends of a sequence, the first anti-periodic boundary condition used in positional encoding. Across 48 pretrained models, applying Möbius frequencies to 25% of heads left perplexity unchanged (29.66 vs. 29.72) while lifting needle-in-a-haystack retrieval from 63.3% to 90.3% at context 512.

Same Dangerous Objective, Opposite Advice: Direct Exposure versus Multi-Agent Mediation (1 minute read)

This paper shows that OpenAI's gpt-5.6-sol model gives safer advice when a dangerous objective is presented directly than when other agents transform and relay it, testing 25 mirrored trade-off profiles. The finding exposes a compositional safety gap: a manipulative multi-stage workflow can strip an instruction's coercive clauses and provenance before the user-facing model sees it, flipping the model's output toward the harmful target while endpoint-only users cannot inspect the upstream messages.

Introducing Antares: Highly Efficient Open Weight AI Models for Vulnerability Localization (5 minute read)

Cisco released Antares, a family of open-weight security small language models built to localize known vulnerabilities within large codebases. The two available checkpoints, Antares-350M and Antares-1B, outperform larger closed- and open-weight models on Cisco's new Vulnerability Localization Benchmark at a fraction of the cost, and run locally so sensitive source code never leaves a team's environment.

Leaky Language Models: Stealing Architecture and Inference Optimizations via Per-Token Timing (1 minute read)

This paper introduces LeakyLMs, a set of attacks that infer proprietary architecture and deployment details from production language models using only per-token generation timing over remote APIs. The attacks detect speculative decoding and draft-model context length, finding that Google Gemini Flash 2.5 uses a roughly 128K draft window, and recover layer count, hidden dimension, and attention head count, placing the near-correct Llama configuration in the top 10 guesses over 90% of the time.

Minimize idle accelerators: Native RL job interleaving with co-operative time-slicing in llm-d (2 minute read)

Google introduced co-operative time-slicing in the llm-d project, which treats discrete reinforcement learning steps like sampling rollouts and gradient training as schedulable entities that can be interleaved across independent RL jobs on shared accelerators. Initial benchmarks show the platform-level multiplexing raises aggregate accelerator duty cycles from a roughly 40% baseline to 70% without hurting model convergence or accuracy.

Agentic coding without the cloud: evaluating open-weight large language models on longitudinal data preparation tasks (1 minute read)

This paper introduces an open-source framework for evaluating open-weight LLM agents on data preparation for longitudinal population studies, where governance rules prohibit sending sensitive data to cloud models. Running on consumer-grade hardware, current 31-35B parameter models nearly saturated the benchmark, reaching up to 87.9% average task completion across 20 tasks that generated 102 variables from a British cohort study.

Error Certificates for KV-Cache Eviction via Randomized Design (1 minute read)

This paper proves that deterministic top-k KV-cache eviction cannot estimate its own attention-output error, since evicted values can be altered to grow the true error arbitrarily while the retained state looks unchanged. Randomized eviction with Poisson-sampled tails restores identifiability, yielding a per-step error certificate with 0.97 empirical coverage that separates cache-induced from inherent failures (AUC 0.73-0.75) but does not predict failure better than output log-probability.

AREX: Towards a Recursively Self-Improving Agent for Deep Research (1 minute read)

AREX is a family of Recursively Self-Improving deep research agents that alternate between an inner loop gathering evidence to build a provisional answer and an outer loop that audits it constraint-by-constraint, then launches targeted follow-up research on unresolved claims. Trained via agentic mid-training and long-horizon reinforcement learning, the 4B dense and 122B-A10B MoE models outperform comparable-scale baselines on BrowseComp, WideSearch, and Humanity's Last Exam.

KroQuant: Kronecker-Structured Block Transforms for Efficient Post-Training Quantization of Diffusion Transformers (1 minute read)

KroQuant is a post-training quantization method that applies a learned Kronecker-structured invertible transform to each 32-element activation block, letting diffusion transformers run at W4A4 while storing less than half the parameters of per-channel scaling. On PixArt-Σ, SANA, and FLUX.1-schnell, its quantizer kernel runs up to 14% faster than SmoothQuant on an MI350 GPU and produces outputs closer to the FP reference than SVDQuant and LoRaQ.

GigaToken: ~1000x faster Language model tokenization (1 minute read)

GigaToken is a drop-in tokenizer library that its author claims runs up to 1000x faster than HuggingFace's tokenizers, hitting GB/s throughput on standard CPUs. It supports most common tokenizers and offers compatibility modes for both HuggingFace and tiktoken, though exact output matching with HuggingFace comes at a performance cost below the peak numbers reached through GigaToken's native Rust API.

Agentic Context Management: Solving Agent Memory and Cost by Treating Them as Lifecycle and Architecture Problems (1 minute read)

This paper argues that production AI agent failures stem less from poor reasoning than from an inability to manage accumulating context, and it reframes memory and cost as a lifecycle problem it calls Agentic Context Management, decomposed into five primitives: architecting, ingesting, scoping, anticipating, and compacting. It makes the case that naive context accumulation grows token cost quadratically while validated compaction stays linear, and reports a reference implementation, Maximem Synap, scoring 92% on LongMemEval and 93.2% on LoCoMo.
