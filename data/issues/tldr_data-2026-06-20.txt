Caching for Dollars, Not Hits: An Exact Offline Reference for Cloud-Egress Caching and the Crossover That Decides When It Pays (1 minute read)

This paper reframes cache design around dollars instead of hit rate, arguing that when misses fetch from cloud object storage, the bill is per-GET and per-byte egress, so a rarely fetched but expensive object can cost thousands of times more than a frequent cheap one. The authors build an exact offline dollar-optimum benchmark and show cost-aware GreedyDual cuts LRU's regret to roughly a tenth. A closed-form crossover, GET_fee/egress_rate (about 4KB on S3, 330B on GCS), predicts which deployments actually need dollar-aware caching.

Query-aware Routing for Filtered Approximate Nearest Neighbors Search (1 minute read)

Filtered ANN search pairs vector similarity with attribute predicates, and this paper benchmarks every major categorical method across datasets and three predicate types, finding no single winner even within one dataset. The authors propose query-aware routing: a lightweight model predicts each method's recall per query, then a router consults an offline recall/QPS table to pick the best trade-off. Trained on six datasets and tested on five unseen ones, it leads existing baselines with negligible latency overhead.

How we built an internal data analytics agent (1 minute read)

GitHub built Qubot, a Copilot-powered analytics agent that lets any employee query the company's data warehouse in plain language and get an answer in seconds. The architecture splits into three parts: a UI layer (Slack, VS Code, Copilot CLI), a context layer that encodes which data model, grain, and filters to use, and a query engine. Every answer is saved as a markdown report in a pull request, so teams can refine the query or promote it to a dashboard.

When failover isn't safe: Building high-availability PostgreSQL on Kubernetes (5 minute read)

Datadog rearchitected its Kubernetes-based PostgreSQL clusters to use Patroni-coordinated synchronous replication after a gameday exposed an unsafe failover mode. A simulated zonal failure spiked network latency, stalling replication so far that no replica was current enough to promote without data loss, leaving clusters stuck waiting for latency to subside. The writeup covers how they traded durability against latency and validated the design through benchmarking and failure testing.

The Bi-Channel Networking Paradigm for Database Systems in the Cloud (1 minute read)

This paper argues database systems should stop treating the network as a black box and co-design it with database operations. The proposed bi-channel paradigm splits communication into a high-performance user-space UDP data path for latency- and bandwidth-sensitive work and a kernel TCP control path for coordination and recovery, exploiting modern NICs without reimplementing reliability. The authors saturate 200 Gbit/s on a distributed shuffle with three CPU cores and push millions of messages per second through a replicated key-value store.

AgenticDB: Agentic Performance Reconfiguration for Database Workloads (1 minute read)

AgenticDB reframes database tuning as an agentic reconfiguration loop rather than iterative knob search, proposing both DBMS- and OS-level changes, applying them under safety constraints, and using runtime feedback to diagnose bottlenecks and guide later decisions. Tested on MySQL and PostgreSQL across YCSB, Sysbench, and TPC-H, it beat the strongest baseline by 118.1% on average while cutting time-to-best by 22.6%. The gains came from the OS-level action space, a robust execution lifecycle, and memory that carries experience across tasks.

Source-Grounded Data Generation for Text-to-JSON Learning (1 minute read)

STAGE is a synthetic-data pipeline for training models to extract structured JSON from long unstructured documents like financial filings and clinical records. It uses LLMs to synthesize reports and JSON schemas at scale while validating every ground-truth value against the underlying spreadsheet, so the training data stays accurate. Fine-tuning Qwen3-4B on STAGE data lifted exact match from 31.37% to 74.27% and value accuracy from 45.46% to 90.69% on the authors' 851-example benchmark.

The only scalable delete in Postgres is DROP TABLE (5 minute read)

PlanetScale argues that the only Postgres delete that actually scales is dropping or truncating an entire table, because MVCC makes large DELETEs "work added" rather than work done. Big batch deletes leave dead tuples behind, don't return disk to the OS until a lock-heavy VACUUM FULL, skip index cleanup, and add write and replication load that can stall other writers. The takeaway is a schema-design one: partition or shard so bulk cleanup becomes a DROP TABLE or TRUNCATE instead of a DELETE.

Policy-aware Vector Search: A Vision for Fine Grained Access Control in Vector Databases (1 minute read)

This paper proposes fine-grained access control for vector databases, formalizing a policy model and enforcement problem for the RAG and organizational AI pipelines that increasingly query embeddings in security-sensitive contexts. The authors show the core tension: enforcing per-user policies correctly while keeping ANN recall high and query latency low, since vector search mixes structured filters with approximate semantic results in ways relational FGAC patterns do not handle. The work compares enforcement strategies and lays out open problems rather than shipping a finished system.

Beyond Accuracy: Measuring Logical Compliance of Predictive Models (1 minute read)

This paper introduces the Rule Violation Score (RVS), a metric that measures how well a predictive model respects predefined logical constraints, independent of its accuracy. RVS handles hard and soft rules separately, works on any model over a relational vocabulary, and compiles to auto-generated SQL queries for Horn rules. The authors show that two models with comparable accuracy can diverge sharply in logical compliance, a gap standard metrics miss and a real concern for healthcare, finance, and autonomous systems.

Vibe Coding for Visualization Implementation: An Empirical Study of Practices and Challenges (1 minute read)

This paper studies how people actually use AI "vibe coding" tools to build data visualizations, running 16 participants through two viz tasks plus interviews to map their prompting, evaluation, and iteration habits. The finding that matters for data teams: visualization is a worse fit for natural-language generation than general coding, because charts demand precise intent-to-pixel alignment that users struggle to specify and verify, leaving them stuck in iteration loops when the output looks plausible but encodes the data wrong.

DeQL: A Decision Query Language for Prescriptive Analytics over Relational Data (1 minute read)

DeQL extends SQL to express decision queries, adding two constructs: CREATE CANDIDATES, which defines the option space from relational sources, and DECIDE, which declares decision variables, named constraints, and an objective. The user states what to optimize and the engine picks the solver, keeping the structure of subset selection, allocation, scheduling, and assignment problems visible inside the warehouse. This first specification fixes the core constructs and adds extensions for optimization under uncertainty, inline model scoring, and time-bounded solving.
