SemCEB: A Cardinality Estimation Benchmark for Semantic Operators (1 minute read)

SemCEB is the first benchmark for cardinality estimation over semantic operators, the LLM-backed SQL filters and joins whose predicates are written in natural language. It spans 102 hand-curated queries across a range of selectivities, testing filters and joins in isolation. The stakes are cost: per-tuple LLM calls make bad plans orders of magnitude more expensive. Sampling proved robust but slow and pricey, while Semantic Histograms ran cheaper but stayed sensitive to predicate type.

When Is a Columnar Scan Bandwidth-Bound? A Decode-Throughput Law and Its Cross-Hardware Validation (1 minute read)

This paper offers a one-parameter law for when a columnar scan is actually memory-bandwidth-bound rather than compute-bound. The key finding is that a decoder's value throughput is independent of bit-width, so the achieved bandwidth fraction follows f = min(1, T_dec·b/8β), with the compute-to-bandwidth ridge at b* = 8β/T_dec. Fitting one parameter per strategy reproduced measured bandwidth fractions within median error 0.027 on x86/AVX2 and 0.003 on a held-out Apple M4.

The Table Says Otherwise: Testing LLMs with Counterfactual Relational Data (1 minute read)

This paper introduces ContraTable, a benchmark that tests whether LLMs answer table questions from the provided data or from memorized facts. It pairs an original database with a counterfactual version that keeps the same schema but rewrites country, club, and player attributes across 214 matched questions. Models handle single-table lookups but degrade on joins, comparisons, and temporal reasoning, falling back on pretrained knowledge when the table conflicts with familiar facts. The takeaway: table-QA evaluation should measure faithfulness, not just accuracy.

SelPE: Progressive Selection for Private Structured Text Synthesis (1 minute read)

SelPE is a framework for generating synthetic structured text, like clinical triage notes or transaction logs, under differential privacy when only a handful of real examples exist. Instead of noisy aggregation or private model training, it spends the privacy budget on a sequence of top-1 selections and splits generation into semantic abstraction and schema realization. The authors report better structural validity, fidelity, and downstream utility than tabular or text-only DP methods in low-data regimes.

A Compositional Language for Property Graphs (1 minute read)

This paper argues that the newly standardized graph query languages GQL and SQL/PGQ have a core flaw: they aren't compositional, and despite handling reachability and all first-order queries, they can't express everything in NLOGSPACE. The authors propose a fix built on regular path queries with variables and a graph-to-graph language called #Datalog that can construct new nodes, edges, and paths, then map it to concrete additions the standards committee could adopt.

A Set-Theoretic Approach to Detecting Logic Bugs in DBMS Inner Join Optimizations (1 minute read)

This paper introduces JoinEquiv, a metamorphic testing tool that catches logic bugs in INNER JOIN optimization by rewriting a query into semantically equivalent forms using set-theory transformations, then flagging any case where the results diverge. Running it against MySQL, TiDB, DuckDB, and Percona surfaced 29 previously unknown issues, 27 now officially confirmed. The takeaway for data teams: optimizer correctness, not just crashes or latency, is a live failure mode worth testing against.

SQLConductor: Search-to-Policy Learning for Step-wise Text-to-SQL Orchestration (1 minute read)

SQLConductor proposes step-wise orchestration for text-to-SQL, training a policy model to pick the next subtask action based on intermediate artifacts and feedback rather than committing to a fixed pipeline upfront. The authors use Monte Carlo Tree Search to explore candidate workflows, then distill the robust patterns into a deployable policy via stability-weighted fine-tuning and curriculum RL. A compact policy coordinating frozen larger models hits 73.2% execution accuracy on BIRD-Dev, beating systems that train bigger SQL backbones directly.
