Introducing Flink's Native S3 FileSystem: Built for Performance, Designed for Production (5 minute read)

Apache Flink 2.3 introduced flink-s3-fs-native, a Hadoop-free S3 filesystem built specifically for Flink that ships as an experimental opt-in plugin and supports both RecoverableWriter for exactly-once sinks and a fast read path for checkpointing. The native plugin ends the long-standing choice between the Hadoop and Presto plugins, drops the hadoop-common dependency tree and AWS SDK v1, and delivers measurable performance gains already running in production at scale.
