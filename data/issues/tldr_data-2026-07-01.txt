From monolith to Lakebase to LTAP: rethinking the database from storage up (1 minute read)

Databricks engineer Reynold Xin walks through Lakebase, the company's serverless Postgres, by decomposing a monolithic database into independent services: the write-ahead log that makes commits fast and the data files that make reads fast, externalized rather than pinned to one machine. The payoff is LTAP, where transactions and analytics run on a single copy of the data in real time, dropping the CDC and mirroring layers most teams bolt on to move OLTP data into the warehouse.

How Far Do On-Prem Open LLMs Get on Text-to-SQL? A Cross-Family Size x Technique Frontier on BIRD (1 minute read) [flag]

This paper benchmarks on-prem open-weight LLMs on Text-to-SQL using BIRD's dev split, testing Qwen2.5-Coder, CodeLlama, and Llama-3.x under one matched protocol with McNemar significance tests. Model generation beats raw size: Qwen2.5-Coder 7B scores 39.1 vs CodeLlama's 20.9, while Llama-3.3-70B hits 49.2. Large Databases Need Small, Open-Weight Language Models (1 minute read)

This paper argues that closed-source LM APIs are unnecessary for language-model-enhanced database operators, where token-based pricing can push a single set of experiments past $10,000. The authors show that quantized, open-weight models running locally on 16GB of VRAM match or beat proprietary APIs on accuracy at lower latency. Integrated into BlendSQL v0.1.0, the local setup cut costs 390x and latency 3.8x versus a proprietary API.

CLIP: Lightweight Cosine-Law-Based Inverted-List Pruning for IVF-Based Vector Search (1 minute read)

CLIP is a pruning technique for IVF-based vector search that cuts the cluster and vector scans those indexes normally do exhaustively, eliminating irrelevant clusters in O(1) time and filtering vector batches in logarithmic time using cosine-law lower bounds. The paper reports up to 78% pruning and 69% higher query efficiency over static IVF baselines, plus an LSM-inspired variant that defers index maintenance to background compaction for 141% higher throughput on dynamic workloads.

MaDI-Bench: An End-to-End Data Integration Benchmark (1 minute read)

MaDI-Bench is the first benchmark to evaluate data integration end-to-end across the full pipeline of schema matching, value normalization, entity blocking, entity matching, and data fusion, rather than scoring each step in isolation. The authors validate it against human-engineered, best-of-breed, and LLM-based pipelines, and include a method for generating task variants to slow benchmark saturation as systems improve. It measures both step-wise and end-to-end performance, with all artifacts public.

Mandol: An Agglomerative Agent Memory System for Long-Term Conversations (1 minute read)

This paper introduces Mandol, an agent memory system that replaces the usual mix of separate vector and graph databases with a single memory-native store, fusing key-value, vector, and graph structures behind unified hybrid retrieval operators. By keeping LLMs out of the retrieval path and adding query-adaptive routing, denoising, and token-constrained context generation, it cuts cross-database I/O. The authors report best overall accuracy on LoCoMo and LongMemEval, plus a 5.4x retrieval and 4.8x insertion speedup under 10 QPS on consumer hardware.

When LLMs Read Tables Carelessly: Measuring and Reducing Data Referencing Errors (1 minute read)

This paper presents the first systematic evaluation of data referencing errors (DREs) in LLMs reading tables, where models cite or omit cell values incorrectly even when they understand the table structure. The errors show up in every model tested, from 1.7B to 20B parameters, and corrupt intermediate reasoning steps rather than just final answers. Using a lightweight 4B critic to filter and reject-sample outputs lifted answer accuracy by up to 12% and hit a 78.2% F1 at catching the errors.

Experience Graphs: The Data Foundation for Self-Improving Agents (1 minute read)

This paper argues that long-horizon agents produce a structured "experience graph" of artifacts, tool outputs, rewards, and causal lineage that most frameworks throw away as JSON checkpoints and session logs. The authors propose Trellis, a database that treats that graph as queryable, governed state, recasting frontier selection as a query, cross-session reuse as vector-seeded graph retrieval, and training-data extraction as a materialized view. In Meta's KernelEvolve kernel optimizer, that reuse hit target speedups roughly 10x faster at 52% lower token cost.

Making Multimodal LLMs Reliable Chart Data Extractors: A Benchmark and Training Framework (1 minute read)

This paper benchmarks how well multimodal LLMs reverse-engineer data tables from chart images without visible value labels, and finds they reconstruct table structure reliably but miss on precise numeric values. The authors then train a model using a progressive, human-like chart-reading process, reaching state-of-the-art numerical accuracy with just a 7B-parameter model and supporting mixed-initiative extraction workflows. For data teams, it points to a cheaper path for turning published charts back into queryable tables.

Reading the internals of Postgres: Database cluster, databases, and tables (1 minute read)

A developer's working notes from Hironobu Suzuki's "The Internals of PostgreSQL" walk through how Postgres organizes storage: a database cluster is a group of databases managed by one instance, not a set of nodes, and every object is tracked by an OID in system catalogs that are themselves just regular tables. The useful detail is that pg_database is shared cluster-wide while most catalogs are per-database, and user objects start at OID 16384 with the lower range reserved for init objects.
