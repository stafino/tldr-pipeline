How to achieve pruning when querying by non-partitioned columns in PostgreSQL (5 minute read)

Partition pruning normally only works when you filter by the partition key, which makes key selection a high-stakes decision. This walkthrough shows how to get PostgreSQL to prune partitions even when querying on non-key columns, using a session/event table partitioned by timestamp as the running example. The trick is exploiting correlations in the data so the planner can still derive a timestamp predicate and skip whole partitions.

Google's TabFM skips per-dataset training and still predicts on tables it's never seen (2 minute read)

Google Research introduced TabFM, a tabular foundation model that reframes prediction as in-context learning, generating results for an unseen table in a single forward pass with no per-dataset training. It targets the operational debt of the gradient-boosted-tree workflow, where every dataset means its own feature engineering, hyperparameter search, and drift-driven retraining pipeline. The pitch for data teams is collapsing weeks of that pipeline work into an API call, though production reliability against a tuned XGBoost baseline remains the open question.

Introducing Feature Views (1 minute read)

Databricks released Feature Views in Public Preview, a managed framework for authoring an ML feature once and letting the platform serve it across experimentation, batch training, and real-time inference. A single definition computes both training and serving values, closing the train-serve skew gap that breaks most real-time ML, and switching a feature between batch and streaming sources takes only a few lines of code.
