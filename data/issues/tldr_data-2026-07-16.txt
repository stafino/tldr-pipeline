Introducing Apache Spark 4.2 (1 minute read)

Databricks released Apache Spark 4.2, adding metric views for a native semantic layer in Spark SQL, vector and top-K retrieval primitives, first-class change data capture, and an Arrow-first Python path. Metric views are the notable piece for data teams, making dimensions and measures first-class objects so non-additive measures like ratios and retention keep their intended aggregation semantics across BI, SQL, and agents. Available now in Databricks Runtime 19 Beta.

Routing it through the gateway means the same permissions, audit logging, and budget controls already applied to proprietary models, and teams can fine-tune on their own codebases instead of paying per-token API pricing.

Making 768 servers look like 1 (1 minute read)

PlanetScale walks through scaling Postgres from a single node to 768 servers holding a petabyte, using sharding as the only viable path past a few terabytes. The piece grounds the argument in the Universal Scalability Law: resource contention makes vertical scaling sub-linear, and past a point incoherence degrades performance outright, which caps how far a bigger box and read replicas can carry you. The operational goal is keeping that fleet addressable as one logical database.

GRAFT: Graph-Matched Retrieval and Fusion of Tables in Data Lakes (1 minute read)

GRAFT reframes table retrieval in data lakes as a graph-matching problem, pairing a query-derived intent graph against a heterogeneous lake graph so that joinable and unionable tables are pulled together rather than scored in isolation. The authors argue that point-wise retrieval hands downstream data agents fragmented evidence they cannot integrate. On Spider and BIRD adapted to the tabular lake setting, GRAFT reports 7.8% higher F1 and 10.6% higher Sufficiency than the strongest baseline while keeping search efficient.

Data-Native AI Agents: Why Agents Must Move to Your Data (1 minute read)

The authors argue that enterprise AI agents should run inside the lakehouse rather than in a parallel AI stack wired back to it, since exporting data to external vector databases and SaaS LLMs forces governance to be rebuilt on the side. On-prem DBaaS in 2026: Platforms, standards, and gaps (1 minute read)

The author argues that database provisioning is still the unsolved piece of platform engineering, with teams forced to choose between running Postgres, MariaDB, or Redis themselves on Kubernetes operators or depending on a platform team's Crossplane compositions that provision cloud-managed services like RDS underneath. Neither pattern delivers a consistent on-prem experience, so organizations keep rebuilding the same DBaaS layer. The piece surveys where current platforms and standards fall short.

Policy-Conditioned Constrained Decoding for Column-Level Access Control in Text-to-SQL (1 minute read)

This paper introduces PCC-SQL, a text-to-SQL system that enforces column-level access policies during decoding rather than by refusing queries after the fact. It defines policy over how a column is used (output, filter, aggregation) and applies a per-token logits mask that deterministically blocks violations in a single pass. Across three benchmarks and three open models, it hit a 0% leakage rate and up to 88.7% coverage on Spider-CU while staying within 10% of direct-prompting token cost.

Let's Build PlanetScale from Scratch: Infrastructure (1 minute read)

The author is building Homescale, an open-source clone of PlanetScale's branching model that creates writable database instances and point-in-time branches from immutable snapshots without copying the underlying files. The design borrows Docker's image/container split: an immutable image, a writable container cloned from it, and branches forked from a container's current state. Engine-specific logic sits behind an adapter, so the storage model works for any database that keeps durable state on a block device.

GraphPolaris: A System for Query, Analysis, and Visualization of Graph Databases (1 minute read)

GraphPolaris is a no-code visual analytics system for querying and visualizing graph databases, built around GPQL, a formal query grammar that generates valid, database-agnostic queries from user interactions rather than hand-written Cypher or Gremlin. The authors position it as the graph equivalent of Tableau or Looker, targeting fraud detection, supply-chain, and social-network analysis where tabular BI can't reason over nodes and edges. Evaluation spans telecom and supply-chain case studies and a 22-month formative study with analysts.

How to solve PostgreSQL multilingual full-text search limitations with AlloyDB AI (2 minute read)

AlloyDB AI Functions now let you call Gemini directly from SQL to tokenize continuous text, fixing PostgreSQL full-text search for Chinese, Japanese, and Korean where whitespace-based parsers collapse whole sentences into one useless lexeme. The pitch is skipping the ETL layer entirely, since row-wise Gemini API calls over large tables were the fragile part, and segmentation now happens inline before to_tsvector builds the index.

Regatta launches its unified OLTP, OLAP and vector database (5 minute read)

Regatta shipped RegattaDB into general availability, a distributed SQL database that runs OLTP, OLAP, and vector workloads on one serializable concurrency model and drops in as a Postgres replacement. The company benchmarked a distributed JOIN across 20B rows on 50 commodity nodes in three minutes while sustaining 50,000 ACID updates per second on the same dataset, which is the pitch for cutting the ETL hop between operational stores and the warehouse.
