Open Knowledge format v0.2 tackles agentic trust (3 minute read)

Google released Open Knowledge Format v0.2, which adds optional frontmatter fields for provenance, trust, freshness, lifecycle, and attestation so that agents consuming a knowledge bundle can judge each concept written by another agent. The format stays minimally opinionated, with type as the only required field and every new field opt-in, so a v0.1 bundle that adopts none of the additions remains valid.
