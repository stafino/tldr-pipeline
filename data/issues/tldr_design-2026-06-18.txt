How I migrated a large Next.js project from Tailwind CSS v3 to v4 (3 minute read)

Tailwind CSS v4 migration is not a one-command upgrade, as this walkthrough of moving a 180-component Next.js marketing site shows: the config file disappears entirely, design tokens move into an @theme block in CSS, and utilities like bg-opacity-* and the transform stubs are renamed or removed. The payoff is concrete, with build times dropping via the new Rust engine and CSS output shrinking roughly 18% after redundant @layer scaffolding goes away.
