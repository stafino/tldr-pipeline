The Shifting Line Between CSS States and JavaScript Events (1 minute read)

CSS keeps absorbing behavior that once required JavaScript event listeners, and this walkthrough maps the pseudo-classes that now track interaction states: :hover, :active, :focus-visible, :focus-within, and :has(). It also covers the proposed event-trigger from the Animation Triggers spec, an unsupported feature that would let CSS respond to events directly, with a useful clarification that pseudo-classes match states rather than events even when they feel like listeners.
