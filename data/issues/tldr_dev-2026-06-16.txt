Configuration Smells in AGENTS.md Files: Common Mistakes in Configuring Coding Agents (1 minute read)

This paper presents the first catalog of "configuration smells" in AGENTS.md and CLAUDE.md files, the instruction files that steer coding agents on architecture, conventions, and testing. After mining repositories and reviewing 100 popular open-source projects, the authors define six smells with automated detection heuristics, finding Lint Leakage (linter rules duplicated into agent instructions) the most common at 62% of files.

Organizational Cohesion in Microservice Architectures: A Multi-Project Empirical Study (1 minute read)

This paper introduces organizational cohesion, a measure of how closely developer contributions align with microservice boundaries, operationalized as Pairwise Team Cohesion (PTC) by adapting the Sensitive Class Cohesion Metric. The authors validate it through a longitudinal study of Netflix's Spinnaker platform and six other open-source microservice systems, finding systematic divergences between service decomposition and the team structures that actually maintain those services.

The time the x86 emulator team found code so bad they fixed it during emulation (1 minute read)

Raymond Chen recounts how Windows' x86-on-ARM emulation team hit application code so pathologically inefficient that the just-in-time translator effectively rewrote it on the fly, producing emulated output that ran better than the original native instructions. The post uses the case to explain how binary translation can recognize and collapse redundant instruction patterns, turning a quirk of bad code into a window onto how the emulator's optimizer actually works.

Game Engine White Papers: Commander Keen (1 minute read)

This post dissects how Commander Keen pulled off smooth side-scrolling on 1990 PC hardware that was never designed for it, using John Carmack's adaptive tile refresh to redraw only the screen cells that changed between frames. The write-up reconstructs the EGA memory tricks, off-screen page buffering, and tile bookkeeping that let the engine hit console-grade scrolling where competitors settled for full-screen redraws.

LLM-as-Code Agentic Programming for Agent Harness (1 minute read)

This paper argues that token explosion, control-flow hallucination, and unreliable completion in LLM agents are architectural consequences of making a probabilistic model orchestrate deterministic looping, branching, and sequencing. The authors propose Agentic Programming, where the program owns all control flow and the LLM is invoked as an adaptive component only for reasoning or generation, with each call's context built from the execution history's call tree as a DAG.

Agent trajectories as programs: fingerprinting and programming coding-agent behavior (1 minute read)

This paper introduces methods for fingerprinting coding agents by their procedural behavior rather than benchmark scores, using a compressive vocabulary-induction technique to represent how an agent solves problems. Across ten agents on SWE-Bench, a probe over these procedural signatures attributes an unseen trajectory to the correct agent with 85.7% accuracy, and behavior clusters most tightly among models from similar release periods.

SPARK: Security Knowledge Priming and Representation-Guided Knowledge Activation for LLM-based Secure Code Generation (1 minute read)

SPARK is an inference-time security harness that improves LLM secure code generation by retrieving relevant CWE entries and appending a short structured cue to the prompt, activating latent security knowledge without any retraining. The authors argue pretraining corpora already contain rich security material and that the real bottleneck is activation, positioning SPARK against heavier defenses like fine-tuning or external knowledge retrieval that add compute overhead and data bias.

Snyk VulnBench JS 1.0: Can LLMs Find the Same Bugs Twice? (1 minute read)

This benchmark ran 300 repeated agentic LLM security scans on identical JavaScript, prompts, and harness, finding that reference-matched vulnerabilities were highly repeatable (134 of 158 appeared in all five runs) while unmatched extra findings were not, with 80 of 161 surfacing in only one of five repetitions. The authors argue LLM review and Snyk's deterministic SAST are complementary: models reliably catch high-signal exploit shapes, while static analysis is better at systematically enumerating repeated data-flow sinks.

How TimescaleDB compresses time-series data (1 minute read)

The post breaks down TimescaleDB's Hypercore engine, which converts row-based chunks into a columnar format by packing up to 1,000 rows into a single compressed batch and applying per-type schemes like delta-of-delta, Gorilla, and dictionary encoding. It covers how segment-by and order-by configuration shapes the compression ratio and query performance, with the trade-off that updates to compressed data are slower than on uncompressed chunks.

Selection Without Signal, Recovery Through Expression: A Measurement Study of Post-Hoc Falsification Operators for Frozen Small Code Models (1 minute read)

The Ghosts of Polymarket: When Off-Chain Matches Meet On-Chain Reverts (1 minute read)

Researchers analyzed Polymarket's hybrid architecture, which matches orders off-chain but settles on-chain, exposing a consistency gap they call Ghost Fills, where matched orders fail at settlement. Their GHOSTHUNTER tool reconstructed 1,952,440 reverted transactions and attributed them to four attack vectors—nonce bump, balance drain, allowance revoke, and proxy trap—realized across 35 variants that invalidate orders before finalization.

Graphical-Probabilistic Modeling of Generative Flows in LLM-Native Software Systems (1 minute read)

This paper proposes Generation Networks, a graphical probabilistic modeling framework for documenting the stochastic, prompt-dependent generative flows in LLM-native software systems. The authors argue that current practice relies on low-level prompting and context-engineering heuristics that lack the modularity and abstraction needed for design-level reasoning, and offer the framework as a foundation for stating and analyzing properties of LLM-based designs.

C^2: Cache-Conscious Succinct Tries with Adaptive Unary Path Compression (1 minute read)

C² introduces two optimizations for succinct tries: a cache-friendly bitvector layout that cuts cache misses during trie navigation, and adaptive unary path compression that eliminates space wasted on redundant single-child chains. Applied to FST, CoCo-trie, and Marisa across six datasets, the layout change improves query performance by 1.12x to 1.58x while the path compression yields a 1.3x smaller memory footprint on average.

How memory safety CVEs differ between Rust and C/C++ (1 minute read)

The author analyzes memory safety CVE data and argues that Rust's guarantees shift the nature of vulnerabilities rather than just reducing their count, since unsafe blocks and logic errors still produce memory bugs that look different from C/C++ classics. The post contends that Rust's compiler eliminates most use-after-free and buffer-overflow patterns up front, concentrating remaining issues in a smaller, auditable surface where reviewers can focus their attention.

The author maps each fallacy to modern failure modes in microservices and cloud systems, noting that newer abstractions hide the network without removing its costs.

AI demands more engineering discipline. Not less (xpost) (1 minute read)

Don't trust large context windows (1 minute read)

The author argues that large context windows degrade model performance well before hitting their advertised token limits, with recall and reasoning falling off as the prompt fills.
