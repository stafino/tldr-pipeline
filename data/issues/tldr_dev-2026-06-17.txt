All Smoke, No Alarm: Oracle Signals in Agent-Authored Test Code (1 minute read)

A study of 86,156 test-file patches from 33,596 agent-authored PRs across five coding agents found that 80.2% contain weak or no explicit oracle signals, meaning the tests run code without actually verifying behavior. After adjusting for agent, PR size, and language, strong-oracle PRs were 28% more likely to merge, suggesting test-file counts overstate verification strength and quality gates should check for assertions, not just test presence.

Cordon: Semantic Transactions for Tool-Using LLM Agents (1 minute read)

Cordon is a transactional runtime that stages a tool-using agent's irreversible effects in shadow state and an effect outbox, validating the full multi-step execution flow before committing or releasing any external action. The system replaces per-call guardrails with a task-scoped boundary supporting commit, rollback, and audit, catching cross-step violations that existing defenses miss while adding modest approval and latency overhead.

Structural Role Injection in Handlebars-Templated LLM Prompts: Triple-Brace Interpolation, Delimiter Family, and the Limits of HTML Auto-Escaping (1 minute read)

Handlebars' double-brace escaping, the documented safe default in Microsoft Semantic Kernel, only neutralizes role delimiters built from angle brackets, leaving Llama-2 [INST], Human:/Assistant:, and Markdown ### injections fully intact. Across 5,760 trials, GPT-3.5 Turbo followed task-hijack instructions in 91% of escaped prompts while Claude Haiku 4.5 resisted almost entirely, showing escaping cannot substitute for structural separation of instruction and data.

The Value of Adaptivity in LSM Bloom-Filter Tuning: A Log-Law and a Two-Clock Frontier (1 minute read)

This paper analyzes when adaptive Bloom-filter tuning in LSM trees actually beats the static Monkey optimum, deriving a log-law that makes optimal bits-per-key affine in the logarithm of access frequency and showing that coarse hotness estimates lose little because the workload enters only logarithmically. On Twitter production traces and a real RocksDB engine, reallocating filters only at compaction captures 96-99% of continuous tracking's benefit, since compaction already rebuilds filters for free on its own clock.

When the Next Step Is Not One Step: Distribution-Aware Execution Modeling for Concurrent Go Programs (1 minute read)

This paper treats scheduler nondeterminism in concurrent Go programs as a training signal, running each program many times to build an empirical distribution of next events and fine-tuning a 7B model to match it with a KL objective. On 798 held-out predictions from real CockroachDB, Kubernetes, gRPC, and etcd bugs, it hits 36.2% accuracy and cuts Expected Calibration Error from 0.205 to 0.169 versus cross-entropy.

PracRepair: LLM-Empowered Automated Program Repair Inspired by Human-Like Debugging Practices (1 minute read)

PracRepair is an LLM-based automated program repair framework that builds on-demand static-dynamic context from failing executions, runs question-driven failure diagnosis to form explicit repair hypotheses, and refines patches using validation diagnostics and trace-level behavioral changes. On Defects4J V1.2/V2.0 it fixes 162/171 bugs under GPT-4o, beating state-of-the-art baselines and generalizing to real-world bugs across multiple models.

Position: Coding Benchmarks Are Misaligned with Agentic Software Engineering (1 minute read)

The authors argue that coding benchmarks built in the pre-agent era are misaligned with how agents actually work, since they collapse model, harness, environment, and feedback into one end-to-end score graded against a single reference solution. They identify three failures: scores conflate the model with the surrounding harness, valid alternative solutions get penalized, and the lack of component-level signal makes the system hard to iterate on.
