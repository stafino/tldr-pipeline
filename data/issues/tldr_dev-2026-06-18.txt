Why `vitest --changed` misses some tests (and how runtime coverage fixes it) (3 minute read)

`vitest --changed` and `jest --findRelatedTests` select tests by walking the static import graph, so they miss any test whose connection to the changed file exists only at runtime, such as a plugin loaded through a registry of data-driven paths. The post shows this failure case and proposes selecting tests from runtime coverage instead, identifying which tests actually executed a file rather than which ones statically import it.

Why AI coding debt is different (3 minute read)

The author argues that AI-generated code creates "cognitive debt" rather than just bad code: it ships clean and passes CI, but leaves no one holding the mental model of how or why the system was built. He points to Google's disclosure that 75% of new code is now AI-generated and predicts the industry will be forced toward the rigorous pre-ship verification culture hardware engineers have practiced for decades.

Homebrew tightens tap security, begins work on its interface (1 minute read)

Homebrew 6.0.0 now requires any third-party tap, along with its formulae and casks, to be explicitly trusted before its Ruby code is evaluated or run, closing a long-standing gap where tap code executed unsandboxed by default. Official taps stay trusted out of the box, and the brew tap command gains new options for managing which sources are allowed to run.
