Bun has an open PR adding shared-memory threads to JavaScriptCore (1 minute read)

Jarred Sumner has an open WebKit PR adding shared-memory threads to JavaScriptCore, where `new Thread(fn, ...args)` runs a real closure on another core in the same heap, with no structured clone, message passing, or stringified-source-in-a-blob worker dance. Alice is impatient (1 minute read)

This post explains the inspection paradox: users experience a t-weighted version of your latency distribution, so the mean wait they feel is E[X] + Var(X)/E[X], not your reported mean. An interactive simulation drives the point home, with a 30-minute median TTR and 10-hour p99 yielding a one-hour MTTR on your dashboards but a six-hour mean recovery for customers.

There are no instances in ATProto (1 minute read)

Dan Abramov argues that "instances" are a category error in atproto, which separates hosting from aggregation the way blogs separated publishing from readers like Google Reader, rather than enclosing users inside self-hosted communities the way Mastodon does. Your data lives in a personal repository you control, and apps are mere projections over the network, so there is no server you "join" and no community walls to pick between.

Linux eliminates the strncpy API after six years of work, 360 patches (1 minute read)

Linux 7.2 removes the strncpy() API from the kernel, the end of a six-year effort spanning roughly 362 commits to migrate every caller off an interface long deprecated for its counter-intuitive NUL-termination semantics and redundant destination zero-filling. Kernel code now uses targeted replacements instead: strscpy() and strscpy_pad() for NUL-terminated destinations, strtomem_pad() for fixed-width fields, and memcpy_and_pad() or memcpy() for bounded copies.

Epoll vs. io_uring in Linux (1 minute read)

This post compares Linux's two asynchronous I/O interfaces, explaining why epoll's design forces two syscalls per I/O event (it signals readiness, leaving you to call read/write yourself) and how that context-switch overhead scales badly under high connection counts. io_uring instead reports completion rather than readiness, using shared submission and completion ring buffers so a single io_uring_enter call can batch many operations, which led the author to rewrite their TinyGate reverse proxy around it.

Surprising economics of load-balanced systems (1 minute read)

Modeling a load-balanced fleet as an M/M/c queue, the author shows that mean request latency asymptotically approaches the one-second service time as server count grows, even with per-server load held constant at 80%. The counterintuitive result is that bigger fleets queue a smaller fraction of requests, so scaling out improves observed latency for free rather than just adding throughput, with the same trend holding at the tail percentiles.

I hate compilers (1 minute read)

The author needed reproducible builds of `wasm2js` for Anubis, but identical source produced byte-different binaries across machines, so they vendored a deterministic copy compiled to WebAssembly via wasi-sdk. Getting there meant passing `--no-wasm-opt` to stop Clang silently invoking a system tool during linking, and disabling address-space randomization with `setarch` to kill ~29-byte diffs from pointer-dependent catch-reference ordering, with separate x86_64 and arm64 checksums as the remaining trade-off.
