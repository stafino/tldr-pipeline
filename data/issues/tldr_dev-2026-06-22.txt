Memory Safe Inline Assembly (1 minute read)

Fil-C, the memory-safe C dialect, now supports GCC/clang inline assembly syntax in a pre-release build, preserving the exact assembly template you write while guaranteeing memory safety. The post details how mistakes that would silently miscompile in standard C, like omitting a `+` constraint or a `cc` clobber, instead degrade to a panic or an illegal instruction trap at worst.

Deno Desktop (1 minute read)

Deno v2.9.0 introduces `deno desktop`, a command that turns any Deno project from a single TypeScript file to a Next.js, SvelteKit, or Vite SSR app into a self-contained, cross-compilable binary bundling the runtime and a web rendering engine. It defaults to the OS webview for small binaries with optional bundled Chromium, swaps socket IPC for in-process bindings, and ships built-in bsdiff auto-update, though it currently runs only in the canary build.

Show HN: Recall – Local project memory for Claude Code (1 minute read)

Recall is a local memory plugin for Claude Code that logs each session to an append-only `history.md` and condenses it into a compact `context.md` summary using a classical Python summarizer rather than an LLM call. It solves the cold-start problem without spending model tokens or sending transcripts to any external API, so resuming a project loads from a ~1-2K token digest instead of re-explaining everything each session.

Hundreds of AI-powered iOS apps found exposing credentials (1 minute read)

Researchers at Wake Forest University analyzed 444 iOS apps with LLM features and found 282, or 64%, exposed exploitable credentials, including 54 plaintext API keys and 136 leaked authentication tokens. The most popular affected app had over 2.3 million ratings, and 55% of leaks routed traffic through custom developer backends, meaning provider-side fixes alone cannot close the gap.
