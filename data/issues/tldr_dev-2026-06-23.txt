The new HTTP QUERY method explained (1 minute read)

RFC 10008 introduces the HTTP QUERY method, a safe, idempotent verb that carries filter criteria in a request body instead of cramming complex relational queries, deep nesting, and arrays into GET query strings that hit URL length limits and get logged by middleware. The post explains why a GET with a body fails in practice, since proxies and servers inconsistently reject, drop, or honor the body.

Habituation at the Gate: Rising Approval and Declining Scrutiny in Human Review of AI Agent Code (1 minute read)

A longitudinal study of 400 repeat reviewers and 11,429 reviews over seven months found that human approval of AI-agent pull requests rose from 30.1% to 36.8% as reviewers gained experience, an effect that persists after controlling for calendar time and PR size. The authors read it as reflexive habituation rather than calibrated trust, since review latency grew 3.5x while inline comment volume fell 22%, meaning reviewers sat longer in queue but inspected code less.

Beyond Simpson's Paradox: A Cascade of Confounders in AI Agent Pull-Request Co-Authorship (1 minute read)

The authors analyze 33,596 pull requests from five AI coding agents and show that the apparent co-authorship merge gap (53.8% vs 79.8% pooled) is a Simpson's Paradox driven by Codex dominating 64.9% of the data. After controlling for repository selection and PR structure like commit count, no agent retains a clear co-authorship effect, exposing the pooled statistic as a selection and structure artifact rather than causal evidence.
