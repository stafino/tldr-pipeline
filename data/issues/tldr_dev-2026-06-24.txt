He resists adopting it for code he cares about, citing models that write overly defensive, locally-reasoned code and his own unwillingness to ship systems he cannot fully explain.

Vulnerability reports are not special anymore (1 minute read)

The author, former lead of the Go Security team, argues that vulnerability reports no longer warrant the special responsiveness and attribution maintainers have traditionally owed researchers, because LLMs have made the underlying insight and confidentiality cheap. With anyone able to generate plausible findings, the bottleneck has shifted from discovery to triage, leaving a security@ inbox with roughly the same signal-to-noise ratio as raw LLM output.

Prompt Injection as Role Confusion (1 minute read)

The author argues that prompt injection is fundamentally a role-confusion problem: because an LLM receives system prompts, user messages, tool outputs, and its own reasoning as one undifferentiated token stream, it cannot reliably tell its own thoughts from injected text. Framing roles as the core abstraction lets the post derive new attacks, explain existing mech-interp results, and predict when an injection will succeed.
