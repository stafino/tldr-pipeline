CLIR: Liveness-Driven and Structure-Aware Fuzzing for the Cranelift Compiler (1 minute read)

In 72 hours it found 24 bugs across all target architectures, 21 confirmed and 9 fixed, detecting 8x more unique bugs than cranelift-fuzzgen.

Augmentation with Dilution: A Large-Scale Empirical Study of Human Contributor Ecosystems After AI Coding Agent Adoption (1 minute read)

A study of 11,097 GitHub repositories from 2023 to 2026 finds that AI coding agent adoption does not reduce the absolute number of human contributors, but cuts their relative density and drops the newcomer participation share by 3.7 percentage points. The authors call this "augmentation with dilution," noting that review depth rises 5.3% as the burden shifts from writing code to reviewing AI-generated pull requests.

New Enterprise-Ready MCP Specification Brings New Security Challenges (1 minute read)

MCP's 2026-07-28 revision makes the protocol stateless at the layer, closing off session hijacking and unsolicited server prompts while tightening authentication, with legacy versions deprecated over a 12-month window. Akamai's analysis warns the rewrite shifts risk onto implementations: predictable tracking IDs and state objects open the door to cross-tenant workflow hijacking, and new MCP HTTP headers introduce desync attacks and secret leakage if developers map API keys or PII into them.

A Deterministic Control Plane for LLM Coding Agents (1 minute read)

A study of 6,145 agent config files across 10,008 GitHub repos finds that LLM coding rules and agent definitions propagate as undeclared shared components, with 10.1% being exact SHA-256 duplicates across independent repos and under 1% declaring any permission boundary. The authors argue this configuration layer should be governed by a deterministic, tool-agnostic control plane with content-addressed lockfiles, tiered permissions, and audit logs, rather than delegated to further LLM orchestration.

Same Scrutiny, More Time: Eye Tracking Insights into Reviewing LLM-Labelled Code (1 minute read)

A Wizard-of-Oz eye-tracking study found that labeling code as LLM-generated did not change how thoroughly engineers reviewed it, but it did make them fixate longer and shift strategy toward checking logical correctness and consulting the original prompt. The authors flag a gap between reviewers' stated intentions and actual behavior, arguing companies should revisit AI policies and treat the prompt as a reviewable software artifact.
