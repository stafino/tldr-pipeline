DSpark: Speculative decoding accelerates LLM inference [pdf] (1 minute read)

DSpark applies speculative decoding to accelerate LLM inference by drafting multiple candidate tokens with a small model and verifying them in parallel against the full model. The paper reports throughput gains while preserving the target model's output distribution, with the draft-and-verify loop tuned to keep acceptance rates high without retraining the base model.

Qwen-AgentWorld (5 minute read)

The token is valid — but your headless Claude Code agent just 401'd forever (3 minute read) [flag]

A debugging post traces why long-running headless Claude Code agents return HTTP 401 "Invalid authentication credentials" even though their static OAuth token still returns HTTP 200 on raw /v1/messages calls fired at the same instant. The author shows the rejection is bound to the instance's server-side session identity rather than the token, and that once an instance 401s it hard-latches and never recovers until the process restarts, making any token-validity probe structurally blind to the failure.

British Columbia, Time Zones, and Postgres (5 minute read)

British Columbia switched to permanent UTC-7 on March 8, 2026, which breaks any Postgres `timestamptz` appointment stored before the rule change and queried after. Because these columns store UTC and reconvert using current tzdata, future November-to-March bookings can come back an hour off, and the post shows a SQL check to confirm whether your tzdata package has picked up the new offset.

New agentic memory framework uses 118K tokens per query. LangMem burns through 3.26M. (2 minute read)

Researchers at the National University of Singapore built MRAgent, an agentic memory framework that replaces the static "retrieve-then-reason" pipeline with active reconstruction, starting from small prompt triggers like a name or place and following metadata stepping stones to assemble evidence one piece at a time. Researchers at Mozilla's 0DIN demonstrated an attack where a clean GitHub repo with no malicious code tricks Claude Code into opening a reverse shell while it tries to recover from a planted setup error. A C++ implementation of a fast hash map and hash set using hopscotch hashing (1 minute read)

hopscotch-map is a header-only C++ hash map and hash set that uses open-addressing with hopscotch hashing to keep collision chains cache-local, outperforming std::unordered_map in most cases while using less memory than google::dense_hash_map. It ships power-of-two and prime growth variants, heterogeneous lookups, support for move-only and non-default-constructible keys, and a bounded-worst-case version for cases where hash DoS is a concern.
