The four horsemen behind thousands of Postgres outages (5 minute read)

The author identifies four recurring Postgres failure modes behind production outages, starting with vacuum and transaction ID wraparound. Because Postgres marks deleted rows rather than removing them, the vacuum competes with query I/O, and if it falls behind reclaiming the 32-bit transaction IDs, Postgres shuts down the database to avoid data corruption. The post frames these as targets for pgrust, the author's Postgres reimplementation.

Quadrupling code performance with a "useless" if (1 minute read)

The author traces a pointer-chasing loop in a domain-specific compressor, `j = next_j[i][j]`, whose single dependent load serializes every iteration and starves the CPU of instruction-level parallelism. Adding a seemingly useless `if` breaks the loop-carried dependency chain so the processor can speculate ahead across iterations, quadrupling throughput despite executing more instructions.

HTTP gets a QUERY method so complex searches can stop pretending to be POST (1 minute read)

The git history command (1 minute read)

Git 2.54 and 2.55 introduced an experimental `git history` command with three subcommands (fixup, reword, split) that rewrite old commits and autorebase every local branch descended from them, going further than `rebase --update-refs`. All operations are atomic and refuse anything that could produce a conflict, delivering much of jj's history-editing ergonomics without leaving the core git distribution or handling merge commits.
