The Tower Keeps Rising (1 minute read)

Using Bruegel's Tower of Babel as a frame, the author argues that AI agents make individual developers dramatically faster at changing a codebase, but large projects have never been bottlenecked by code output. They are limited by shared understanding of a system's concepts, boundaries, and invariants, and much of that understanding was previously maintained by the friction of code review and cross-team conversation that agents now let developers bypass.

Trust but Verify? Uncovering the Security Debt of Autonomous Coding Agents (1 minute read)

An empirical study of 4,022 agent-generated pull requests found that 38.9% contain at least one security smell, with supply chain integrity issues making up 82.3% and hard-coded credentials accounting for 99.6% of critical-severity findings. The authors note that human collaborators introduced 67.6% of genuine leaked secrets and that existing automated and manual reviews missed 81.1% of them before merge, pointing to eroding developer vigilance in AI-assisted workflows.

Software Supply Chains are Dead: Use-Case-Oriented Regeneration (1 minute read)

The authors argue that generative AI and supply chain attacks have inverted the build-vs-reuse calculus, proposing "use-case-oriented regeneration" that synthesizes only the slice of a dependency a repository actually exercises. Across 180 repository-dependency pairs, an agentic workflow preserved 99.8% of observed behavior while cutting the exported API surface by 93%, trading external trust for locally verifiable, repo-specific code.

With AI Agents, Attention is the New Bottleneck (5 minute read)

AI agents removed execution as the constraint on shipping software, but the author argues that just exposed the next bottleneck: the human work of deciding what to build, judging output, and catching which parallel task is about to fail. The fix is to manage cognitive load deliberately by automating low-stakes decisions through lint rules, defaults, and agent instructions, and by estimating attention cost per task the way teams already estimate points.

Part 1: Why I Rarely Use useEffect Anymore (and what I use instead) (2 minute read)

The author argues that most useEffect calls are unnecessary, since the hook is meant only for synchronizing with external systems like network requests, timers, and subscriptions. Deriving values, syncing props into state, and filtering lists should happen during render or in a useMemo, avoiding the double-render and dual-source-of-truth bugs that effects introduce.

Dependabot version updates introduce default package cooldown (1 minute read)

Dependabot version updates now wait until a new release has been on its registry for three days before opening an update PR, a default that applies across all supported ecosystems on github.com with no configuration required. The cooldown gives supply-chain signals time to surface before a bad release merges, and it skips security updates entirely so critical fixes still open immediately. Set a custom window or opt out via the cooldown option in dependabot.yml.
