Swarming Claude Code and Codex in Parallel: Running Multiple Agents at Once with tmux and Git Worktrees (2 minute read)

The author runs multiple Claude Code and Codex agents in parallel using a three-layer protocol, where a plan.json declares workers that an orchestrator script expands into isolated git worktrees launched inside a tmux session. Separating each agent onto its own branch drops merge-conflict risk to zero and removes the serial bottleneck where one job's commits change the preconditions for the next, at the cost of managing seed-path copying and artifact consolidation across sessions.

Guide to AI Tokenomics: Eleven Principles for Token Efficient Software Engineering (3 minute read)

Google Cloud engineers lay out eleven principles for token-efficient AI coding, arguing that context bloat is the main driver of latency, hallucination, and cost as agents write more of the code. Their tactics include starting with a balanced default model, packaging reusable skills in SKILL.md files, delegating output-heavy work to sub-agents, and checkpointing often so sessions can restart from a clean, low-token state.

Giving an AI assistant read-only access to Microsoft Loop — without breaking permissions (3 minute read)

loop-reader-mcp is an open-source (MIT) MCP server that gives an AI assistant read-only access to Microsoft Loop pages, which have no content API and store data in SharePoint Embedded containers that only accept app-only tokens. It avoids permission-flattening by splitting discovery from retrieval, running security-trimmed Graph Search under the user's delegated token via OAuth On-Behalf-Of and using the app identity only to download content the user can already see.
