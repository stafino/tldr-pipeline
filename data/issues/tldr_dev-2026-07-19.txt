Regressive JPEGs (1 minute read)

The author abused JPEG's progressive mode, where a file is split into scans that each fill in a spectral range, to concatenate multiple same-resolution images into one file that animates as it downloads over a slow connection. The trick caps out fast because most decoders stop after roughly nine scans as a zip-bomb guard, and baseline single-scan frames fail since progressive scans cannot mix DC and AC coefficients.

Static search trees: 40x faster than binary search (2024) (1 minute read)

The author implements a static B+ tree over sorted 32-bit integers and optimizes it to roughly 40x the throughput of binary search, reporting results in ns per query. The post works through manual SIMD node search, batching and prefetching to hide memory latency, pointer arithmetic that removes instructions from the inner loop, and prefix partitioning to cut cache misses on the upper levels, with all benchmark code in the repo.

Why your Cursor rules never fire: globs, alwaysApply, and description explained (3 minute read)

The author explains why `.cursor/rules` files silently never attach: each of Cursor's four rule types is gated by a different frontmatter field, and a rule with no `alwaysApply`, `globs`, or `description` quietly defaults to Manual. The three failure modes are writing `globs` as a YAML array or quoted string instead of a bare comma-separated one, using `.md` instead of `.mdc`, and omitting the description that Agent-Requested rules match against.

Topcoat: The full full-stack framework for Rust (1 minute read)

Topcoat is a server-rendered fullstack Rust framework from the Tokio team that lets async components query the database directly, removing the separate API layer. Its `$(...)` expressions are type-checked Rust that compile to both server-side render code and JavaScript, so client interactivity runs without a wasm bundle or client build step. The project is early-stage and expects breaking changes.

The author, who maintains the Handy desktop app, built it as a near drop-in whisper.cpp replacement after hitting the limits of ONNX's CPU-only performance, and every model ships numerically validated and WER tested against its reference implementation.
