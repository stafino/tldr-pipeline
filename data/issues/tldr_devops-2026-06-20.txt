Build your own vulnerability harness (1 minute read)

Cloudflare's Project Glasswing team details how to build a model-agnostic vulnerability-scanning harness, arguing that generic coding agents fail because they hold one hypothesis at a time and lose state during context compaction. The architecture treats models as interchangeable, using one for discovery and a different one for validation so findings are cross-checked by distinct logic, then traces vulnerabilities across cross-repo dependencies down to a triaged queue.

Surprising economics of load-balanced systems (1 minute read)

Marc Brooker works through an M/M/c queuing model where c single-request servers sit behind a load balancer, with offered load scaled linearly to hold per-server utilization constant. The counterintuitive result, derived from Erlang's C formula, is that mean client latency drops toward the one-second service floor as c grows, because larger pools queue a shrinking fraction of requests. The practical takeaway for capacity planning: bigger shared pools beat many small ones at the same utilization.

Caching for Dollars, Not Hits: An Exact Offline Reference for Cloud-Egress Caching and the Crossover That Decides When It Pays (1 minute read)

This paper argues that classic cache eviction optimizes the wrong metric: with cloud object storage billed per GET and per byte of egress, miss rate is irrelevant and dollar cost is what matters. The authors build an exact offline dollar-optimal reference and show cost-aware GreedyDual cuts LRU's dollar-regret to roughly a tenth. A closed-form crossover, s* = GET_fee/egress_rate (about 4KB on S3, 330B on GCS), predicts which deployments need dollar-aware caching at all.

Scaling Ray Serve LLM on GKE: Performance without losing the developer experience (2 minute read)

Google and Anyscale rebuilt Ray Serve LLM on GKE for up to 5x higher throughput and 8x lower latency, while keeping its Python-native serving APIs intact. Three changes drive it: a built-in HAProxy layer that absorbs request routing so the Python runtime stops saturating under load, direct token streaming that bypasses the ingress router on the return path, and a v2 vLLM executor that moves Ray out of the data plane for async scheduling. Benchmarks ran Gemma 4 E2B on A4 VMs with NVIDIA HGX B200.

2,000 retired Google Pixel phones get a second life as a private cloud (1 minute read)

A team rebuilt 2,000 retired Google Pixel phones into a working private cloud, racking the handsets as low-power ARM nodes rather than e-wasting them. The writeup details the practical hurdles: building custom enclosures and cooling for consumer hardware, flashing a maintainable OS image across the fleet, and orchestrating nodes that ship with batteries and no server-grade remote management. The payoff is cheap, energy-efficient compute reclaimed from hardware most orgs would have recycled.

Novo Nordisk Breach Exposes Software Development Pipeline Risk (1 minute read)

The author argues that the Novo Nordisk breach, traced to a leaked GitHub token, proves most teams scope secrets management wrong by treating it as a tooling problem instead of an identity one. The takeaway for platform teams: credentials in the development pipeline are machine identities that need rotation, scoped permissions, and revocation, not just a vault to hide them in. Static long-lived tokens remain the weakest link.
