New DirtyClone Linux Kernel Flaw Lets Local Users Gain Root via Cloned Packets (1 minute read)

JFrog Security Research published the first working exploit for DirtyClone (CVE-2026-43503, CVSS 8.8), a Linux kernel flaw where two packet-cloning helpers drop the flag marking memory as file-backed, letting a local user overwrite a binary like /usr/bin/su in the page cache and gain root. The patch landed in mainline May 21, but multi-tenant servers, CI runners, and Kubernetes hosts that allow unprivileged user namespaces remain exposed until they update.

New Linux pedit COW Exploit Enables Root Access by Poisoning Cached Binaries (1 minute read)

CVE-2026-46331, a publicly exploited out-of-bounds write in the Linux kernel's act_pedit traffic-control action, lets an unprivileged user poison the cached copy of a setuid binary like /bin/su in memory and open a root shell while file-integrity checks stay clean. The attack needs act_pedit loadable and unprivileged user namespaces open, conditions present by default on RHEL 10 and Debian 13, which Red Hat rates important.

Securing CI/CD for an open source project, part 3: Credentials, verification, and what’s next (1 minute read)

The final post in Cilium's CI/CD hardening series covers credential isolation and release provenance. CI credentials can push only to the development registry (quay.io/cilium/*-ci), while production push secrets sit behind a GitHub protected environment that requires explicit maintainer approval, so a compromised CI workflow can ship a malicious -ci image but never overwrite a real release tag. Every released image is signed with Sigstore Cosign using keyless OIDC and ships with an SBOM, leaving no long-lived signing keys to steal.
