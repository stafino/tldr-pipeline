Kepler, re-architected: Improved power accuracy and a community call to action! (1 minute read)

Kepler, the CNCF sandbox project that attributes Kubernetes power consumption per Pod and exports it as Prometheus metrics, has been rewritten to drop its eBPF dependency, which required CAP_BPF and CAP_SYSADMIN privileges and under-reported energy from short-lived processes. The new architecture aims to improve power-attribution accuracy, ease deployment on VMs, and lower the barrier for community contributions ahead of training power-estimation models.

Meta fights soaring hardware costs by reusing old DDR4 server memory in new DDR5-only servers — custom CXL 2.0 chip marries legacy DDR4-2400 with cutting-edge DDR5-6400 (1 minute read)

Meta built Vistara, an in-house CXL 2.0 ASIC that attaches DDR4 RDIMMs recovered from decommissioned servers to new DDR5-only AMD Turin machines, dodging record DDR5 prices by reclaiming memory that would otherwise be scrapped. Each MemServer pairs 768GB of local DDR5-6400 with 256GB of CXL-attached DDR4-2400 exposed as a separate NUMA node, letting Linux migrate cold pages to the slower 76GB/s tier while hot data stays in 614GB/s DDR5.

Dragonfly v2.5.0 is released (1 minute read)

Dragonfly v2.5.0 adds direct repository downloads from Hugging Face and ModelScope, routing Git LFS model weights through its P2P network while a new Kubernetes mutating webhook injects P2P capability into Pods without rebuilding images. The release also ships comprehensive rate limiting across the control plane and client, a Manager-console blocklist for emergency download control, and a dfctl CLI for managing local task storage.
