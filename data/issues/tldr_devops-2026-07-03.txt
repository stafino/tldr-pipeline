Meta reuses old RAM in new servers with custom bridge chip (1 minute read)

Meta developed a custom CXL bridge chip called Vistara to decouple decommissioned DIMMs from their original memory channels, letting it reuse older RAM in new servers without the performance hit of plugging legacy modules directly into memory slots. The move targets the roughly 40% of Meta's millions of servers bottlenecked on memory, and lands as RAM prices are forecast to double by the end of 2026 amid a shortage expected to run through 2027.

PostgreSQL and the OOM killer: Why we use strict memory overcommit (1 minute read)

Ubicloud's team explains why they run PostgreSQL under strict memory overcommit (vm.overcommit_memory=2), which forces malloc failures instead of letting the Linux OOM killer terminate a backend. Killing a single backend risks corrupting shared memory segments, so the postmaster responds by dropping every connection and forcing crash recovery. The post also covers a three-character kernel bug that made them temporarily disable the setting, plus their heuristic for tuning the overcommit ratio.

How data sovereignty is changing cloud native infrastructure design (1 minute read)

The CNCF blog argues that data sovereignty has shifted from a geography problem to a jurisdictional one, since laws like the U.S. CLOUD Act tie data access to corporate control rather than server location. Choosing a Frankfurt region doesn't shield a hyperscaler's data from its parent company's home laws. The author frames this as a resilience concern, not just privacy, and notes regulated European enterprises are increasingly assembling sovereign platforms from open-source Kubernetes components instead of buying sovereignty as a hyperscaler add-on.
