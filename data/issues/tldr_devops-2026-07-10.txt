Announcing etcd v3.7.0 (2 minute read)

SIG etcd released etcd v3.7.0, headlined by RangeStream, which streams large range results in chunks instead of buffering the full set in memory on both server and client. The release also boots entirely from v3store, completes a protobuf overhaul, and cuts etcd CPU usage on the Kubernetes control plane, with RangeStream reaching k8s v1.37 behind the EtcdRangeStream feature gate.

Google pays $250K for Linux vulnerability allowing guest VM escapes (1 minute read)

Google paid a $250K bounty for CVE-2026-53359, a use-after-free in KVM's shadow MMU emulation that lets a guest VM escape isolation and gain root on the host across both AMD and Intel processors. The 16-year-old flaw lets a single rented cloud instance panic the host kernel to take down every co-tenant VM or run arbitrary code on the hypervisor, and a proof-of-concept crash exploit is already public.

15-Year-Old GhostLock Flaw Enables Root and Container Escape on Most Linux Distros (1 minute read)

Nebula Security disclosed GhostLock (CVE-2026-43499), a 15-year-old use-after-free in the Linux kernel that lets any logged-in user gain root and escape containers with 97% reliability, using only ordinary threading calls no elevated permissions required. The flaw ships by default in nearly every distribution since 2011 and scores 7.8, but with working exploit code now public, teams should install their distribution's latest kernel rather than the first patched build.

Januscape: 16-Year-Old Linux KVM Bug Enables Cloud VM Escape Attacks (5 minute read)

Security researcher Hyunwoo Kim disclosed Januscape (CVE-2026-53359), a 16-year-old use-after-free bug in KVM's x86 shadow MMU that lets guest-side actions alone corrupt host kernel memory and break guest-host isolation across both Intel and AMD. The public proof-of-concept reliably panics the host, taking down every co-tenant VM, and it directly threatens multi-tenant public clouds like GCP and AWS that expose nested virtualization.

Introducing Meerkat: an experiment in global consensus (1 minute read)

Cloudflare's Research team introduced Meerkat, an experimental global consensus service built on the leaderless QuePaxa algorithm from EPFL, in what it calls the first industrial deployment of QuePaxa at scale. Unlike Raft, every replica can write at any time and progress never stalls on timeouts, addressing the unavailable-leader incidents Cloudflare has hit across its 330-plus data centers, though the service stays internal-only for now.

Network boundary for AI agents using NGINX and OpenTelemetry (1 minute read)

The author builds a network boundary for AI agents from two components already common in cloud native stacks: NGINX as an enforced traffic control plane and OpenTelemetry as the audit plane. NGINX proxies inbound and outbound traffic while iptables drops all other egress, making the boundary an architectural property rather than a policy the agent must honor. The native OTEL module emits a span per request, so every external call an agent makes is correlated to a user interaction and persisted to Jaeger, Grafana, or a SIEM.

TLS certificates for internal services done right (1 minute read)

The author argues that public-CA TLS with split-horizon DNS beats self-signed certificates for internal services, since managing one WAF is easier than distributing a private CA to every client. The walkthrough resolves an app like grafana.tuxnet.dev to a public IP for outside resolvers and an internal IP over the VPN, letting Let's Encrypt issue a valid cert. The stack is NetBird for split-horizon DNS, acme.sh for issuance, and nginx as the WAF rejecting non-VPN traffic.

You shouldn't trust Trusted Publishing (5 minute read)

The author argues that debating whether you can trust PyPI's Trusted Publishing is a category error, since the scheme is machine-to-machine authentication built on OIDC federation, not something a human vouches for. The real win is operational: it swaps long-lived, over-scoped API tokens for short-lived, minimally scoped credentials issued to a CI/CD identity, shrinking the pool of standing secrets a maintainer can leak. The post notes the trade-offs, including a complex data model and per-provider OIDC handling.

Navigating the ingress-NGINX retirement (1 minute read)

The CNCF published a guide to the community ingress-nginx controller reaching end of life after March 2026, clarifying that the Kubernetes Ingress API itself stays supported. Teams choose between a lift-and-shift to an Envoy-based controller like Contour, which keeps existing Ingress YAML but breaks every nginx.ingress.kubernetes.io/* annotation, or migrating to the Gateway API. The post recommends auditing annotation debt, automating translation with ingress2gateway, and rolling out incrementally on non-critical workloads first.

Some Notes on Lambda MicroVMs (5 minute read)

AWS launched Lambda MicroVMs, and this post documents the operational quirks the author hit in the first day of use. MicroVMs run any code in a fresh VM within seconds for up to 8 hours, support full OS capabilities including nested Docker, and offer a first-class shell via the CreateMicrovmShellAuthToken API. The gotchas are networking: outbound UDP is blocked by default so container DNS needs `--dns 169.254.169.253`, and Lambda network connectors create ENIs that stay hidden unless you pass `IncludeManagedResources=true`.

Why we cannot wait for better post-quantum signature algorithms (1 minute read)

Cloudflare argues the industry cannot wait for better post-quantum signature schemes and must migrate now with ML-DSA, the NIST-standardized scheme available today. The trade-off is real: ML-DSA is far larger on the wire than RSA or ECC and breaks many optimizations teams relied on, but successors like FN-DSA and the nine schemes NIST advanced to its signatures on-ramp won't be ready in time. Cloudflare is targeting 2029 to be fully post-quantum secure, with most of its traffic already on ML-KEM encryption.

Woodruff: You shouldn't trust trusted publishing (5 minute read)

William Woodruff argues that Trusted Publishing is authentication, not a quality signal, and should not be read as any indication that a package is safe or good. It establishes a trust relationship between a CI/CD identity and a project on a registry like PyPI, confirming only that an upload was authenticated. PyPI deliberately refuses to render it as a green checkmark to stop teams from misusing it as a proxy for package trust.

How to teach SRE AI agents to fail safely and earn your team’s trust (2 minute read)

The author argues that trust in agentic SRE is an engineering outcome, not a marketing claim, earned during real failures rather than clean demos. The proposed architecture separates reasoning from actuation: the agent investigates, forms ranked hypotheses, and stages remediation plans, but every change routes through a deterministic safety layer that validates permissions, risk, production state, and blast radius first. Autonomy expands only after sustained proof of safe behavior, mirroring Google's progressive-authorization model.

Practical challenges in managing Kubernetes at enterprise scale (3 minute read)

The author argues that Kubernetes at enterprise scale is a governance problem, not an engineering one, since the platform moves operational headaches around rather than eliminating them. The hard part isn't manifests or Helm charts but standardization across hundreds of engineers who each drift into their own templates, naming conventions, and resource settings. The fix is a paved road: approved templates, default observability and security controls, and self-service that lets app teams ship without becoming Kubernetes experts.

The open source library holding up your stack might have one maintainer (1 minute read)

A new paper sorts open source software into fourteen sub-genres defined by who starts and funds a project, arguing that a lone-maintainer utility and a foundation-governed platform behave differently in production despite sharing the "open source" label. The typology, drawn from a screen of nearly 4,000 papers, flags funding and governance as the predictors of whether critical dependencies like OpenSSL, curl, and log4j hold up under load.
