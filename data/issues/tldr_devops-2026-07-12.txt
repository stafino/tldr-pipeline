We scaled PgBouncer to 4x throughput (1 minute read)

ClickHouse's Managed Postgres team describes running a fleet of PgBouncer processes bound to the same port with so_reuseport, since PgBouncer is single-threaded and pins one core no matter how many the box has. On a 16-vCPU c7i.4xlarge, the fleet hit roughly 336k transactions/sec against a single process's 87k peak, about 4x. The write-up covers the operational catch: cancel requests can land on the wrong process, fixed by peering, with the connection budget split across the fleet so Postgres is never oversubscribed.

ZeroFS vs. Amazon S3 Files (1 minute read)

ZeroFS and Amazon S3 Files both expose POSIX filesystems over object storage, but split on whether the bucket stays usable as plain S3. S3 Files preserves one-file, one-object identity and exports asynchronously after 60 seconds of inactivity, while ZeroFS packs, compresses, and client-side encrypts extents into opaque segments, trading direct S3 access for cheaper small-file writes.

Mesh LLM: distributed AI computing on iroh (1 minute read)

Mesh LLM pools GPUs across the machines a team already owns and exposes them as one OpenAI-compatible API, routing each request to run locally, hand off to a peer with the model loaded, or split a too-large model across several boxes as a pipeline. Built on iroh endpoints with a pluggable plugin architecture, it ships a catalog of 40+ models spanning laptop-sized 500M weights up to 235B mixture-of-experts giants.
