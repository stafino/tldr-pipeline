A broken DNSSEC rollover took down .AL. Now 1.1.1.1 tells you when validation is bypassed (1 minute read)

A botched DNSSEC key rollover at Albania's .AL registry on July 3 broke validation across the entire TLD, leaving government, bank, and media sites unreachable on every validating resolver until Cloudflare applied a Negative Trust Anchor to suspend validation on 1.1.1.1. For the first time, 1.1.1.1 also returned a new Extended DNS Error code on affected responses, signaling clients that an answer was served unvalidated rather than restoring resolution silently.

Full Broker Takeover, No Login Required: Miggo Discovers Critical RabbitMQ Vulnerabilities Putting Application Data at Risk (5 minute read) [flag]

Miggo disclosed two access-control flaws in RabbitMQ, the most widely deployed open-source message broker, including CVE-2026-57219, an unauthenticated endpoint that leaks the broker's OAuth client secret and hands an attacker a path to full admin takeover under standard Auth0, Entra ID, and Keycloak setups. Both bugs date to release 3.13.0 and are patched in 4.3.0, 4.2.6, 4.1.11, 4.0.20, and 3.13.15, with no evidence of exploitation in the wild.

RabbitMQ Flaws Could Leak OAuth Secrets and Expose Cross-Tenant Queue Metadata (1 minute read) [flag]

Miggo's security team disclosed two access-control flaws in RabbitMQ, present since early 2024 in release lines 3.13.0 and later, one leaking the broker's OAuth client secret to an unauthenticated attacker (CVE-2026-57219, CVSS 8.7) and one letting any logged-in user read cross-tenant queue metadata (CVE-2026-57221). Patch to 4.3.0, 4.2.6, 4.1.11, 4.0.20, or 3.13.15, rotate exposed OAuth secrets, and lock port 15672 to trusted networks.

Bulkhead: Automated Semantic Detection and Remediation of Container Escape Vulnerabilities (1 minute read)

This paper introduces Bulkhead, a framework that pairs LLMs with formal verification to detect and patch container path-traversal flaws that let workloads escape filesystem isolation, a class of bug that has grown as clusters mount shared GPUs and agent workspaces into containers for AI jobs. It locates cross-boundary call chains, generates proof-of-concept exploits to confirm each vulnerability, then produces patches checked against model-checking templates to keep remediation from breaking system calls.

Is a Pod the right deployment unit for an AI agent? (1 minute read)

The kagent team argues that the Kubernetes Pod, their first answer for isolating AI agents, is the wrong lifecycle abstraction as fleets scale. A Pod cleanly delivers per-agent identity via ServiceAccounts, network policy enforcement, and observability attribution, but agents run for seconds and sit idle, making a dedicated always-on Pod per agent wasteful. The post pushes toward a lighter execution model that fits bursty, ephemeral, subagent-spawning workloads without abandoning Kubernetes-native security and scheduling.

Taming the Drift: Context-aware Repair of Dockerfile Drift during Software Evolution (1 minute read)

This paper introduces Cadre, a context-aware framework for repairing Dockerfile drift, the divergence between a Dockerfile and its evolving source that breaks CI/CD builds. Cadre builds a dependency graph mapping each instruction to its file-level and inter-instruction dependencies, then patches only the context causally relevant to a failure. On a 1,040-instance benchmark it hits a 35.22% repair rate, 1.24x the best LLM baseline, while keeping 95% of prompts under 30k tokens and avoiding the overflow failures that sink diff-only methods.

OVS and OVN Explained: The Networking Stack Behind OpenStack (5 minute read)

This tutorial explains the two-layer networking stack behind OpenStack Neutron: Open vSwitch provides the high-performance dataplane on each hypervisor, while Open Virtual Network adds the distributed control plane that programs it. It walks through the OVS-versus-OVN split, tunnel encapsulation with Geneve and VXLAN, and how to trace a packet from instance to instance. The practical payoff is debugging cross-host connectivity failures, where same-node pings succeed but traffic dies between compute nodes because of logical flow denials rather than dataplane bugs.
