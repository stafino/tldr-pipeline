A broken DNSSEC rollover took down .AL. Now 1.1.1.1 tells you when validation is bypassed (1 minute read)

Running a self-hosted LLM in Kubernetes with vLLM (1 minute read)

This CNCF tutorial documents standing up self-hosted LLM inference on Kubernetes, using vLLM for serving and LINSTOR for replicated persistent storage of model weights. Because vLLM exposes an OpenAI-compatible REST API, existing LangChain or OpenAI-SDK code repoints with a URL change, making a hybrid setup practical for cost predictability, latency control, and data-residency requirements. The walkthrough runs Llama-3.2-1B on CPU, so GPU-less labs can reproduce it.

On-prem DBaaS in 2026: Platforms, standards, and gaps (1 minute read)

This CNCF post maps the state of on-prem Database-as-a-Service in 2026, arguing that database provisioning remains fragmented even as platform engineering and Kubernetes adoption have matured. Teams still face a tradeoff between running their own operators (Patroni, MariaDB, key-value) and centralizing behind Crossplane compositions that provision cloud-managed services like RDS. The author's point is that both approaches force organizations to reinvent DBaaS, and no standard yet delivers a consistent self-service experience across on-prem and cloud environments.

Making 768 servers look like 1 (1 minute read)

PlanetScale walks through scaling a relational database from a single node to 768 servers storing a petabyte, arguing that sharding is the only durable answer once a Postgres or MySQL instance exceeds a few terabytes. The post grounds the case in the Universal Scalability Law, showing why read-replicas and bigger boxes hit CPU and IOPS ceilings that vertical scaling and replication cannot clear. It then traces the path from four shards to 768, treating the fleet as one logical database.

Node.js security starts before CI (3 minute read)

The author argues Node.js dependency security has to move upstream of CI, to the moment a developer decides to trust a package, since every npm install and transitive dependency is an unreviewed trust decision. Recent supply-chain incidents make the case: compromised Axios versions with install-time payloads in March 2026, and 84 malicious versions across 42 packages shipped through TanStack's own release pipeline in May. Post-merge scanners catch these too late.

Securing AI at Enterprise Scale: The Google Kubernetes Engine Blueprint (2 minute read)

Google shared a blueprint for securing AI workloads on GKE, organizing controls across three layers: infrastructure, model, and application. The infrastructure layer leans on Confidential GKE Nodes to extend hardware memory encryption and attestation to H100 GPUs and TPUs, plus Workload Identity Federation to fetch model weights from Cloud Storage without long-lived keys. The model layer adds supply-chain provenance so what you train matches what you serve, addressing prompt injection and weight-integrity gaps that traditional SBOMs miss.

Bring GitLab Duo Agent Platform to your terminal (3 minute read)

GitLab made Duo CLI generally available in GitLab 19.2, bringing its agentic chat into the terminal with built-in awareness of your projects, pipelines, permissions, and existing agent setup. It runs interactively or headless inside CI jobs, authenticates through `glab duo cli`, and gives platform teams the same admin controls, identity model, and MCP diagnostics they govern the rest of GitLab with.

When a version bump breaks your build, GitLab fixes it (2 minute read)

GitLab launched Dependency Scanning Auto-Remediation in beta, which opens a merge request to upgrade a vulnerable package, then uses AI to fix any build-breaking changes and iterates until the pipeline passes. Every fix runs through existing approval gates and audit trails, with each MR attributed to a dedicated service account, targeting the roughly one in eight dependency updates that introduce a breaking change.

Introducing Precursor: detecting agentic behavior with continuous client-side signals (5 minute read)

Cloudflare launched Precursor, a client-side verification system that injects JavaScript to continuously collect behavioral signals across a full session, feeding them into Enterprise Bot Management in real time to separate human traffic from automated and agentic bots. It extends detection beyond one-off Turnstile challenges at login or checkout to the entire user journey, targeting automation that can pass individual CAPTCHAs but cannot fake consistent human behavior over time.

Turn multi-step software delivery into agentic flows you can trust (3 minute read)

GitLab made Custom Flows in the Duo Agent Platform generally available in GitLab 19.2, letting teams define multi-step delivery workflows once, trigger them from native GitLab events, and run them inside CI/CD pipelines with human-in-the-loop checkpoints at sensitive steps. The 19.2 release adds a "work item status changed" trigger, bulk enablement across up to 100 projects, and flow recommendations surfaced directly in Duo Agentic Chat.

GitLab Duo Security Review spots logic flaws scanners miss (2 minute read)

GitLab launched Security Review Flow in public beta, an AI-driven flow on the Duo Agent Platform that reasons about code intent to catch authorization gaps, data-exposure bugs, and business-logic race conditions during merge-request review. It targets the flaws static scanners miss, like broken object-level authorization, because the code is valid and only violates domain context the scanner never sees, previously catchable only through expensive manual review.
