Isolation Failure From Shared Storage: Characterizing and Exploiting Page-Cache SCA Leakage Across Containers and VMs (1 minute read)

Researchers demonstrated that the host page cache leaks file-access timing across container and VM boundaries, breaking isolation on Docker, gVisor, Kata with QEMU or Cloud Hypervisor, and QEMU/KVM whenever the I/O path touches shared host-cacheable files through OverlayFS, virtio-fs, or loop devices. Direct I/O and dedicated block devices largely kill the signal, which the authors showed recovering coarse activity from a WordPress and MySQL deployment.

Why your agent needs access to your documentation (1 minute read)

After shipping an in-product agent with roughly 30 native tools, the authors found that a single `search_knowledge_base` tool was the most-used one across 1,192 conversations, nearly matching all native tools combined. Documentation search acted as the failover in 32.1% of conversations, answering setup and troubleshooting questions no purpose-built tool covered. For CNCF projects with large, fast-moving docs, the split is native tools for current state, docs for what it means.

Why goodput matters more than throughput for LLM serving (1 minute read)

The author argues that throughput is the wrong headline metric for LLM serving, since requests per second can climb while TTFT and TPOT quietly blow past targets. Goodput counts only completed requests that meet those latency SLOs, making it the number worth tuning against. The walkthrough serves Qwen2.5-7B on vLLM on a single A10G in EKS, driving load with GuideLLM and instrumenting through Prometheus, Grafana, and NVIDIA's DCGM exporter, with the Kubernetes manifests published for reuse against your own traffic.

FailureAtlas: A Taxonomy of Failure Modes in Multi-Provider LLM Serving Infrastructure (1 minute read)

This paper introduces FailureAtlas, a two-axis taxonomy of failure modes in multi-provider LLM gateways, classifying them by origin layer (network, streaming, session state, model behavior, cost governance) and by whether they fail loud or silent. The authors' main finding is that the worst failures are silent: they return HTTP 200, pass every health check, and corrupt application state. Five cataloged entries include root-cause analysis, and three ship with reproduction scripts.

Generosity Under Conditions: Hardening Google Cloud Access Management (2 minute read)

A Google Cloud developer advocate walks through using IAM conditions to constrain roles that resource-level bindings cannot, including capping `roles/iam.admin` so holders cannot grant themselves arbitrary roles or mint new ones. The post pairs that with a conditional-access example for AI integrations, and argues conditions are the practical fallback when least privilege breaks down on shared projects where multiple teams and workloads touch the same resources.

Platform engineering for the agentic enterprise: Managing applications, resources, and AI agents (1 minute read)

The author argues that internal developer platforms were built on the assumption that a human developer is the primary consumer, and that AI agents provisioning infrastructure, deploying apps, and investigating incidents break that model. The proposed fix is a single governance layer covering both, with humans and agents hitting interfaces suited to their capabilities under the same policy enforcement, RBAC, and audit trail rather than a bolted-on API surface for agents.

ClouDens: Operational Context-Aware Anomaly Detection for Large-scale Cloud System Monitoring (1 minute read)

The authors studied telemetry logs from IBM Cloud Console and propose ClouDens, an anomaly detection framework that partitions high-dimensional telemetry into domain-guided subsets and models service dependencies as a context-aware graph fed to spatio-temporal GNNs. It beats a GRU baseline on NAB scores for count-based features, catching anomalies earlier and with wider coverage. The paper also quantifies how much feature subsetting, scoring strategy, and sparsity imputation each move detection performance, which matters for anyone benchmarking their own alerting stack.

Google Cloud outage shows it’s still hard to understand hyperscalers’ real resilience regimes (1 minute read)

The author argues that Google Cloud's latest outage exposed how little customers can actually verify about hyperscaler resilience, since published region and zone isolation guarantees rarely map to the shared control planes that fail in practice. Multi-region architectures priced as insurance still inherit correlated failure paths that no SLA document describes. The practical takeaway is to treat provider resilience claims as unverified until an incident retro proves the blast radius.

AI speeds software development. Is your secret security keeping up? (1 minute read)

HashiCorp argues that AI-assisted development is outpacing secret management, citing GitGuardian data showing nearly 30M secrets detected in public GitHub repos in 2025, up 34% year over year, with AI-generated code leaking at roughly twice the GitHub-wide rate. The post pushes detection left into IDE scanning, pre-commit hooks, and CI/CD checks rather than post-commit cleanup. It also argues remediation needs exposure context, since undifferentiated findings turn into a backlog no security team can prioritize.

The drivers behind software delivery inefficiency (1 minute read)

Researchers surveyed engineering organizations to identify what actually slows software delivery, finding that most lost time comes from waiting on other teams, unclear requirements, and flaky or slow build and test pipelines rather than from individual coding speed. The authors argue that these delays compound, since each handoff adds queue time that no amount of local developer-productivity tooling recovers. For platform teams, the leverage is in CI reliability and reducing cross-team dependencies, not in per-developer metrics.
