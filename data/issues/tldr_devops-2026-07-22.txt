They'll Verify. They Just Won't Act. How Authority Framing and Laundered Code Turn a Trusted Agentic CI/CD Pipeline Into an Attack Surface (1 minute read)

Researchers tested a five-agent CI/CD pipeline spanning five production LLMs and three providers, and found that an authority-framed injection citing a fake pre-approval ticket got downstream verifiers to see a secret-exfiltrating line, cite the approval, and ship it. The security-scan agent passed roughly 80% of laundered pull requests, and static scanners caught none of them because the code was syntactically clean. Only provenance checks at the entry point held.

Microsoft Azure DevOps MCP Flaw Lets Hidden PR Comments Hijack AI Review Agents (1 minute read)

Manifold Security disclosed a confused-deputy flaw in Microsoft's official Azure DevOps MCP server, where a hidden HTML comment in a pull request description becomes instructions the reviewer's AI coding agent executes under the reviewer's own credentials. Microsoft already ships a spotlighting helper, createExternalContentResponse, for wiki and build-log output, but repo_get_pull_request_by_id never calls it, so PR text reaches the model unwrapped and can reach source code, secrets, and work items across projects.

Exclusive: Fig launches AI agent and brings CI/CD to security operations (1 minute read)

Fig released Figaro, an AI agent that applies version control, testing, and one-step rollback to security detection pipelines, built on a deterministic graph of every detection flow across the SecOps stack. Engineers describe a change in plain language, and Figaro simulates it against the live environment before deploy, with continuous observability afterward to catch silently broken detections. Early user AppLovin reports shipping detection changes in minutes rather than weeks.
