Critical Copilot vulnerability allowed hackers to steal 2FA code from users (1 minute read)

A critical vulnerability in Microsoft Copilot, dubbed SearchLeak, let attackers exfiltrate users' 2FA codes by injecting prompts that coerced the LLM into leaking content it had access to during a session. The flaw reflects a recurring failure mode in LLM security, where untrusted input and privileged data share the same context, and patching individual exploits does not address the underlying prompt-injection exposure.

How memory safety CVEs differ between Rust and C/C++ (1 minute read)

This analysis compares memory-safety CVEs across Rust and C/C++ codebases, finding that Rust's safety guarantees push most remaining memory bugs into `unsafe` blocks or upstream C dependencies rather than safe Rust itself. For security teams weighing a migration, the operational takeaway is to focus auditing effort on `unsafe` boundaries and FFI surfaces, where Rust's compiler protections no longer apply, rather than treating the language as eliminating memory-corruption risk outright.

From Third-Party to First-Party: Measuring and Protecting Against Modern Web Tracking Mechanisms (1 minute read)

A new study measuring top sites found that over 54% now deploy first-party tracking (FPT) or server-side tracking (SST), relocating tracking logic into the browser's first-party context or the site's backend to evade client-side detection. The authors introduce a provider-independent detection method and use script-similarity clustering to map the ecosystem, giving defenders and privacy tooling a way to spot tracking that traditional blocklists and client-side tools now miss.

Transferable Self-Evolving Playbooks for Agentic Security Auditing (1 minute read)

EvoHunt automates audit playbook creation for LLM security agents, replacing human-curated prompts and heuristics with a closed evolution loop: an audit agent runs the playbook, an evaluator scores findings against ground truth, and a reviser commits updates. The authors show evolved playbooks transfer the audit procedure to weaker agents, raising their vulnerability-discovery capability without retraining the underlying model. For defenders, this points toward scalable agentic auditing where procedural knowledge, not just model strength, drives results.

Measurement Study of Post-Quantum Readiness of Internet: 2026 (1 minute read)

A measurement study of 32,011 domains found that 49.3% now support hybrid post-quantum key exchange such as MLKEM768 with X25519, but 15.70% still negotiate only TLS 1.2, concentrated in banking and government. The authors show post-quantum adoption is uneven across sectors, leaving the most sensitive infrastructure least prepared for harvest-now-decrypt-later attacks. Defenders in lagging sectors should prioritize TLS 1.3 migration and enable hybrid key exchange before classical ECC and RSA become quantum-breakable.

AI’s constant patching treadmill can be a security problem (1 minute read)

Backslash Security reviewed two months of Claude Code update logs and found Anthropic silently shipped 30-plus security-relevant patches between April and early June 2026, covering prompt injection, data poisoning, OAuth credential leakage, and arbitrary code execution. One fix closed a bypass where adding a single backslash defeated guardrails against catastrophic deletion commands. The takeaway: AI coding tools patch at a far higher cadence than traditional software, so enable auto-updates and track changelogs for undisclosed fixes.

SoK: Taxonomizing the Low-Level Attack Surface of Modern Web Browsers (1 minute read)

This paper systematizes the low-level attack surface of modern web browsers, mapping where memory-corruption flaws arise across input classes and components to expose regions underexplored by current fuzzing and testing. The authors structure the surface along input types and browser components, then chart how memory-corruption bugs cluster within that taxonomy. For AppSec and red teams, it offers a defense-oriented map of which browser subsystems warrant deeper security testing rather than continued effort on already-saturated targets.

DoubtProbe: Black-Box Jailbreak Defense via Structural Verification and Semantic Auditing (1 minute read)

DoubtProbe is a black-box jailbreak defense that reframes detection as consistency checking under controlled transformation, on the observation that most jailbreaks reorganize a harmful goal rather than remove it. A structural branch reconstructs the request from a normalized representation while a semantic branch audits intent, flagging inputs whose meaning shifts across transformation. The approach targets evolving prompt packaging and structure manipulation that known-attack coverage and prompt-level judgment tend to miss.

SPARK: Security Knowledge Priming and Representation-Guided Knowledge Activation for LLM-based Secure Code Generation (1 minute read)

SPARK is an inference-time security harness that reduces LLM-generated code vulnerabilities without retraining, arguing the security knowledge is already latent in pretraining and only needs activation. It retrieves relevant CWE entries per coding task and appends a short structured cue to the prompt, surfacing the model's existing safety representations. The approach sidesteps the compute overhead and data bias of fine-tuning or heavy retrieval-augmented defenses.

Stop Using JWTs (1 minute read)

The author argues developers should stop using JWTs for user session management and use regular cookie sessions instead. The case rests on a few points: the JWT spec targets very short-lived tokens (~5 minutes), secure "stateless" auth is not actually feasible since you need server-side state to revoke tokens, and a JWT carrying only a session ID gains nothing over a plain session cookie while adding spec-level risk. Google's JWT usage is SSO transport between hosts, not browser sessions.

Binary Decompilation LLM with Feedback-Driven Multi-Turn Refinement (1 minute read)

AutoDecompiler is a decompilation-specialized LLM trained with reinforcement learning that reframes binary decompilation as iterative multi-turn refinement, revising its output based on compilation, execution, and input/output testing feedback rather than generating a single pass. The approach targets a common failure mode in LLM decompilation, where generated code compiles and reads cleanly yet diverges from the original binary's behavior, misleading downstream vulnerability discovery and malware analysis.

obliv-clang: Real-World Oblivious Programming in C++ (1 minute read)

This paper introduces obliv-clang, a compilation-time tool that checks the obliviousness of C++ programs to defend against timing and access-pattern side-channel attacks on confidential data in trusted execution environments. It supports rich C++ features including arbitrarily nested pointers, formally proves the soundness of its checking rules, and aims to integrate with existing industry codebases while keeping compilation overhead and runtime performance costs low.

MASCOT-Android: A Curated Dataset and Automated Collection Pipeline for Android Malware Source Code Specimens (1 minute read)

MASCOT-Android is a curated dataset of Android malware source code paired with an automated framework that scrapes GitHub for new specimens, using repository documentation as the primary collection signal. A LinearSVC classifier trained on character-level TF-IDF features from README files alone distinguishes malware from benign repos at 96.28% accuracy and a 1.06% false-positive rate, with tunable confidence thresholds to balance coverage against FPR.
