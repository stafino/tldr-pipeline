What happens to oversight when AI agents write a lab’s own code (1 minute read)

Researchers at the University of Oxford and SaferAI applied systems-safety methods from aviation, nuclear, and chemical engineering to a generic frontier AI lab, examining the oversight around coding agents that write and run code across production and training infrastructure. They found control actions with no named owner, including the authority to pause or roll back a deployment and the catastrophic-risk reporting SB 53 now mandates, plus offline monitoring that reviews internal coding sessions only after they finish.

Homebrew tightens tap security, begins work on its interface (1 minute read)

Homebrew 6.0.0 now requires any third-party tap, and its formulae and casks, to be explicitly trusted before their Ruby code is evaluated or run, closing a long-standing gap where tap code executed unsandboxed by default. Official Homebrew taps remain trusted automatically, and the brew tap command adds options for managing trust on the taps a user opts into.

AWS Continuum brings AI models to code vulnerability management (1 minute read)

AWS released Continuum for code vulnerabilities in gated preview, a model-agnostic system that runs discovery, prioritization, validation, and remediation across a vulnerability's lifecycle, building sandboxed exploit proofs to filter false positives and recommending network, policy, or code fixes. It reasons over structured and unstructured inputs, starts in a learn mode with human review, and routes work to whichever frontier model performs best per task.

The Australian Government to Require SMS/MMS Sender ID Registraion (1 minute read)

The register becomes mandatory after a transition period, with unregistered sender IDs filtered or blocked by carriers to curb the SMS phishing that drives much of Australia's scam-message volume.
