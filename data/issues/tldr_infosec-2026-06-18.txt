Attacks & Vulnerabilities

Massive breach spills credentials for thousands of sensitive networks (1 minute read)

Researchers disclosed a credential-spraying campaign against internet-facing FortiGate firewalls that compromised nearly 74,000 devices across 194 countries, dumping plaintext credentials for organizations including Oracle, Chevron, Lenovo, FedEx, a NATO defense contractor, and Fortinet itself. Roughly half of all exposed Fortinet firewalls are affected and most remain online with valid credentials, so operators should treat compromised devices as live, rotate credentials, and audit downstream RADIUS and Active Directory access immediately.

Microsoft Confirms RoguePlanet Defender Zero-Day, Says Patch is in Development (1 minute read)

Microsoft confirmed a Defender zero-day codenamed RoguePlanet (CVE-2026-50656, CVSS 7.8), a race condition in the Malware Protection Engine that hands attackers a SYSTEM-level shell regardless of whether real-time protection is enabled. No patch has shipped yet, public PoC code from researcher Chaotic Eclipse already exists, and this is the fourth Defender flaw the same researcher has disclosed after BlueHammer, UnDefend, and RedSun.

FortiBleed leak exposes Fortinet VPN credentials for 73,000 devices. (1 minute read)

A leak dubbed "FortiBleed," discovered by researcher Bob Diachenko, exposed valid FortiGate SSL VPN credentials, including plaintext passwords, for 73,932 firewall URLs at organizations including Chevron, Samsung, AT&T, and Mercedes-Benz. Diachenko attributes the haul to a Russian-speaking group that intercepted and cracked SSL VPN authentication hashes; affected organizations should treat exposed appliances as compromised, force password resets, and audit Active Directory for lateral movement.

144 Mastra npm Packages Compromised via Hijacked Contributor Account (1 minute read)

Attackers hijacked the dormant npm account of a former Mastra contributor ("ehindero") and mass-published 144 malicious "@mastra/*" packages on June 17, each adding a tainted "easy-day-js" dependency whose postinstall hook drops a cross-platform infostealer targeting browser history and 160+ crypto wallet extensions. Audit Mastra installs from the past 24 hours, rotate any exposed credentials and wallet keys, and pin to releases carrying SLSA provenance attestations, since the attacker published from a personal token without them.

PhantomSkill: Malicious Code Injection in Agent Skill Ecosystems (1 minute read)

This paper presents PhantomSkill, an attack framework that hides malicious behavior in the auxiliary resources of third-party agent skills rather than their text descriptions, using a technique called VulMask to rewrite overt malware into trigger-activated, vulnerability-shaped code that evades automated reviewers. The authors argue skill ecosystems need resource-level vetting and execution-time containment, treating exploitable vulnerabilities in agent skills as potential malicious payloads.

Strategies & Tactics

CodeSentinel: A Three-Layer Defense Against Indirect Prompt Injection in Code Contexts (1 minute read)

CodeSentinel is an inference-time sanitizer that defends code LLMs against indirect prompt injection hidden in retrieved context like comments, strings, identifiers, and decoy code. It uses Tree-sitter to extract high-risk CST nodes, then applies syntax-guided pre-filtering, CST-guided Dynamic Min-K% scoring, and node perturbation analysis to detect and neutralize adversarial triggers before they reach the model. Across six attack families it hit 0.80 average node-level F1, outperforming CodeGarrison, DePA, and KillBadCode.

OpenAnt: LLM-Powered Vulnerability Discovery Through Code Decomposition, Adversarial Verification, and Dynamic Testing (1 minute read)

OpenAnt is an open-source pipeline that pairs static analysis with LLM reasoning to find vulnerabilities in large codebases, decomposing them into reachability-filtered units to cut the analysis surface by up to 97%. Candidate bugs pass through adversarial verification under simulated attacker constraints, then dynamic validation in auto-generated, sandboxed exploit environments. On OpenSSL, WordPress, and Flowise, the closed-loop approach surfaced previously unknown vulnerabilities while substantially reducing the false positives that make static scanners hard to triage.

Evaluating Prompting-Based Defenses Against Domain-Camouflaged Injection Attacks (1 minute read)

This paper evaluates five prompting-based defenses against domain-camouflaged injection attacks, which hide malicious instructions in domain-appropriate vocabulary to slip past syntactic detectors, across three model families and 3,510 trials. Paraphrasing retrieved content before agent processing was the most consistent defense, cutting camouflage attack success by 55-84% and beating a Llama Guard 4 configuration on every model. Effectiveness is heavily model-dependent, and financial deployments retained 26-33% baseline attack success with no prompting defense fully closing the gap on weaker models.

CloakLM: Obfuscating GPU Memory Layout to Mitigate Model Ex-filtration for Serving (1 minute read)

CloakLM is a software-only framework that obfuscates GPU memory layout to block model exfiltration from untrusted accelerator infrastructure, countering attacks like Hermes (PCIe reconstruction) and TunnelS (HBM dumps). It combines PCIe traffic shaping, weight shuffling, and HBM page remapping so authorized inference sees a valid layout while observers see fragmented state. The authors report near-native performance on LLaMA and Qwen under vLLM, with no hardware changes required.

Code-Augur: Agentic Vulnerability Detection via Specification Inference (1 minute read)

This paper introduces Code-Augur, an agentic vulnerability-detection harness that forces LLM agents to commit their tacit security assumptions as in-source assertions, then runs a guided fuzzer to falsify them. When the fuzzer trips an assertion, it surfaces either a real vulnerability or a flawed specification to refine, grounding the agent's reasoning in actual runtime behavior. The authors report 22 new vulnerabilities in open-source projects and higher detection rates than prior agents, using widely available models like Sonnet and DeepSeek rather than specialized ones like Claude Mythos.

Quantifying Compromise Risk in Exceptional Access Architectures Under Sparse and Indirect Evidence (1 minute read)

This paper builds a structured uncertainty framework to quantify systemic compromise risk in lawful exceptional access (EA) architectures, modeling transmission-layer (T-EA) and over-the-top (OTT-EA) designs separately. Across all calibrations, EA-equipped systems carry strictly higher risk than no-EA counterfactuals, with T-EA's annual compromise probability ranging 1.4% to 12.9% and OTT-EA risk concentrated in the tail under correlated campaigns. Because key-material exfiltration is irreversible, the cumulative multi-decade risk weighs heaviest on OTT-EA's larger user populations.

Lifecycle-Aware Dynamic Analysis for Secure ML Model Execution (1 minute read)

This paper proposes Moat, a dynamic, lifecycle-aware approach to securing ML model execution that watches the effects a model has on the host system rather than scanning artifacts for known signatures or format-specific rules. The key observation is that models operate in well-defined lifecycle phases with structured, predictable host interactions, so deviations flag malicious behavior. Evaluated against 77,974 Hugging Face artifacts and 31 CVE PoCs, the reference implementation Re-Moat detected every tested attack class at a near-zero false-positive rate.

AI’s constant patching treadmill can be a security problem (1 minute read)

Backslash Security reviewed two months of Claude Code update logs and found Anthropic silently shipped 30-plus security-relevant patches between April and early June 2026, covering prompt injection, data poisoning, OAuth credential leakage, and arbitrary code execution. One fix closed a bypass where adding a single backslash defeated guardrails against catastrophic deletion commands. The takeaway: AI coding tools patch at a far higher cadence than traditional software, so enable auto-updates and track changelogs for undisclosed fixes.

Compute-Budgeted Exploitability Evidence Graphs for Prospective Vulnerability Triage (1 minute read)

This paper proposes a leakage-safe protocol for evaluating vulnerability triage prospectively, scoring each CVE only from public evidence (advisories, exploit archives, fix commits, forum discourse) visible by a fixed decision time, with every score paired with an auditable certificate of supporting signals. On 12,012 prospective CVEs, budgeted evidence selection raised leakage-safe recall@50 from 0.010 to 0.026, while a naive random split inflated apparent recall by 8.5x. The takeaway for defenders: most retrospective exploitability benchmarks overstate real-world prioritization performance.

Detecting Hidden ML Training With Zero-Overhead Telemetry (1 minute read)

This paper tests whether GPU compute-governance monitoring can survive an adversary trying to hide training runs. Using only zero-overhead, privacy-preserving NVML telemetry, the authors built a classifier that hits 98.2% binary accuracy at flagging training workloads across 9 GPU models and 4 architecture generations, and holds 43-87% accuracy against adversarially disguised workloads over 5 rounds of monitor-evader iteration. The takeaway for governance schemes is that content-agnostic side-channel signals are robust enough to detect hidden training without ever touching model weights or data.

TRAP: Benchmark for Task-completion and Resistance to Active Privacy-extraction (1 minute read)

This paper introduces TRAP, a benchmark testing whether AI agents can use private fields like passport numbers to complete tasks without leaking them to an attacker prompting in natural language. Across 22 frontier and open-source models, all leaked private data, and stronger instruction-following correlated with higher leakage; prompt-based defenses cut leakage only by degrading task accuracy. The authors prove no soft-constraint defense can hit both goals, and propose replacing private fields with hash keys before they reach the model.

CHERI-D: Secure and efficient inline object ID for CHERI temporal memory safety (1 minute read)

This paper introduces CHERI-D, an architectural extension that stores per-object ID metadata inline with allocation data to give CHERI hardware native temporal memory safety. It enables strict use-after-free mitigation rather than the weaker use-after-reallocation protection of the current software-based approach, Cornucopia Reloaded, while substantially cutting the revocation overhead that solution incurs. Evaluated in both simulation and hardware, it points toward UAF defenses cheap enough to run by default.

Stop Using JWTs (1 minute read)

The author argues developers should stop using JWTs for user session management and use regular cookie sessions instead. The case rests on a few points: the JWT spec targets very short-lived tokens (~5 minutes), secure "stateless" auth is not actually feasible since you need server-side state to revoke tokens, and a JWT carrying only a session ID gains nothing over a plain session cookie while adding spec-level risk. Google's JWT usage is SSO transport between hosts, not browser sessions.

TGCM: Topic-Guided Generative Disentanglement of Interleaved APT Technique Sequences (1 minute read)

This paper introduces TGCM, a generative disentanglement framework that separates concurrent APT campaigns from interleaved audit logs, jointly inferring how many campaigns are present and assigning each technique to its source. It uses Consistency Models for single-step inverse mapping and a topic-guided prior built from MITRE ATT&CK narratives to keep recovered attack chains coherent. Across DARPA TC-E3/E5 traces and 15 baselines, it improved separation under heavy technique sharing and generalized zero-shot to in-the-wild interleaved data without retraining.

Signature filtering: a lightweight enhancement for statistical watermark detection in large language models (1 minute read)

This paper introduces signature filtering, a detection-time add-on that strengthens statistical watermark detection in LLM outputs without changing how watermarks are embedded. It learns a small set of "signature" tokens that make watermark tests unreliable and strips them before detection, raising detection rates in weak-signal and low-entropy settings from 8–31% to 78–99% while keeping false positives negligible. The method is model-agnostic and holds up under scrambling and 25–50% token perturbation, improving provenance checks for LLM-generated text.

Runtime Compliance Verification for AI Agents (1 minute read)

This paper introduces C-Trace, a framework that enforces GDPR compliance on AI agents at runtime by expressing rules like consent, purpose limitation, data minimization, and the right to erasure as formal predicates over execution traces. A runtime monitor intercepts every tool call and model output, rejecting non-compliant actions. Across four case studies, it held attack success rate at or below 12% under 10% extractor noise and 0% with perfect extraction, with false positives at or below 16%.

AI vs Rule-Based Secret Scanning in CI: Which Actually Works (3 minute read)

The author argues the AI-vs-rule-based secret scanning debate is a false binary, drawing on running both in production across orgs from 5 to 200 developers. Rule-based tools like Gitleaks (150+ secret types, fully offline, deterministic) catch known patterns cheaply, while ML scanners like GitGuardian and Nightfall add context to flag obfuscated or base64-encoded secrets that defeat regex. The practical takeaway is to layer them, since leaked credentials in CI logs and git history remain a top initial-access vector.

AI Sandboxes: A Threat Model, Taxonomy, and Measurement Framework (1 minute read)

This paper proposes a threat model, taxonomy, and measurement framework for AI sandboxes, the bounded environments used to test, verify, and validate digital AI, embodied autonomy, and cyber-physical systems. The authors formalize the sandbox boundary and a weakest-link rule for composing per-dimension evidence, and define measurements spanning fidelity, controllability, observability, containment, reproducibility, and governance. Notably, their threat model includes attacks on the assurance apparatus itself, clarifying which risks a sandbox can actually contain and what evidence it can support for regulatory assurance.

Evaluating the Effectiveness of LLMs in Aiding Compliance Testing of PKCS#1-v1.5 (1 minute read)

This paper tests whether grammar-level mutation plus LLM code synthesis can generate spec-compliance test inputs for PKCS#1 v1.5 signature verification, evaluated against the Morpheus oracle across 48 cryptographic library implementations. The approach reproduced 10 of 13 known violation categories and surfaced one previously unreported discrepancy, but LLM hallucination corrupted 82.5% of generated scripts. The authors quantify a gap between 99.8% operational reliability and 17.5% semantic fidelity, arguing LLM synthesis is trustworthy only where structural correctness, not faithful mutation, is what matters.
