Attacks & Vulnerabilities

Massive breach spills credentials for thousands of sensitive networks (1 minute read)

Researchers disclosed a credential-spraying campaign against internet-facing FortiGate firewalls that compromised nearly 74,000 devices across 194 countries, dumping plaintext credentials for organizations including Oracle, Chevron, Lenovo, FedEx, a NATO defense contractor, and Fortinet itself. Roughly half of all exposed Fortinet firewalls are affected and most remain online with valid credentials, so operators should treat compromised devices as live, rotate credentials, and audit downstream RADIUS and Active Directory access immediately.

CISA warns Fortinet users to secure devices after FortiBleed leak (1 minute read)

CISA warned Fortinet customers to secure their appliances after the "FortiBleed" leak exposed plaintext VPN and admin credentials for 73,932 FortiGate firewalls across 21,632 domains and 194 countries, with threat actors already using the data to target internet-facing devices. Affected owners should terminate all SSL VPN and admin sessions, reset VPN and admin passwords, enable phishing-resistant MFA, and review logs for lateral movement.

Splunk Enterprise Vulnerability Exploited in Attacks Days After Disclosure (1 minute read)

Splunk confirmed limited in-the-wild exploitation of CVE-2026-20253, a critical unauthenticated flaw in Splunk Enterprise where the PostgreSQL sidecar service endpoint lacks authentication controls, allowing any network-reachable attacker to create or truncate arbitrary files and achieve RCE. Patches landed June 10 for versions 10.2 before 10.2.4 and 10.0 before 10.0.7; CISA added it to KEV on June 18 with a June 21 federal remediation deadline.

Microsoft Confirms RoguePlanet Defender Zero-Day, Says Patch is in Development (1 minute read)

Microsoft confirmed a Defender zero-day codenamed RoguePlanet (CVE-2026-50656, CVSS 7.8), a race condition in the Malware Protection Engine that hands attackers a SYSTEM-level shell regardless of whether real-time protection is enabled. No patch has shipped yet, public PoC code from researcher Chaotic Eclipse already exists, and this is the fourth Defender flaw the same researcher has disclosed after BlueHammer, UnDefend, and RedSun.

How hacker group TeamPCP exploited the open source trust model and distribution method to compromise and inject malware into over 1,000 software packages (Matt Kapko/CyberScoop) (1 minute read)

The hacker group TeamPCP compromised over 1,000 open source software packages by abusing the trust and distribution model of public registries to inject malware into legitimate projects. The campaign turned the open source supply chain into a malware delivery vector, so defenders should audit recently updated dependencies, pin known-good versions, and verify package integrity rather than trusting registry availability alone.

Popa operates as a plugin of the Vo1d malware campaign, scattering ad fraud, account takeovers, and data-scraping traffic across more than 1.4 million IP addresses.

Strategies & Tactics

Build your own vulnerability harness (1 minute read)

Cloudflare details how it built a model-agnostic vulnerability harness for Project Glasswing, deliberately swapping models across the pipeline so one model handles discovery and a different one validates, cross-checking findings through distinct reasoning. The architecture persists hundreds of investigations across runs, traces vulnerabilities through cross-repo dependencies, and filters thousands of raw candidates down to a triaged queue of actionable fixes.

npm v12’s Biggest Security Change: From Implicit to Explicit Trust (1 minute read)

npm v12, expected July 2026, shifts package installation from implicit to explicit trust by blocking three high-risk vectors by default: install scripts (allowScripts), Git-repository installs (--allow-git), and remote-URL installs (--allow-remote). JFrog's research team ties these mechanisms to roughly 53% of malicious npm attacks over the past year, including multiple Shai-Hulud variants that abused install-time execution to steal credentials and pivot into CI/CD. Teams should audit builds for dependencies relying on these behaviors before upgrading.

Calibration Without Comprehension: Diagnosing the Limits of Fine-Tuning LLMs for Vulnerability Detection in Systems Software (1 minute read)

A new study, CWE-Trace, evaluated eight base LLMs and 15 LoRA fine-tuned variants on 834 curated Linux kernel samples across 74 CWEs, using a strict temporal split to rule out data leakage. Fine-tuning only shifted output thresholds without changing decision policy ("calibration without comprehension"), and contamination gave no measurable edge. Best detection hit just 52.1%, barely above chance, with exact CWE ranking below 1.3% Top-1. Treat LLM vuln-detection output as a weak signal, not a reasoning engine.

How software development’s speed obsession enabled TeamPCP’s chaos crusade (1 minute read)

TeamPCP has compromised and injected malicious code into more than 1,000 open-source packages in under four months, starting with Trivy in February, exploiting the industry's unverified trust in third-party dependencies. The scale rides on automated CI/CD pipelines and AI coding agents that install unvetted packages with no human in the loop. The tactics aren't novel; defenders should enforce dependency pinning, signature verification, and gating of agent-installed packages before ingestion.

DISARM: Target Electronic Device Informed Mitigation of Software Runtime Side-Channel Vulnerabilities (1 minute read)

DISARM is a joint hardware-software methodology for mitigating runtime timing side-channels that uses measured timing values from real embedded devices to generate targeted software fixes, avoiding the over-fixing and under-fixing that plague hardware-agnostic tools. The authors validate it across 22 benchmarks for C, C++, and Java, reporting lower execution-time and code-size overhead with better correctness than PENDULUM and DifFuzzAR on five embedded and edge devices.

Quantization as a Malicious Task: Removing Quantization-Conditioned Backdoors via Task Arithmetic (1 minute read)

This paper introduces QVec, a defense against quantization-conditioned backdoors (QCBs), where a model behaves benignly in full precision but activates malicious behavior once quantized for edge deployment. The authors treat the weight difference between the full-precision and quantized models as a structured malicious task vector and subtract it via task arithmetic, requiring no retraining, no trigger samples, and a single quantization pass. Across image classifiers and multiple LLM attack scenarios, QVec suppressed backdoor activation while preserving clean accuracy.

MosaicLeaks: Can your research agent keep a secret? (5 minute read)

MosaicLeaks introduces a deep-research benchmark showing that agents combining private documents with web retrieval leak sensitive enterprise data through their outbound query logs, where an observer reassembles fragments via the mosaic effect without ever seeing the documents. The authors found leakage is common across tested models and that training purely for task accuracy worsens it. Their privacy-aware RL method, PA-DR, cut answer and full-information leakage from 34.0% to 9.9% while raising strict chain success from 48.7% to 58.7%.

Companies are discarding the logs they need to catch a breach (1 minute read)

Half of enterprises drop or never collect an average of 86% of their logs to cut costs, according to a Dynatrace survey of 450 IT leaders, and many retain the rest only briefly. Since intrusions often sit undetected for weeks, investigators reaching for the trail find the relevant entries sampled away or aged out. The retention decision typically sits with observability or platform teams chasing spending targets, outside security's control.

A Measurement Study of Cryptographic Misuse in Embodied AI Mobile Applications (1 minute read)

Researchers published the first large-scale measurement study of cryptographic misuse in embodied AI mobile apps, analyzing 507 applications (EAIAppZoo) and surfacing 12,975 misuse findings at 80.74% precision. The flaws are structural rather than random: latency-sensitive control paths weaken transport encryption, while offline provisioning and legacy IoT SDKs drive hardcoded credentials. The authors show these mobile-side weaknesses bypass network protections, letting adversaries intercept command channels and hijack physical control of EAI devices.

PUFFERDOS: Efficient and Effective Attack String Generation for Regular Expression Denial of Service Vulnerabilities (1 minute read)

This paper introduces PUFFERDOS, a ReDoS attack-string generator that synthesizes inputs feasible within realistic length budgets and validates them at the program level, addressing prior tools that produced strings too long or unreliable to trigger vulnerable regexes in real programs. It defines three vulnerable regex patterns, then refines candidates with ReDoS-specific compositional concolic execution to guarantee real-world exploitability. The takeaway for defenders: pair regex linting with program-level validation, since static pattern-matching alone overstates which ReDoS findings are actually exploitable.

Multi-View Decompilation for LLM-Based Malware Classification (1 minute read)

This paper argues that LLM-based malware triage on decompiled pseudo-C is fragile when it relies on a single decompiler, since tools like Ghidra and RetDec are lossy and expose different artifacts of the same binary. The authors built a benchmark of benign utilities and malicious programs decompiled with both tools, and found that feeding LLMs both views raises malicious-class F1, mostly by improving recall. Multi-decompiler prompting is a training-free way to cut false negatives in LLM-assisted triage.

Analyzing Defensive Misdirection Against Model-Guided Automated Attacks on Agentic AI Systems (1 minute read)

This paper argues that conventional detect-and-block defenses against prompt injection and jailbreaks can let attacker success rate approach 100% as query budgets grow, because predictable refusals give model-guided attack tools clean feedback to optimize against. The authors propose detect-and-misdirect: returning safe but deliberately misleading responses to poison the attacker's automated judge. Their proof-of-concept, CMPE, cut estimated ASR upper bounds by up to two orders of magnitude and nearly eliminated verified successes in PAIR and GPTFuzz runs.

Secure Coding Drift in LLM-Assisted Post-Quantum Cryptography Development: A Gamified Fix (1 minute read)

This paper introduces "Secure Coding Drift," a model for how sustained reliance on LLM-generated code gradually erodes secure coding practices in post-quantum cryptography work, where constant-time execution, side-channel resistance, and exact parametrisation are mandatory. The authors frame the risk as a longitudinal behavioral phenomenon rather than a static vulnerability, and propose a gamified framework that embeds adversarial evaluation, behavioral feedback, and security scoring into the workflow to turn the LLM into an active reviewer rather than a passive code generator.

When Global Gating Is Enough: Admission-Time Hubness Control in Anisotropic Vector Retrieval Systems (1 minute read)

This paper proposes admission-time hubness control to defend RAG systems against poisoning, where a single injected document becomes the nearest neighbor for many unrelated queries. Rather than periodic reverse-kNN scans that leave an exposure window, the authors score each candidate against sentinel queries and quarantine hub-like documents before insertion. A single global gate hit recall 1.0 at the decisive embedding point and 0.91 on HotFlip attacks with 1% false positives, adding roughly 3.1% to HNSW ingestion latency and scaling flat to 10^6 vectors.

A Layered Security Framework Against Prompt Injection in RAG-Based Chatbots (1 minute read)

This paper introduces a three-layer middleware framework that blocks both direct and indirect prompt injection in RAG chatbots without modifying the underlying LLM. Layer 1 screens input, Layer 2 enforces a provenance-based instruction hierarchy so retrieved documents cannot override operator policy, and Layer 3 audits output for policy violations and semantic drift. Across 5,080 samples on GPT-4o, Llama 3, and Mistral 7B, it cut Attack Success Rate from 71.4% to 11.3% at a 4.8% false-positive rate and 61.2ms median latency.

NAMESAKES: Probing Identity Memorization in Text-to-Image Models (1 minute read)

This paper introduces a fully black-box probe that detects whether a text-to-image model has memorized a real person's likeness, requiring no reference photos, training-data access, or model internals. Tested on the new NAMESAKES dataset of over 1,000 public-figure names across fame levels, the probe reliably separates memorized identities from fabricated ones. For teams assessing T2I deployments, it offers a practical way to audit privacy exposure without white-box access to the model.

When Lower Privileges Suffice: Investigating Over-Privileged Tool Selection in LLM Agents (1 minute read)

This paper introduces ToolPrivBench, a benchmark measuring whether LLM agents pick higher-privilege tools when a sufficient lower-privilege alternative exists. Across eight domains and five risk patterns, the authors find over-privileged selection is common in mainstream agents and worsens after transient tool failures, and that general safety alignment and prompt-level guardrails don't reliably enforce least privilege. Their privilege-aware post-training defense substantially cut unnecessary high-privilege tool use while preserving capability, pointing to alignment rather than prompts as the durable control for agentic tool access.

Sovereign Execution Brokers: Enforcing Certificate-Bound Authority in Agentic Control Planes (1 minute read)

This paper introduces the Sovereign Execution Broker (SEB), a runtime enforcement boundary that sits between an agent's proposed action and the production API call, ensuring mutations only execute against a valid, certificate-bound contract. SEB verifies the requested mutation matches the certified execution contract, checks validity windows and revocation epochs, detects live-state drift, then mints short-lived scoped identity and logs signed decision records. The model only holds if production mutation APIs reject any non-broker identity, which is the deployment burden it places on operators.

Your browser tab could become encrypted storage for someone else’s files (1 minute read)

A research paper from IENYC's Gregory Magarshak describes Safecloud, a decentralized storage design where browser tabs act as encrypted storage nodes: files are chunked and encrypted on the owner's device before leaving, so storage nodes ("Drops") hold only ciphertext in IndexedDB and routing nodes ("Jets") never see keys. Every key derives from a single 32-byte root secret, and three overlapping trees handle integrity, confidentiality, and authorization over one addressing path, enabling content-addressed dedup and single-key segment streaming without exposing plaintext to any participant.

AutoTam: Specifying Secure Protocol Implementations with Tamarin Model Generation (1 minute read)

AutoTam introduces a language-first method for formally verifying cryptographic protocol implementations, generating Tamarin prover models from a domain-specific language and proving that verified trace properties carry back to the running code. The authors pair symbolic execution for memory-safety analysis with model verification, then demonstrate the approach on signed Diffie-Hellman and a WireGuard VPN implementation that interoperates with existing clients at acceptable performance.
