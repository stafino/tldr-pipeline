PixelSmash – Critical FFmpeg Vulnerability Turns Media Files into Weapons (1 minute read)

JFrog disclosed PixelSmash (CVE-2026-8461, CVSS 8.8), a heap out-of-bounds write in FFmpeg's MagicYUV decoder that escalates from a crash to reliable RCE when an application processes a single crafted media file. Researchers demonstrated full code execution against Jellyfin and Nextcloud via a 50 KB AVI upload; upgrade FFmpeg now, or disable the MagicYUV decoder at build time if it isn't needed.

New Exploit Bypasses Apple’s Boot Defenses, Affects Millions of iPhones (1 minute read) [flag]

Paradigm Shift disclosed Usbliter8, an unpatchable BootROM exploit in Apple's SecureROM that chains a USB controller bug and a firmware config weakness to trigger an out-of-bounds write and gain full code execution before the OS loads on A12/A13 iPhones and S4/S5 Apple Watches. The attack requires physical USB access and cannot read user data directly since the Secure Enclave stays intact, but PoC code is public and the affected chips cannot be fixed via software, making it most useful to forensics vendors.

Decades-Old Squid Proxy Flaw ‘Squidbleed’ Can Expose User Data (1 minute read)

Squidbleed (CVE-2026-47729), a Heartbleed-style memory leak that has sat in Squid Proxy's FTP parser since 1997, lets an attacker controlling a reachable FTP server read past a buffer boundary into uncleared HTTP request data from other users, exposing credentials, session tokens, and API keys in shared proxy environments. Exposure is limited to cleartext HTTP and TLS-terminating deployments; patches shipped in Squid 8 and 7.6, and disabling unused FTP support mitigates the risk.

Hundreds of AI-powered iOS apps found exposing credentials (1 minute read)

Researchers at Wake Forest University analyzed 444 iOS apps with LLM features and found 282 (64%) exposing exploitable credentials, including 136 leaking authentication tokens, 92 allowing unauthenticated backend access, and 54 hardcoding plaintext API keys. The leakage was systemic rather than niche, affecting apps with up to 2.3 million ratings, and 55% routed traffic through custom developer backends that provider-side mitigations alone cannot fix.

Fortinet says the FortiBleed campaign harvesting credentials from its firewalls and VPNs exploits no new vulnerability, instead reusing credentials from prior breaches and brute-forcing devices lacking MFA, yielding 86,000 working credentials across 194 countries. Compromised FortiGate customers should terminate admin and VPN sessions, rotate credentials, enforce MFA, and upgrade to releases supporting PBKDF2 hashing of administrator credentials.

New OXLOADER Loader Uses Malicious Google Ads to Deliver CastleStealer (1 minute read)

Elastic Security Labs disclosed REF8372, a financially motivated campaign that uses malicious Google Ads impersonating a verified Ukrainian advertiser to deliver CastleStealer via a new loader dubbed OXLOADER. Victims searching for software like Node.js are redirected to fake sites serving Storj-hosted batch scripts that fetch OXLOADER, which uses control-flow flattening, mixed Boolean-Arithmetic, and DLL side-loading to evade static detection and deploy the .NET stealer.

How Claude Mythos found a 15-year-old bug in Mozilla Firefox | Brian Grinstead (1 minute read)

Mozilla distinguished engineer Brian Grinstead detailed the agentic harness behind Firefox's record month of security fixes, including a 15-year-old bug, and argues the pipeline deserves roughly half the credit versus Anthropic's Mythos model. Key components: an LLM judge that scores and ranks files before spending compute, a verifier subagent that kills false positives, and a goal-loop that retries scoped problems past the point a human quits. Humans still review every patch.

Encrypted DNS still tells an eavesdropper where to look (1 minute read)

Encrypted DNS over TLS, HTTPS, QUIC, and CoAP hides query contents but leaves plaintext headers that mark a flow as DNS, letting an eavesdropper on a wireless link single it out to block, profile, or analyze it. A new study trained a Random Forest classifier on 296 IoT scenarios and found addresses, ports, and monotonic counters leak the most. The authors propose eliding headers via SCHC compression and CoAP block-wise transfer to even out packet lengths.

What the Latest ShinyHunters Breaches Reveal About Modern Cyberattacks (1 minute read)

ShinyHunters' latest wave of breaches (University of Nottingham, DentaQuest, 7-Eleven, Medtronic, Wynn Resorts) shows the group rarely exploits software flaws, instead abusing identity: infostealer-harvested credentials, OAuth token theft, MFA fatigue, vishing, help desk impersonation, and overly permissive SaaS and guest-access configs across Salesforce, Snowflake, and Okta. Defenders should enforce phishing-resistant MFA, audit OAuth grants and guest-user permissions, and monitor for anomalous authentication rather than relying on perimeter controls.

A Glimpse into the “Search Your Target” Market for Stolen Credentials (1 minute read)

Flare analyzed 470 underground forum posts from January 2025 to June 2026 advertising "search your target" services, a dedicated layer that lets buyers query infostealer databases of tens of billions of lines for credentials matching a specific company, domain, platform, or geography instead of buying bulk combo lists. The brokers sit between infostealer infections and account takeover, offering filtering, deduplication, and formatting into outputs like URL:LOGIN:PASS and MAIL:PASS. Buyer feedback flags a gap between advertised and actual quality, with lower volumes and many invalid or duplicated credentials.

Agent Beacon: Open-source telemetry layer for AI agents (1 minute read)

Asymptote Labs released Beacon, an open-source (MIT, mostly Go) telemetry layer that configures OpenTelemetry export and hooks for AI coding agents like Claude Code, Codex CLI, and Cursor, then normalizes their file edits, command execution, tool use, and approval decisions into a single JSON event log. It runs locally without an account, binds its dashboard to loopback, offers redaction and retention modes, and can forward events into customer-managed SIEMs.

Google Sets Sept. 30 Deadline for Android Developer Verification in Four Countries (1 minute read)

Google will begin enforcing Android developer verification on September 30, 2026, blocking normal installs of unverified apps on certified devices in Brazil, Indonesia, Singapore, and Thailand across Google Play and OEM stores from Samsung, Xiaomi, OPPO, vivo, Honor, and Transsion. Developers must register a legal identity and sign each app to stay installable; unregistered apps will only reach certified phones via ADB or Google's 24-hour high-friction sideload flow, hitting independent and open-source channels hardest.
