FFmpeg fixes PixelSmash flaw in widely used video decoder (1 minute read)

FFmpeg patched PixelSmash (CVE-2026-8461, CVSS 8.8), a heap out-of-bounds write in the MagicYUV decoder triggered by a malicious AVI, MKV, or MOV file processed through libavcodec. JFrog achieved full RCE against a Jellyfin 10.11.9 server via its automatic library scan; Kodi, OBS, PhotoPrism, Nextcloud, and desktop thumbnail generators are also exposed, with RCE viable when ASLR is disabled or chained around.

FFmpeg PixelSmash Flaw Allows RCE on Video Players, Media Servers, NAS Appliances (1 minute read)

FFmpeg's libavcodec contains a heap out-of-bounds write in the MagicYUV decoder (CVE-2026-8461, CVSS 8.8), dubbed PixelSmash by JFrog, exploitable for unauthenticated RCE via a crafted 50KB AVI, MKV, or MOV file with no user privileges required. Because FFmpeg backs most media pipelines, the bug triggers when video players, file-manager thumbnailers, upload-processing servers, NAS appliances, or smart TVs decode the file, enabling zero-click attacks through torrent clients that auto-scan media folders.

Eight-Year-Old Samsung KNOX Flaw Exposed Millions of Galaxy Devices to Kernel Attacks (1 minute read)

Samsung patched CVE-2026-20971 (CVSS 7.8), an eight-year-old use-after-free in KNOX affecting nearly every Galaxy device from the S9 to S25, including A-series and both Exynos and Qualcomm variants. LucidBit Labs triggered the race condition from an untrusted app by loading a non-ELF file to bypass KCFI protections, achieving controlled kernel memory corruption; the fix shipped in Samsung's January 2026 update.

Nearly half of LG smart TV apps contain residential proxy SDKs (1 minute read)

Researchers at Spur scanned 6,038 LG and Samsung smart TV apps and found 2,058 of them, nearly half on LG, bundling residential proxy SDKs from vendors like Bright Data that route strangers' internet traffic through the TV's home connection. The SDKs ask for consent once during setup and keep proxying after the app is closed, often with the proxy company itself acting as publisher, making these devices long-lived, unaudited exit nodes on home networks.

Exposed data includes names, phone numbers, email and physical addresses, and CRM records, but products, infrastructure, and customer vaults were unaffected; LastPass has rotated the exposed tokens and warns of phishing from attacker-controlled domains.

A new unpatchable flaw in Apple chips opens the door to an iPhone jailbreak (1 minute read)

Paradigm Shift, a Barcelona offensive-security firm, published "usbliter8," a Boot ROM flaw with a proof-of-concept affecting Apple's A12 and A13 chips in the iPhone XS, XR, and 11. Because the bug sits in immutable, burned-in code it cannot be patched, but exploitation requires physical access and must be chained with further bugs to achieve a full jailbreak or forensic unlock.

Microsoft fixes AutoGen Studio flaw that enabled code execution (1 minute read)

What the Fortibleed campaign means for organizations running FortiGate firewalls (1 minute read)

The FortiBleed campaign compromised thousands of FortiGate firewalls and SSL VPN gateways by logging in with leaked or brute-forced credentials, then intercepting live authentication traffic to harvest credentials across 24 protocols and crack hashes on rented Vast.ai GPUs, in some cases reaching full domain control. Defenders should audit FortiGate admin accounts for attacker-planted names like forticloud-sync and support_fortinet, rotate exposed credentials, and check the SOCRadar and CloudSEK datasets for exposure.

Prompt Injection as Role Confusion (1 minute read)

This writeup argues prompt injection is fundamentally a role-confusion problem: an LLM receives system prompts, user messages, tool outputs, and its own reasoning as one undifferentiated token stream, and the role tags meant to partition that stream are a weak signal the model can be made to misread. The authors use this framing to construct new attacks, explain mechanistic-interpretability findings, and predict when injections succeed, arguing that hardening role separation matters more than patching individual payloads.

GitHub Updates actions/checkout to Block Common Pwn Request Attack Patterns (1 minute read)

GitHub updated actions/checkout to refuse fork pull request code by default in pull_request_target and workflow_run workflows, blocking the most common pwn request pattern that lets attackers steal the privileged GITHUB_TOKEN and repository secrets. The default-deny behavior shipped in v7 on June 18 and backports to all supported major versions on July 16. Workflows that genuinely need fork checkout must explicitly set allow-unsafe-pr-checkout to true, so audit any pipeline relying on that combination before the backport breaks it.

Linux and Secure Boot certificate expiration (2025) (1 minute read)

The Microsoft 2011 key used to sign the shim first-stage UEFI bootloader expires in September, after which Microsoft will stop signing shim with it, breaking Secure Boot installs of new Linux distributions on affected systems. A replacement key has existed since 2023 but may not be present in firmware and can require a vendor firmware update that may never ship. Defenders running Secure Boot fleets should audit firmware key databases now and push fwupd/LVFS updates before the cutover.
