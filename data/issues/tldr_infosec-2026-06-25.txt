Zero-Day Exploitation of Vulnerability (CVE-2026-20245) in Cisco Catalyst SD-WAN Manager (2 minute read)

Mandiant disclosed active zero-day exploitation of CVE-2026-20245, a privilege-escalation flaw in Cisco Catalyst SD-WAN Manager that lets an attacker with a compromised admin account reach root via a malicious CSV upload that bypasses file-upload filtering. LLM security advice looks solid until you check the hard cases (1 minute read)

HelpBench, built by University College London and Google, tested 18 leading LLMs against 450 real privacy and security help questions sourced from Reddit and scored by expert rubrics. Advice averaged 82%, strongest on scams and account compromise, but roughly one in ten answers fell below 65%, with the worst failures in abuse and stalking scenarios. Models advised removing phone spyware without warning that abrupt removal can tip off an abuser, and praised an encrypted-vault deletion that left the original file recoverable. Watch what your users are being told.

Decoupling Reconnaissance and Exploitation: Measuring the Capability Boundaries of LLM-Based Web Penetration Testing (1 minute read)

This paper proposes a two-stage evaluation framework that separates exploitation from reconnaissance to measure where LLM-based penetration-testing agents actually fail, using ground-truth injection across 70 web vulnerability testbeds. Given accurate vulnerability context, five open-source agents hit a 90% functional success rate on exploitation, but autonomous reconnaissance plateaus near 50% recall, mostly from failing to parse unstructured telemetry. Architecture matters: multi-agent designs handle long-chain deserialization better, while monolithic and graph-driven agents win on short-chain injections and access-control flaws.

How Reliable Is Your Jailbreak Judge? Calibration and Adversarial Robustness of Automated ASR Scoring (1 minute read)

This paper audits the automated judges that assign attack-success rates in LLM jailbreak research, testing two families against 596 human-labeled HarmBench completions. Dedicated classifiers over-flag (recall 0.974) but resist surface attacks, while LLM-as-judges hold precision yet show erratic recall (0.06 to 0.65); benign framing wrappers flip every LLM-judge 57% to 100% of the time without altering the harmful content. The authors recommend reporting judge precision and recall on a human-labeled slice and adding an adversarial check.

Helpful or Harmful? Evaluating LLM-Assisted Vulnerability Patching via a Human Study (1 minute read)

This paper proposes a controlled human study, using a balanced crossover design, to test whether LLM-assisted vulnerability patching actually outperforms manual debugging or just produces superficial fixes. The key methodological detail is hidden "Ghost Tests" that validate patch security beyond the visible functional checks, catching repairs that pass functionality but fail security validation. Only a pilot has run so far, so treat the full results as forthcoming rather than established.

Scoring AI hackers when there is no answer key (1 minute read)

FrontierCyber, a new benchmark from AI security lab Irregular, drops models onto real systems with live defenses and no planted bugs or hints, then scores how far they get toward a security goal. The targets are everyday phones, hosted services, databases, and networks, each keeping its real sandboxing, authentication, and network boundaries intact. Because difficulty is unknown up front, Irregular predicts each challenge's band before runs and wires systems with canary strings and planted markers to credit partial progress.

Representation Matters: An Empirical Study of Program Representations for LLM Vulnerability Reasoning (1 minute read)

This paper introduces RepBench, a benchmark comparing how program representations affect LLM vulnerability detection across 107 Joern-derived C/C++ testcases spanning five CWE categories. The AST+PDG variant hit 83.2% accuracy versus 53.5% for raw source code, and graph-only prompts beat source-plus-graph while using far less context. The takeaway for AppSec teams building LLM detection tooling: feed models static-analysis graphs, not raw source, since dumping in full source dilutes the vulnerability-relevant evidence and degrades reasoning.

SoK: AI Secure Code Generation: Progress, Pitfalls, and Paths Forward (1 minute read)

This SoK evaluates AI secure code generation with a three-level framework measuring models' natural-language understanding of secure-coding principles, their code-level actuation during generation, and the gap between the two. Principle understanding strongly predicts security, functional correctness, and joint outcomes, but a large knowledge-actuation gap persists: models recognize the right security principles yet fail to implement them in working code. Teams relying on coding agents should keep security review in the loop rather than trusting that the model knows better.

Security and Privacy in Retrieval-Augmented Generation: Architectures, Threats, Defenses, and Future Directions for Building Trustworthy Systems (1 minute read)

This survey maps the security and privacy attack surface of Retrieval-Augmented Generation across centralized, on-device (Micro-RAG), federated, and hybrid deployments. The authors build a taxonomy spanning the retrieval, context-construction, and generation stages, cataloging attack classes including membership inference, index inference, knowledge-base poisoning, gradient leakage, and collusion, then review architectural, algorithmic, and cryptographic defenses against each. Useful for teams threat-modeling RAG pipelines beyond the usual prompt-injection focus.

Leaking Circuit Secrets: Gradient Leakage Attacks on Graph Neural Networks (1 minute read)

This paper presents the first systematic evaluation of gradient leakage attacks against graph neural networks used in circuit design and hardware-security tasks. Testing GraphSAGE, GCN, GIN, and GAT on ISCAS'85, EPFL, and TrustHub netlists, the authors show GLAs can recover gate types and hardware-Trojan properties, aiding adversaries in defeating logic locking or evading Trojan detection. Attention-based GAT leaks the most while injective GIN resists best, and standard defenses like differential privacy and gradient clipping only help in narrow settings and degrade accuracy.

Automated Detection of Configuration-Specific Security Vulnerabilities via Patch Analysis (1 minute read)

PatchLens is a static analysis technique that recovers Vulnerability Impact Conditions, Boolean predicates over compile-time options identifying which configuration variants of C/C++ software contained a given flaw, without compiling any variant. Evaluated across 1,192 Linux kernel, 289 FFmpeg, and 100 PHP patches, it produces compact predicates and shows that CVE texts encode the required options only about 1% of the time, enabling variant-aware CI triage, targeted fuzzing, and feature risk scoring.

BlowLive: Blow-Based Multi-Factor Biometrics with Liveness Detection and Revocability (1 minute read)

This paper introduces BlowLive, a multi-factor biometric framework that fuses blow-acoustic signals with facial recognition and derives stable cryptographic keys from noisy inputs via a fuzzy extractor, hitting 99.56% accuracy on blow-acoustics and 100% on fusion across 50 participants.
