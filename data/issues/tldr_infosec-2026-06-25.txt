Interesting Paper Exploring Prompt Injection (1 minute read)

A new paper, "Prompt Injection as Role Confusion," argues that LLMs don't actually represent role tags as security boundaries but instead learn the textual style of each role/instruction block, which is why prompt injection succeeds. The authors conclude that without genuine role perception, injection defense stays whack-a-mole, and continuous role boundaries enable attacks that subtly shift model state through innocuous-looking text at scale.

New Gaslight macOS Malware Uses Prompt Injection to Disrupt AI-Assisted Analysis (1 minute read)

SentinelOne disclosed Gaslight, a Rust-based macOS implant and infostealer attributed with high confidence to North Korea-aligned actors, which embeds a prompt injection payload of fabricated system-failure messages to make LLM-assisted triage agents abort their own analysis. It persists via a LaunchAgent masquerading as "com.apple.system.services.activity," runs a Telegram bot C2 with a six-command shell, and harvests keychain, browser, and Terminal data into an exfiltrated ZIP.

New MCP specification kills old risks but opens fresh attack surfaces, Akamai finds (1 minute read)

Akamai's analysis of the MCP 2026-07-28 specification finds the overhaul removes protocol-level risks but shifts security burden onto developers. The rebuild kills protocol-managed sessions (eliminating Mcp-Session-Id hijacking), restricts server-initiated prompts, and mandates OAuth 2.1 with PKCE. The tradeoff: the new stateless model hands clients tracking IDs and state objects, so servers using predictable identifiers or skipping integrity validation expose workflow-hijacking and cross-tenant data access. Audit implementations before the July 28 release; the deprecation window for legacy functionality runs 12 months.
