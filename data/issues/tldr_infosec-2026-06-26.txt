A single config file in a cloned repository could steal your AWS credentials through Amazon Q Developer (1 minute read)

A high-severity flaw in Amazon Q Developer (CVE-2026-12957) let a malicious repo auto-register an attacker-controlled MCP server on clone, with no consent prompt, inheriting the developer's full AWS credentials, IAM role, and IDE environment variables. Wiz Research reported it on April 20 and Amazon patched on May 12 by requiring explicit MCP approval, so update Language Servers for AWS and the VS Code, JetBrains, Eclipse, and Visual Studio plugins.

New Linux pedit COW Exploit Enables Root Access by Poisoning Cached Binaries (1 minute read)

CVE-2026-46331, a local-privilege-escalation flaw nicknamed "pedit COW" (out-of-bounds write in the Linux kernel's act_pedit traffic-control action, rated important by Red Hat), poisons the cached page of setuid /bin/su in memory to spawn a root shell while on-disk file-integrity checks stay clean. A working exploit appeared within a day of the June 16 assignment and lands root on default RHEL 10 and Debian 13, which leave unprivileged user namespaces open.

New DirtyClone Linux Kernel Flaw Lets Local Users Gain Root via Cloned Packets (1 minute read)

DirtyClone (CVE-2026-43503, CVSS 8.8) is a Linux kernel privilege escalation that lets a local user with CAP_NET_ADMIN corrupt file-backed page-cache memory through a cloned IPsec packet and overwrite an in-memory binary like /usr/bin/su to gain root. The patch landed in mainline on May 21; multi-tenant servers, CI runners, and Kubernetes hosts on Debian and Fedora, where unprivileged user namespaces grant that capability by default, should patch now.

Securing CI/CD for an open source project, part 3: Credentials, verification, and what’s next (1 minute read)

Cilium's maintainers detailed how their CI/CD pipeline isolates credentials so a compromised workflow can't reach production. CI registry credentials can only push to development image tags (quay.io/cilium/*-ci), while production push credentials sit behind a GitHub protected environment that requires explicit maintainer approval and never lands on the runner. Every release image is signed with Sigstore Cosign using keyless OIDC, eliminating long-lived signing keys, and checkout steps set persist-credentials: false so the GITHUB_TOKEN never leaks into git config.

Proof’s x401 establishes an open protocol for AI agent identity and authorization (1 minute read)

Proof launched x401, an open, issuer-neutral protocol that lets any website or API request and verify the identity and authorization behind an AI agent before it acts. Services define which claims, issuers, and assurance levels they accept; the agent presents a compatible credential, and the service verifies issuer, claim, scope, and action. Proof plans to submit x401 to the FIDO Alliance's agentic authentication workgroup, with the spec published at x401.id.
