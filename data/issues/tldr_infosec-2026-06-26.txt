A Deterministic Control Plane for LLM Coding Agents (1 minute read)

A study of 10,008 public GitHub repositories (6,145 agent config files) finds that LLM coding-agent configurations behave as an unmanaged supply chain: 10.1% of tracked paths are SHA-256 exact duplicates across independent repos, 75.5% of clone pairs cross org boundaries, and under 1% declare permission boundaries. The authors propose Rel(AI)Build, a deterministic control plane that content-addresses agent definitions, enforces tiered permissions and blocklists before LLM invocation, and logs a hash-chained audit trail.

Inherited Circuits, Learned Semantics: How Fine-Tuning Creates Evasion Vulnerabilities Invisible to Standard Evaluation (1 minute read)

This paper shows that fine-tuning an LLM for security classification can introduce evasion vulnerabilities that standard held-out evaluation misses. Studying Foundation-Sec-8B-Instruct, the authors find it inherits its classification circuit from base Llama, then specializes it into brittle token-level indicator rules that fail under behavior-preserving PowerShell transformations like alias substitution and case mutation. They propose a pre-deployment linear probe and indicator-token sign test to flag command families where semantics drift after fine-tuning.

Chai: Agentic Discovery of Cryptographic Misuse Vulnerabilities (1 minute read)

Chai is an AI system that discovers and validates cryptographic misuse vulnerabilities by inverting differential testing: instead of auditing one codebase for many flaws, it catalogs flaws at the library level and propagates them across a cryptographic dependency graph. Evaluated on X.509, JWT, and SAML libraries, it surfaced over 100 vulnerabilities, including a previously unknown critical bug in an SSL library powering billions of devices. The approach targets crypto misuse, a class that lacks the runtime instrumentation that makes memory-safety bugs cheap to confirm.

MIRROR: Novelty-Constrained Memory-Guided MCTS Red-Teaming for Agentic RAG (1 minute read)

MIRROR is a red-teaming framework that runs memory-guided Monte Carlo tree search across four attack surfaces of multimodal agentic RAG systems, including text poisoning, image injection, and orchestrator-level tool manipulation. A deterministic Novelty Gate blocks candidates that duplicate retrieved context, addressing the 73-84% template recycling the authors measured in prior text-poisoning tools. MIRROR hits 76% ASR on image poisoning versus 52% for baselines and 97% on orchestrator attacks at half the query cost; the authors released ART-SafeBench with 41,815 records for testing defenses.

Adaptive Evaluation of Out-of-Band Defenses Against Prompt Injection in LLM Agents (1 minute read) [flag]

This paper organizes out-of-band prompt-injection defenses for LLM agents (CaMeL, FIDES, Progent, FORGE) as classical integrity controls (Biba, reference monitors, least privilege), and warns that all are validated only on static benchmarks, the same gap that let adaptive attacks break twelve in-band defenses at over 90% success. An independent reproduction on AgentDojo with a self-hosted Qwen2.5-7B held: Progent cut mean attack success sixfold (25.8% to 4.2%), and a hand-crafted adaptive attack stayed at 2.6%. White-box GCG attacks remain untested.

What happened after 2k people tried to hack my AI assistant (1 minute read)

A developer ran hackmyclaw.com, inviting the public to email Fiu, an OpenClaw AI assistant, and trick it into leaking a secrets.env file. After reaching Hacker News, Fiu absorbed 6,000+ emails from 2,000+ people using future-self impersonation, fake compliance audits, and multilingual prompt injection, yet never leaked credentials or sent an unauthorized reply. A short anti-injection system prompt held, though batch processing initially cross-contaminated context, arguing for fresh per-email isolation when running agents on untrusted input.

Privacy-Aware Infrastructure in the AI-Native Era: An Asset Classification Case Study (1 minute read)

Meta detailed its privacy-aware infrastructure (PAI) approach to asset classification at scale, the "understand" layer that feeds retention, access, and purpose enforcement. The architecture is deliberately LLM-narrow: models interpret only novel or ambiguous assets (a field named "age" could be a person's age or a cache TTL), then distill their reasoning into versioned, human-reviewed deterministic rules that handle routine production decisions. The pattern keeps enforcement low-latency, replayable, and auditable while shrinking the model's production footprint over time.

Helpful or Harmful? Evaluating LLM-Assisted Vulnerability Patching via a Human Study (1 minute read)

This paper proposes a controlled human study, using a balanced crossover design, to test whether LLM-assisted vulnerability patching actually outperforms manual debugging or just produces superficial fixes. The key methodological detail is hidden "Ghost Tests" that validate patch security beyond the visible functional checks, catching repairs that pass functionality but fail security validation. Only a pilot has run so far, so treat the full results as forthcoming rather than established.

DroidBreaker: Practical and Functional Problem-Space Attacks on Machine-Learning Android Malware Detectors (1 minute read)

DroidBreaker is a problem-space attack framework that evades machine-learning Android malware detectors by manipulating only the APK components most influential to the target model, using build-safe, semantics-preserving injection and obfuscation of API calls, modules, permissions, and URLs. It achieves high evasion rates with few queries in both white-box and black-box settings and sharply cuts detections by commercial scanners on VirusTotal. The authors argue prior attacks overstated success by smoke-testing only installation rather than verifying the modified APK still runs its intended behavior.

Autoformalization of Agent Instructions into Policy-as-Code (1 minute read)

This paper presents an autoformalization pipeline that translates agent prompts, MCP tool descriptions, and natural-language policy documents into formally verified Cedar policies using an LLM generator-critic loop. The approach targets the gap between probabilistic guardrails, which offer no formal guarantees, and hand-coded symbolic enforcement, which doesn't scale. On MedAgentBench, the generated policies cover substantially more of the source specification than the prior hand-coded enforcement, pointing to a path for verifiable agent governance in high-stakes domains.

PRISM: PE Relational Inter-Section Matrix. A 2D Section-Aware Dataset for Static PE Malware Detection (1 minute read)

PRISM is an open dataset and feature representation for static Windows PE malware detection that encodes each binary as a 2D matrix of sections in file order, preserving positional and inter-section context that flat vectors like EMBER discard. It spans 83,633 deduplicated samples across 684 families, and a gradient-boosted classifier matches EMBER's binary-detection accuracy at roughly one-sixth the dimensionality. The structural signal is aimed at less-saturated tasks like malware family classification.

A Mechanistic Explanation of Prompt Injection (and why you should study roles) (5 minute read)

A new analysis argues that prompt injection works because LLMs receive the entire conversation as one undifferentiated token stream, and rely on role tags (<system>, <user>, <tool>) rather than any inherent channel separation to tell instructions from data. The authors use this role-based theory to construct novel attacks, explain prior mech-interp findings, and predict when injections succeed. They call for a research subfield focused on the "science of roles," with the full work published at ICML.

VIGIL: Runtime Enforcement of Behavioral Specifications in AI Agent Skills (1 minute read)

VIGIL is a runtime enforcement framework that checks an AI agent's actual execution trace against behavioral policies drawn from skill specifications, operator constraints, and cross-skill global rules. Its policy language compiles into SMT constraints over finite traces, catching violations that depend on event order, argument relationships, or cross-call value flow rather than single-call filters. On real LLM-agent runs across office, operational, and engineering tasks, it flagged policy violations with over 95% recall and a sub-10% false-positive rate.

Automated Detection of Configuration-Specific Security Vulnerabilities via Patch Analysis (1 minute read)

PatchLens is a static analysis technique that recovers Vulnerability Impact Conditions, Boolean predicates over compile-time options identifying which configuration variants of C/C++ software contained a given flaw, without compiling any variant. Evaluated across 1,192 Linux kernel, 289 FFmpeg, and 100 PHP patches, it produces compact predicates and shows that CVE texts encode the required options only about 1% of the time, enabling variant-aware CI triage, targeted fuzzing, and feature risk scoring.

A privacy-first take on local malware analysis (1 minute read)

Burnyard, a research project from Ohio State University, runs dynamic malware analysis entirely on local hardware, executing samples via instruction-level user-space emulation that hooks every syscall and Windows API call to produce a traced classification across 43 malware families.
