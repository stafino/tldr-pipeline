Strategies & Tactics

New Enterprise-Ready MCP Specification Brings New Security Challenges (1 minute read)

MCP's next specification (MCP 2026-07-28, final on July 28 with a 12-month deprecation window) makes the protocol stateless, killing session hijacking and unsolicited server prompts while tightening authentication. Akamai's analysis warns the shift trades those wins for new risks: predictable tracking IDs and state objects enable cross-agent data access and cross-tenant actions, and new MCP HTTP headers open desync attacks and secret leakage if developers map API keys or PII into them.

A Deterministic Control Plane for LLM Coding Agents (1 minute read) [flag]

This paper studies how LLM coding-agent configuration files (rules, agent definitions, IDE markdown) are governed across 10,008 GitHub repos and finds them effectively unmanaged: 10.1% of tracked config paths are exact SHA-256 duplicates across independent repos, 75.5% of clone pairs cross org boundaries, and under 1% declare any permission boundaries. The authors propose Rel(AI)Build, a deterministic control plane that treats agent definitions as a managed supply chain with content addressing, HMAC-stamped lockfiles, hash-chained audit logs, tiered permissions, and attack-derived blocklists enforced before LLM invocation.

PostCSS Adopted Staged Publishing. 685M Weekly Downloads Now Gated. (2 minute read)

PostCSS maintainer Andrey Sitnik adopted npm Staged Publishing across the high-traffic packages he publishes under a single credential, putting 685M weekly downloads behind a human approval gate. Staged Publishing splits build from release: CI stages a version, but a person must promote it to latest, so a stolen CI token alone can't ship malware. The move counters the CI-as-publisher pattern behind the TanStack and Red Hat compromises, where attackers published packages with valid SLSA provenance.

What happened after 2k people tried to hack my AI assistant (1 minute read)

A researcher built hackmyclaw.com, an OpenClaw email assistant told to guard a secrets.env file, and invited Hacker News to break it. Over 2,000 people sent 6,000+ emails using fake urgency, spoofed "OpenClaw Admin" identities, multilingual prompts, and rapid-fire variations, but none exfiltrated the secrets or forced an unauthorized reply. The operational lessons matter more than the win: inbound volume triggered Gmail fraud suspension, ran up $500 in API costs, and batch processing leaked suspicion across emails until each was isolated in a fresh context.

Inherited Circuits, Learned Semantics: How Fine-Tuning Creates Evasion Vulnerabilities Invisible to Standard Evaluation (1 minute read)

Fine-tuning LLMs for security classification can expand their evasion surface even as held-out accuracy improves, this paper shows. Studying Foundation-Sec-8B-Instruct, the authors find it learns brittle token-level indicator rules that miss behavior-preserving PowerShell transformations (alias substitution, command reconstruction, case mutation) its Llama-3.1 base does not. They propose a pre-deployment check: a linear probe and indicator-token sign test flag command families whose canonical indicators shift role after fine-tuning, prioritizing red-team variant generation.

Privacy-Aware Infrastructure in the AI-Native Era: An Asset Classification Case Study (1 minute read)

Meta detailed its privacy-aware infrastructure (PAI) approach to asset classification, the "understand" layer that tells retention, access, and purpose controls what data they are actually governing. The hybrid pattern uses LLMs narrowly to interpret novel or ambiguous assets, then distills their behavior into versioned, human-reviewed deterministic rules that handle routine production enforcement at low latency. Humans adjudicate reference labels and approve rule promotions, keeping the auditable logic, not the model, in the decision path.

Adaptive Evaluation of Out-of-Band Defenses Against Prompt Injection in LLM Agents (1 minute read)

This paper organizes out-of-band prompt-injection defenses (CaMeL, FIDES, Progent, RTBAS, FORGE) as classical integrity protection, reference monitoring, and least privilege, then warns that all are validated only on static benchmarks. The authors reproduced Progent's adaptive-attack analysis on AgentDojo with a self-hosted Qwen2.5-7B, where it cut mean attack success sixfold (25.8% to 4.2%) and held under a hand-crafted adaptive attack. A stronger white-box GCG attack remains untested.

Empirical Software Engineering TerraProbe: A Layered-Oracle Framework for Detecting Deceptive Fixes in LLM-Assisted Terraform (1 minute read)

TerraProbe is a five-layer oracle framework for checking whether LLM-generated Terraform security fixes actually close the underlying vulnerability rather than just clearing the scanner finding. Across 288 repairs from Gemini 2.5 Flash Lite, GPT-4o, and Claude 3.5 Sonnet, targeted Checkov removal hit 83.3% but full-scanner cleanliness fell to 10.4%, and human review found 71.4% of plan-compared fixes were deceptive. Don't treat a vanished static-analysis finding as remediation; validate plan behavior and security intent.

DKVE: Decentralized Key Validation for End-to-End Encrypted Messaging (1 minute read)

DKVE is a proposed protocol that validates contacts' public keys in end-to-end encrypted messaging by privacy-preservingly cross-checking them against mutual contacts in the user's social graph, using OPRF and OKVS to hide both queries and contact lists. In simulations on real social networks, it detected MitM key substitution with over 97% accuracy on strong-to-moderate-tie graphs and cut key-transparency queries by two orders of magnitude. It complements rather than replaces OOB and KT verification, struggling with bootstrapping and weak-tie networks.

VIGIL: Runtime Enforcement of Behavioral Specifications in AI Agent Skills (1 minute read)

VIGIL is a runtime enforcement framework that checks an AI agent's actual execution trace against behavioral policies drawn from skill specifications, operator constraints, and cross-skill global rules. Its policy language compiles into SMT constraints over finite traces, catching violations that depend on event order, argument relationships, or cross-call value flow rather than single-call filters. On real LLM-agent runs across office, operational, and engineering tasks, it flagged policy violations with over 95% recall and a false-positive rate below 10%.

TESLA-for-5G: Broadcast Authentication for 5G Networks Using TESLA (1 minute read)

This paper proposes TESLA-for-5G (TF5), a broadcast authentication protocol that lets user equipment verify 5G SIB1 messages with symmetric MACs and delayed key disclosure instead of per-message digital signatures, defending against fake base stations that broadcast forged system information during cell selection. Trust is bootstrapped at cell entry with a lightweight GG09 identity-based signature, avoiding certificate distribution, and the design is formally verified in Tamarin under a Dolev-Yao adversary and implemented on OpenAirInterface.

Autoformalization of Agent Instructions into Policy-as-Code (1 minute read)

This paper presents an autoformalization pipeline that translates agent prompts, MCP tool descriptions, and natural-language policy documents into formally verified Cedar policies using an LLM generator-critic loop. Unlike probabilistic guardrails or hand-coded symbolic enforcement, the approach yields formal guarantees while scaling to real policy breadth. On the MedAgentBench benchmark, the autoformalized policies covered substantially more of the source specification than prior hand-coded enforcement.

Agents That Know Too Much: A Data-Centric Survey of Privacy in LLM Agents (1 minute read)

This survey maps LLM agent privacy risks by the data an agent touches (databases, retrieved documents, persistent memory, inter-agent messages) rather than by attack type, covering leakage through queries, intermediate results, and delegated actions across RAG, text-to-SQL, and agent memory. Two gaps recur: only information-flow control addresses compositional and cross-session inference leakage, the least-protected risks, and no benchmark exercises an agent across all its data surfaces under a single privacy policy.

Mostly Automatic Translation of Language Interpreters from C to Safe Rust (1 minute read)

Reboot, a mostly-automatic technique, translated six real-world C interpreters (6k–23k lines) to safe Rust with only 1 to 11 human interventions each, passing 100% of provided test suites and 62%–92% of held-out validation tests. A security case study on mujs showed that heap buffer overflows and use-after-free present in the C original were eliminated in the safe Rust output, making interpreters that handle untrusted input a practical target for memory-safety migration.

Tilikum: Transaction Fair Ordering on a DAG without Weak Edges (1 minute read)

Tilikum is a DAG-based ledger protocol that enforces fair transaction ordering without weak edges, closing the reordering gap that has left DAG consensus protocols exposed to Blockchain Extractable Value (BEV) extraction even as linear chains like Ethereum gained fair-ordering defenses. It uses median-based timestamp aggregation to achieve ordering linearizability, and the Rust implementation reports up to 39x higher throughput than fair-ordering baselines (Narwhal/Tusk, Pompē, Themis, FairDAG) while blocking known DAG-specific reordering attacks.

Design and Performance Evaluation of Secure RF and WiFi-Based Communication in Drone Swarms via Testbed Implementation (1 minute read)

Researchers integrated five encryption schemes (MAVShield, AES-CTR, Speck-CTR, ChaCha20, Rabbit) into a four-UAV swarm testbed to secure MAVLink telemetry, which provides integrity and authentication but ships without encryption, leaving traffic open to eavesdropping. In flight experiments measuring CPU, memory, and packet delivery ratio, the authors' MAVShield framework matched unencrypted performance while resisting key-recovery attacks under algebraic cryptanalysis and Wireshark traffic analysis. Teams running UAV swarms should treat unencrypted MAVLink links as a confidentiality gap.

MIRROR: Novelty-Constrained Memory-Guided MCTS Red-Teaming for Agentic RAG (1 minute read)

MIRROR is a unified red-teaming framework for multimodal agentic RAG that runs memory-guided Monte Carlo tree search with a deterministic Novelty Gate, blocking candidates that copy retrieved context so generated attacks stay novel rather than recycling templates. It reports 76% ASR on image poisoning versus 52% for baselines and 97% on orchestrator attacks at half the query cost, and ships ART-SafeBench with 41,991+ records across four attack surfaces.

PRISM: PE Relational Inter-Section Matrix. A 2D Section-Aware Dataset for Static PE Malware Detection (1 minute read)

PRISM is an open dataset and feature representation for static Windows PE malware detection that encodes each binary as a 2D section-aware matrix, preserving section ordering and inter-section context that flat vectors like EMBER, BODMAS, and SOREL-20M discard. It spans 83,633 deduplicated matrices across 684 malware families, matches EMBER's binary-detection accuracy at roughly one-sixth the dimensionality, and ships with the extraction library, trained models, and analysis pipeline under CC BY-NC-SA and MIT.

zQR: A Verifiable QR-Driven zkSNARK Proof Verification Framework for Mobile Platforms (1 minute read)

zQR is a proof-of-concept framework that encodes zkSNARK proofs in QR codes so offline mobile verifiers can check zero-knowledge proofs without a network connection, using blockchain for auditability and LLMs for automatic circuit generation. The authors benchmark proof generation and verification latency, QR encoding and decoding times, and blockchain gas costs, fitting proofs into a version-19 QR symbol at low error correction.
