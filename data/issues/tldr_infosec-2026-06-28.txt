GLM 5.2 beats Claude in our benchmarks (1 minute read)

Semgrep benchmarked open-weight models against its IDOR detection dataset and found Zhipu AI's GLM 5.2 scored 39% F1, beating Claude Code (32%) and Claude Opus 4.8 at roughly $0.17 per vulnerability found. Both still trailed Semgrep's purpose-built static-analysis harness (53-61% F1), which handles endpoint enumeration and context selection rather than relying on the model alone. The takeaway for teams deploying AI security agents: most detection performance comes from the scaffolding around the model, not the model itself.

New Enterprise-Ready MCP Specification Brings New Security Challenges (1 minute read)

MCP's next specification (MCP 2026-07-28, final on July 28 with a 12-month deprecation window) makes the protocol stateless, killing session hijacking and unsolicited server prompts while tightening authentication. Akamai's analysis warns the shift trades those wins for new risks: predictable tracking IDs and state objects enable cross-agent data access and cross-tenant actions, and new MCP HTTP headers open desync attacks and secret leakage if developers map API keys or PII into them.

What happened after 2k people tried to hack my AI assistant (1 minute read)

A researcher built hackmyclaw.com, an OpenClaw email assistant told to guard a secrets.env file, and invited Hacker News to break it. Over 2,000 people sent 6,000+ emails using fake urgency, spoofed "OpenClaw Admin" identities, multilingual prompts, and rapid-fire variations, but none exfiltrated the secrets or forced an unauthorized reply. The operational lessons matter more than the win: inbound volume triggered Gmail fraud suspension, ran up $500 in API costs, and batch processing leaked suspicion across emails until each was isolated in a fresh context.

Choosing a Public DNS Resolver (1 minute read)

A new guide ranks 29 public DNS resolvers against the criteria operators actually weigh: encrypted transport, DNSSEC validation, IPv6, jurisdiction, and operator type. Peer-reviewed measurements anchor the trade-offs: DoH and DoT add minor overhead while cutting in-transit tampering, but roughly 25% of DoT providers served invalid TLS certs, so favor validating, no-log operators and treat DNSSEC as mandatory.

A Mechanistic Explanation of Prompt Injection (and why you should study roles) (5 minute read)

A research writeup argues that prompt injection is fundamentally a problem of how LLMs perceive roles, since models receive system prompts, user messages, tool outputs, and their own reasoning as one undifferentiated token stream partitioned only by chat-template tags like <system> and <user>. The authors use this theory to construct new attacks, explain anomalous mech-interp results, and predict when injections succeed, and call for a research subfield focused on the "science of roles." Useful framing for defenders weighing why role-based input separation remains brittle.

Your First GRC Agent: A Red Teamer's Walkthrough (1 minute read)

A red teamer argues that agentic AI suits GRC because the work is high-volume, repeatable, and run against known baselines, making it ideal for autonomy, real-time context, and multi-step execution rather than the point-in-time, scheduled artifacts legacy programs produce. The author stresses that judgment stays deterministic and human-owned: the model handles reasoning, summarization, and orchestration while controls, thresholds, and policy decisions come from analysts. The walkthrough builds a single agent that traces control drift and identifies evidence gaps.
