Factoring RSA Keys with Many Zeros (1 minute read)

Researchers analyzing badkeys data found a class of weak RSA keys with regularly spaced blocks of all zeros, including certificates issued to Yahoo, Verizon, and NetApp devices, plus SSH hosts running EnterpriseDT's CompleteFTP (RSA keys from v10.0.0-12.0.0, DSA keys from v10.0.0-23.0.4). The sparse moduli are factorable, and the recurrence of the same failure across independent implementations suggests more vulnerable products remain undiscovered and worth scanning for.

Mozilla warns of indirect prompt injection risk in AI coding agents (1 minute read) [flag]

Mozilla's 0DIN team demonstrated an indirect prompt injection attack in which a malicious GitHub repo steers AI coding agents like Claude Code into running setup commands that fetch a reverse-shell payload at runtime via a DNS TXT record, leaving nothing malicious in the repo for code review or static analysis to catch. Treat setup instructions in unfamiliar repos as untrusted code regardless of agent recommendations, and design agents to surface what a command actually executes rather than its literal string.

Critical SimpleHelp flaw exploited to deploy new stealer malware (1 minute read)

Attackers are exploiting CVE-2026-48558, a critical authentication-bypass in SimpleHelp's OIDC configuration that lets unauthenticated users create privileged technician accounts, to push the TaskWeaver loader and the previously undocumented Djinn Stealer across Windows, macOS, and Linux. Roughly 1,000 internet-exposed SimpleHelp servers ran the vulnerable config at disclosure, so MSPs and IT teams running RMM on the edge should patch immediately and rotate developer, cloud, Git, SSH, and infrastructure credentials.

Researchers Demo New Claude Code Attack Using Harmless-Looking Repositories to Hijack Developer Machines (1 minute read) [flag]

Mozilla's 0Din researchers demonstrated an indirect prompt-injection attack against Claude Code in which a benign-looking repository's setup notes trigger an install error whose message instructs the agent to run a recovery command that executes setup.sh, which fetches a base64-encoded payload from a DNS TXT record and spawns a reverse shell. The payload never lives in the repo and leaves no plaintext reverse-shell signature, so static analysis, network monitoring, and the agent each see only benign activity while attackers exfiltrate credentials and deploy persistent backdoors.

The attack that hijacked Claude Code came through Sentry. Datadog, PagerDuty, and Jira have the same exposure. (3 minute read) [flag]

Tenet Security's "agentjacking" disclosure shows a single crafted Sentry error event, sent via a public DSN that needs no authentication, can inject attacker instructions that Claude Code, Cursor, and Codex execute with developer privileges, succeeding in 85% of 100-plus controlled tests against the 2,388 orgs with exposed DSNs. The same exposure hits any agent pulling from Datadog, PagerDuty, or Jira, so the fix is restricting what shell-capable agents do with returned data, not revoking the intentionally public DSN.

‘DirtyClone’ Linux Kernel Vulnerability Leads to Root Access (1 minute read)

JFrog published technical details and a PoC for DirtyClone (CVE-2026-43503, CVSS 8.8), a Linux kernel local privilege escalation flaw in the networking stack's socket-buffer handling that lets any local user gain root. The bug is a variant of the DirtyFrag/Fragnesia family affecting Debian, Fedora, and Ubuntu with unprivileged user namespaces enabled; only kernels updated to v7.1-rc5 with the complete fix chain are protected, so partially patched LTS branches remain exposed.

Supreme Court delivers ‘major win’ for tech privacy in Chatrie ruling (1 minute read)

The Supreme Court ruled 6-3 in Chatrie v. United States that collecting phone location data from a geographic area constitutes a Fourth Amendment search, extending the Carpenter precedent to geofence warrants. The decision means law enforcement requests for bulk location data from providers like Google now face Fourth Amendment scrutiny, though the court declined to rule on whether Chatrie's specific warrant was valid.

SCOTUS limits the law enforcement use of "geofence" warrants, saying people have "a reasonable expectation of privacy" in their cell-phone location data (TechCrunch) (1 minute read)

The Supreme Court ruled that geofence warrants, which compel providers to identify every device in a geographic area during a time window, violate the Fourth Amendment because users hold a reasonable expectation of privacy in their cell-phone location data. The decision curtails a technique law enforcement has used to canvass suspects from location histories, and agencies relying on bulk geolocation requests will need narrower, particularized warrants going forward.

In major privacy win, Supreme Court rules geofence warrants are protected by privacy rights (1 minute read)

The Supreme Court ruled 6-3 that geofence warrants fall under Fourth Amendment protection, holding that individuals have a reasonable expectation of privacy in their cell-phone location data and that police must obtain a search warrant to compel companies like Google to hand it over. The court declined to ban geofence warrants outright, instead requiring law enforcement to narrow these "reverse" search requests rather than sweeping up the location records of every nearby user.
