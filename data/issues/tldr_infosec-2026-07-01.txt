Securing the AI Agent: A Unified Framework for Multi-Layer Agent Red Teaming (1 minute read)

AI-Infra-Guard is an open-source framework that red teams AI agents by matching a detection paradigm to each layer of the attack surface: infrastructure, protocol/tool, agent behavior, and model. It spans deterministic rule matching over 75+ components and 1,400+ vulnerability rules, LLM-driven auditing of MCP servers and agent-skill packages, multi-turn black-box red teaming, and a jailbreak harness with 26+ attack operators. An Empirical Study of Security Calibration in Large Language Models for Code (1 minute read)

A new empirical study evaluated security calibration in GPT-4o-mini, Gemini-2.0-Flash, and Qwen3-Coder-Next, testing whether models can tell when the code they generate is insecure. The authors found overconfidence is pervasive, calibration-guided automated repair yields only limited fixes while frequently introducing functional regressions, and architectural gating that helps on controlled benchmarks degrades in realistic repository-level settings. For teams shipping LLM-generated code, model confidence is not a reliable signal of security, so keep independent SAST and review in the loop.

Inside the Advisory Database and what happens when vulnerability volume breaks records (1 minute read)

Private vulnerability reports jumped from ~550/week in January to over 3,000/week, and GitHub CNA CVE requests hit nearly 4,000 in May, roughly 10x year-over-year. AI-Generated PowerShell Malware: An Experimental Framework and Dataset (1 minute read)

This paper proposes an experimental framework and sandbox for dynamic analysis of LLM-generated PowerShell malware, and releases a manually curated dataset of real-world PowerShell samples annotated in natural language. Testing permissive open-weight models adapted to malware generation, the authors found AI-generated scripts closely mirror real malware in triggered OS-level malicious events, with a median Jaccard index of 84.5% and complete overlap in 48.4% of cases. Behavioral detections keyed to OS events, rather than static signatures, will hold up better against this generation.

Security--Fidelity Tradeoffs: The Hidden Cost of Prompt Injection Defense (1 minute read)

This paper introduces SecFid, a benchmark showing that indirect prompt injection defenses buy security by suppressing untrusted text, which corrupts tasks like translation and document editing that must preserve it. Across 1,168 examples and 48 configurations, no model or defense wins both: the most secure reach 99.3% security at 71-74% fidelity, while the highest-fidelity model hits 96.5% fidelity at 47.8% security. The right tradeoff depends on deployment, so reporting security without fidelity hides its cost.

A Lifecycle and Application-Stack Survey of Large Language Model Vulnerabilities: Attacks, Risks, Defenses, and Open Problems (1 minute read)

This paper systematizes LLM system vulnerabilities across eight lifecycle stages, from data collection and pretraining through retrieval, tool/agent execution, and deployment, arguing that risk now lives in the full application stack rather than the model weights. The authors map attacks to confidentiality, integrity, availability, safety, and agency-control objectives, emphasizing where trust boundaries fail, how untrusted data becomes executable instruction, and why point defenses rarely compose. They close with a research agenda covering provenance-aware retrieval, tool-call containment, and long-horizon agent evaluation.

Understanding and Evaluating Claw-like Agent Security Through a Computer-Systems Lens (1 minute read)

This paper introduces SafeClawArena, a benchmark of 406 adversarial tasks that tests always-on "Claw-like" agents (OpenClaw, NemoClaw, SeClaw) across skill supply-chain integrity, persistent state exploitation, cross-boundary data flow, and indirect prompt injection. Attack success rates hit 70%, and malicious plugins succeeded 100% of the time regardless of the underlying LLM. SeClaw cut GPT-5.4's rate to 22%, partly via utility tradeoffs, while Claude-Opus-4.6 already sat near that floor everywhere.

Exploring Side-Channel Protections in Hardware Implementations of PQC ML-KEM Verification (1 minute read)

Researchers evaluated side-channel protections for ML-KEM's Fujisaki-Okamoto decapsulation verification across microcontroller and FPGA implementations, testing unprotected, first-order hash-based, and higher-order masked designs. FPGAs leaked more than microcontrollers despite their speed advantage, and the parallelized processing of even higher-order masked designs produced enough first-order leakage for full secret-key recovery. Teams deploying PQC in hardware should not assume masking alone hardens FPGA implementations against power and EM analysis, especially in high-bandwidth configurations.

CVE-TTP KG: Knowledge Graph Linking Software Vulnerabilities to Attack Behaviors (1 minute read)

This paper introduces CVE-TTP KG, a knowledge graph that links CVEs to MITRE ATT&CK tactics and techniques, filling the gap where CVE and NVD entries describe vulnerabilities but not the attacker behaviors that exploit them. A fine-tuned CySecBERT model reaches macro F1 of 87.71% on technique classification and 96.16% on tactics, with the extracted entities and relations loaded into a Neo4j graph for structured threat analysis.

The Decomposition Is the Fingerprint: Per-Component Identity for Agent Skills (1 minute read)

This paper introduces a locality-sensitive fingerprint for agent skills, storing each skill as a per-component triple (prompt, code, tools) in a 120-byte signature compared by Hamming distance. Keeping the components separate lets the fingerprint recover skill-family lineage through paraphrase, renaming, and refactoring, localize which component was reused, and flag tampered copies, hitting 0.974 AUC over 4,950 comparisons. The authors stress it signals identity and lineage for a skill registry, not behavioral safety, and still requires separate behavioral verification.

Mutating the "Immutable": A Large-Scale Study of Git Tag Alterations (1 minute read)

Researchers analyzed 328.4M repositories from Software Heritage and found 10.2M Git tag alterations across 189k repos, disproving the assumption that tags are immutable release anchors. A cross-check against Nixpkgs found 32 packages referencing altered tags, 7 with confirmed build failures, showing that force-pushed tag changes break reproducible builds and supply-chain integrity. Pin dependencies to cryptographic commit hashes rather than tags, and add monitoring for tag mutations as standard practice.

EnclaveX: End-to-End Confidential AI with CPU/GPU TEEs (1 minute read)

This paper presents an end-to-end confidential AI workflow that combines CPU TEEs (Intel TDX, AMD SEV-SNP) with NVIDIA H200 GPU TEEs to keep data and model code confidential from the cloud provider. The authors flag that Kubernetes administrators can still access confidential VM contents, a gap application-level protections must close, and benchmark the performance overhead of running Intel TDX alongside H200 GPUs.
