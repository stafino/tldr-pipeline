Has This Checkpoint Been Abliterated? A Two-Signal Audit and Its Failure Map (1 minute read)

This paper introduces a pre-deployment audit that detects whether an open-weight checkpoint has had its refusal mechanism abliterated, combining an activation refusal-gap signal with a base-to-candidate weight-recovery energy. Across a 273-checkpoint registry spanning Qwen, Llama, and Gemma, the z-sum separates 57 abliterations from 37 benign fine-tunes at AUROC 0.95, transferring to held-out families at 0.89 balanced accuracy. The authors note it is triage, not tamper-proofing: a spoofed reference evades both signals.

Non-interactive SSH attacks dominate after login (1 minute read)

A Czech Technical University study of 11 SSH honeypots in Frankfurt logged 177,622 authenticated sessions over 15 days, and 99.23% were non-interactive: a client logs in, runs a single command, and disconnects in under a second, versus 0.10% interactive shells. Most of that traffic is automated reconnaissance, with the ten most common commands covering 41.59% of sessions. Defenders should tune SSH detection and honeypots around single-command exec requests rather than the interactive shell sessions Cowrie-style fields were built to capture.

How GitHub used secret scanning to reach inbox zero (1 minute read)

GitHub Security cleared 20,000+ secret scanning alerts across 15,000+ repositories in nine months, reaching zero open alerts. The team found that just five repositories accounted for roughly 18,000 alerts, all inactive test fixtures and fake-but-valid credentials, leaving about 2,000 alerts of potential live secrets to triage, assign ownership, and rotate. Their playbook: filter noise first, then build remediation workflows that account for secrets living in support tickets and other non-code locations.

Behind the Refusal: Determining Guardrail Activation via Behavioral Monitoring (1 minute read)

This paper introduces the first black-box guardrail reconnaissance method, detecting whether a target AI system runs a guardrail by monitoring HTTP, lexical, and timing signals with zero prior knowledge. It flags guardrail presence with 100% accuracy, identifies which content categories the guardrail blocks, and distinguishes guardrail blocks from LLM refusals at 98% F1. Defenders should assume attackers can fingerprint guardrails and normalize timing and error responses to reduce this side channel.

ElephantAgent: Contextual State Continuity in Agentic Systems (1 minute read)

This paper introduces ElephantAgent, a protocol that defends agentic systems against tool and memory poisoning by enforcing what the authors call Contextual State Continuity. Before each query, it recomputes a digest of the security-critical contextual state (tool state and memory) and verifies it against an authorized digest held in a linearizable ledger backed by replicated trusted hardware, flagging out-of-band tampering. For in-band semantic abuse, it adds Historical Traceability for post-hoc audit and recovery to a known-good prior state.

Trust Boundary Semantic Gaps: A Multi-dimensional Analysis and Mitigation for Security-by-Design (1 minute read)

This paper introduces the Trust Boundary Semantic Gap (TBSG): an artifact passes correctly implemented syntactic validation (format, protocol, signature) yet still fails to satisfy the receiving domain's semantic security requirements, so a signed update or authenticated token can be accepted and still enable compromise. Analyzing 75 incidents from 2014-2025, the authors organize these gaps along four dimensions (Identity, Spatial, Temporal, Interpretation) and propose TBSAM, a design-time framework that traces gaps to their originating boundary and maps each to architectural controls, demonstrated on a SolarWinds/SUNBURST reconstruction.

Understanding Build Reproducibility in the F-Droid Ecosystem (1 minute read)

Researchers published the first empirical study of build reproducibility decay in F-Droid, rebuilding 18,904 Android app versions that F-Droid had previously confirmed bitwise reproducible between September 2018 and February 2026. Only 83% still rebuilt successfully, with missing dependencies causing 76% of failures; of those that did rebuild, 94% remained bitwise identical. The takeaway for defenders relying on reproducible builds for supply-chain verification: rebuildability decays over time, so archived toolchains and pinned dependencies matter more than a one-time reproducibility check at publish.

SkillFuzz: Fuzzing Skill Composition for Implicit Intents Discovery in Open Skill Marketplaces (1 minute read)

This paper reframes implicit-intent discovery in open LLM skill marketplaces as a fuzzing problem, where individually benign skills combine to redirect an agent toward unintended objectives that isolated audits miss. SkillFuzz extracts structured skill contracts and uses contract-guided Monte Carlo Tree Search to prioritize conflicting compositions without execution, surfacing over 1,000 distinct implicit intents and confirming 80% of its highest-risk flags at execution time. For teams running skill marketplaces, audit compositions, not just individual skills.

kNNGuard: Turning LLM Hidden Activations into a Training-Free Configurable Guardrail (1 minute read)

kNNGuard is a training-free LLM guardrail that classifies unsafe, off-topic, or adversarial prompts from a model's hidden activations, using multi-layer kNN over a bank of just 50 labeled safe and unsafe examples. Across six topical and security domains it matches or beats fine-tuned state-of-the-art classifiers on F1 while running 2.7x faster than the best comparable guardrail and 10x faster than a fine-tuned safety model. Domain adaptation means swapping the labeled bank, buildable in under 10 seconds, with no gradient updates.

Mitigating Package Hallucinations in Large Language Models via Model Editing (1 minute read)

This paper introduces BOUND, a localized model-editing framework that curbs package hallucinations, where LLMs suggest non-existent package names that attackers can pre-register as malicious dependencies (slopsquatting). BOUND locates hallucination-related modules and applies LoRA adapters with a boundary-aware objective, cutting package-level hallucination rates by 79.9% on edit prompts and 65.4% on unseen ones, with smaller gains carrying over to code generation and pip-install recommendation.

Fedora 45 could get Shadow Stack support to prevent ROP attacks (1 minute read)

Fedora 45 may ship with hardware Shadow Stack enabled by default, using Intel CET and AMD Shadow Stack to protect return addresses against return-oriented programming (ROP) attacks. The mitigation requires supporting CPUs and is enforced at the kernel and glibc level, so defenders running Fedora fleets should plan compatibility testing for statically linked or older binaries that may break under enforcement.

HTTP REST API Structure Learning (1 minute read)

This paper introduces HRAL, an unsupervised anomaly-detection approach that learns REST API structure and behavior directly from network traffic, flagging deviations without predefined rules or OpenAPI documentation. It reports 82.07% recall and an 87.24% F1-score, outperforming alternatives when documentation is sparse and reaching 100% detection when paired with signature rules like the OWASP ModSecurity CRS. The approach targets partially documented API environments, where traffic-derived models fill gaps that rule- and schema-based defenses miss.

Knowledge Over Parameters: Evolving Smart Contract Vulnerability Detection (1 minute read)

This paper introduces EvoVuln, a framework that detects smart contract logic bugs by evolving procedural detection rules from as few as five vulnerable and five safe examples per vulnerability type, without any model fine-tuning. An Inversion of Control runtime compiles rules into executable policies that separate deterministic control flow from LLM reasoning, hitting a 71% macro-average F1 across five vulnerability types. The evolved rules transfer across models for a one-time cost under $50, letting a lightweight model beat a larger zero-shot one by 19 points.

Steerability via constraints: a substrate for scalable oversight of coding agents (1 minute read)

This paper argues that the same controls used to manage large human engineering teams (access control, network policies, tooling-enforced conventions) transfer directly to coding agents and cost fewer tokens than agentic scaffolding. In a controlled test, a small reviewer model (Gemma 4 e4b) inspecting a Python codebase with 11 inserted backdoors raised recall from 54.5% unconstrained to 90.9% with a constrained substrate plus a ~200-LoC docs CLI, with the substrate and tooling contributing independently.

AgentFlow: Building Agent Dependency Graphs for Static Analysis of Agent Programs (1 minute read)

AgentFlow is a static analysis framework that recovers agent-specific dependencies (constructors, tool decorators, handoff declarations) that conventional AST-based tools miss, representing them as a typed Agent Dependency Graph. Evaluated on 5,399 real-world agent programs across five frameworks, it generated dependency-aware Agent Bills of Materials and surfaced 238 taint-style prompt-to-tool risks. For teams governing agent software, the ADG gives a practical basis for BOM generation and tracing which prompts can reach which tools.
