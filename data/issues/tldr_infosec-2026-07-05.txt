Sandboxing untrusted agent code with gVisor costs ~200ms per cold start. Blocking syscalls instead of emulating them costs ~8ms (3 minute read)

zviz, an OCI-compatible container runtime written in Zig, isolates untrusted agent code by selectively denying dangerous syscalls with seccomp/Landlock rather than emulating them like gVisor's userspace kernel. The tradeoff is compatibility for speed: gVisor safely emulates dangerous syscalls at roughly 200ms cold starts and 5-250x syscall overhead, while zviz denies them outright with EPERM for ~8ms cold starts and near-native throughput. Ordering matters operationally: capabilities drop and Landlock apply before seccomp loads, so the runtime doesn't block itself while arming.
