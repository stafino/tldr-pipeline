You shouldn't trust Trusted Publishing (5 minute read)

Trusted Publishing isn't a scheme humans are meant to trust, the author argues; it's machine-to-machine authentication built on OIDC federation, so evaluating it as something you personally can or can't trust is a category error. Adopted across PyPI, npm, RubyGems, crates.io, and NuGet, it shrinks the number of long-lived, over-scoped credentials maintainers hold by issuing short-lived, minimally scoped publishing tokens tied to CI/CD identity. It still isn't foolproof, since a credential lives somewhere in the flow.

Microsoft is rewriting Windows patch guidance because of AI (1 minute read)

Microsoft is advising organizations to shorten Windows update deployment timelines, warning that AI is cutting the time attackers need to weaponize vulnerabilities after a patch ships. The recommended settings are aggressive: quality update deferrals under three days, deadlines of zero to one day, and grace periods of no more than two. Defenders can use the new Windows Autopatch report in Intune to surface still-exposed devices, enable Hotpatch for reboot-free fixes, and enforce update compliance through Conditional Access.

Why we cannot wait for better post-quantum signature algorithms (1 minute read)

Cloudflare is moving forward with ML-DSA for its post-quantum signature migration rather than waiting for better schemes, targeting full PQ security by 2029. ML-DSA carries real costs, being much larger on the wire and incompatible with optimizations RSA and ECC allowed, but successors like FN-DSA and NIST's nine on-ramp candidates will not be ready in time. Deploy the algorithm you have now; harvest-now-decrypt-later already threatens authentication, not just encryption.

npm 12 Disables Install Scripts by Default to Reduce Supply Chain Risk (1 minute read)

npm 12 ships with install scripts disabled by default, so dependency lifecycle hooks (preinstall, install, postinstall) and implicit node-gyp builds no longer run unless explicitly allowlisted via npm approve-scripts. Git and remote-URL dependencies are also opt-in now, closing the most common paths for supply-chain compromise. GitHub is also deprecating 2FA-bypass GATs: they lose sensitive account actions in early August and direct publishing in January 2027, pushing automated releases toward trusted publishing (OIDC) or staged human-approved publishes.

Beware What You Autocomplete: Forensic Attribution of Backdoored Code Completions (1 minute read)

This paper introduces CodeTracer, a forensic framework that traces backdoored LLM code completions back to the specific malicious fine-tuning samples that caused them, using only the fine-tuning corpus and the reported miscompletion event. It extracts a behavioral fingerprint from the unsafe output, narrows to semantically relevant training samples, and uses LLM-based reasoning for attribution. Across three vulnerability cases, ten backdoor attacks, and sixteen baselines, the authors report high attribution accuracy, low false-identification rates, and robustness against adaptive attacks.

Skills Registry Threat Models (5 minute read)

Skills registries for AI agents are a strict superset of package-manager clients: installing one skill can trigger install commands across npm, pip, cargo, brew, and more from a manifest the user never read, while the skill body joins the agent's system prompt as a prompt-injection vector and inherits the session's existing bash, file, and network grants. This analysis walks the loader, registry, and runtime, showing that the worst exposures are deliberate design decisions that pass every static scanner rather than flaggable bad lines.

From Legacy Documentation to OSCAL: An MCP-Based Agent Pipeline for Threat-Informed Continuous Compliance in Critical Infrastructure (1 minute read)

This paper introduces an MCP-grounded multi-agent pipeline that turns natural-language system descriptions of OT environments into audit-ready NIST OSCAL artifacts, decoupling LLM reasoning from deterministic retrieval against threat-intel sources to curb hallucinated CVEs and attack paths. On a synthetic water-utility scenario it hit 0.90 CVE recall and perfect D3FEND recall, but errors shifted upstream: a single misextracted asset yields real-but-irrelevant CVEs, making manual review the remaining control point.

Out of Sight: Compression-Aware Content Protection against Agentic Crawlers (1 minute read)

This paper introduces CAPE, a framework that protects textual content from LLM agent crawlers by injecting human-invisible perturbations that survive rendering but corrupt the context-compression step agents use to fit their token budgets. Targeting compression rather than access controls or visible injection, CAPE raised agent information loss by up to 75.8% over the strongest baseline and transferred to LangGraph workflows and GitHub Copilot while leaving content visually identical to the original.

Prismata: Confining Cross-Site Prompt Injection in Web Agents (1 minute read)

Prismata introduces contextual least privilege for autonomous web agents, dynamically deriving per-content trust labels that constrain what an agent can see and do to contain cross-site prompt injection. Its confinement guarantees bound any mislabeling so permissions can only decrease in privilege, and it redacts untrusted content and restricts capabilities without developer annotations. Across recent published web-agent attacks, including adaptive variants, the authors report substantially reduced attack success while preserving benign task utility.

Most data brokers won’t tell you what happened to your deletion request (1 minute read)

A UC Irvine study sent deletion and opt-out requests to every reachable broker on California's public registry in fall 2025, roughly 322 deletion and 360 opt-out requests, and found that about 70% never reported the outcome of a deletion despite the 45-day CCPA deadline. Around 22% demanded identity verification on opt-outs, which the CCPA prohibits, and overall response rates sat between 26% and 38%, meaning the privacy rights consumers use most see the least compliance.

TRACE: A Two-Channel Robust Attribution Watermark via Complementary Embeddings for LLM-Agent Trajectories (1 minute read)

TRACE is a watermarking scheme that lets developers prove attribution of LLM-agent trajectories even when a reseller has full read/write access to the trajectory logs. It embeds two complementary channels: a distortion-free selection channel keyed on local content that resynchronizes after deletions, and a tally channel keyed on the log's structure that survives arbitrary rewriting. On ToolBench and ALFWorld it matched unwatermarked success rates while staying detectable under 70% step deletion, making log corruption the only way to strip attribution.

Mini-Programs, Mega-Problems: Unveiling OAuth-based Authentication Misuses in Mini-Programs via Dynamic Analysis (1 minute read) [flag]

Researchers introduced MINIAUTH, a dynamic-analysis framework that pinpoints and executes OAuth login flows in super-app mini-programs to find runtime authentication misuses that static tools miss. Scanning 44,273 WeChat and 2,721 Baidu mini-programs, it surfaced 1,834 misuse cases, including client-side identity forgery via exposed credentials, auth bypass through static or plaintext identifiers, and a cryptographic flaw in Baidu's OBA APIs that permits brute-forcing session keys. Findings drew CNVD/CNNVD IDs, underscoring the need for platform-level OAuth safeguards over reliance on third-party developers.

TLS certificates for internal services done right (1 minute read)

The author argues that split-horizon DNS is the cleaner way to secure internal-only services, avoiding the operational drag of self-signed certificates. A single domain (e.g. grafana.tuxnet.dev) resolves to a public IP for outside resolvers and an internal IP over the VPN, letting a public CA like Let's Encrypt issue valid certs while a WAF rejects non-VPN traffic. The tradeoff: one WAF rule on the reverse proxy replaces distributing a private CA to every client or telling users to ignore TLS errors.
