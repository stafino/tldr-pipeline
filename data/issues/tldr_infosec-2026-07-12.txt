Researchers disclosed "Ghostcommit," an attack that hides prompt-injection instructions inside a PNG so AI code reviewers, which treat images as opaque binary blobs, wave the malicious pull request through. Forget typosquatting; slopsquatting is the software supply chain threat created by AI coding tools (2 minute read)

Slopsquatting is a supply-chain attack that weaponizes LLM hallucinations: AI coding assistants invent plausible-sounding package names, attackers register those names on public registries and populate them with malware. Because these are novel invented names rather than misspellings of real libraries, registry typosquatting defenses like blocking "crossenv" against "cross-env" don't catch them. Defenders should pin and vet AI-suggested dependencies before install, and validate that every package an assistant recommends actually exists and is legitimately maintained.

The Hotel Upgrade Hack: Mass Assignment Vulnerabilities in APIs (2 minute read)

Mass assignment vulnerabilities let attackers set fields the server never intended to expose by adding extra keys to an API request, binding user input directly to internal models. The canonical case is Egor Homakov's 2012 GitHub exploit, where he injected an organization_id into a profile-update POST to add himself to the rails/rails org without approval. Defenders should enforce explicit allow-lists for bindable fields (Rails strong parameters, DTO schemas, or per-endpoint field whitelists) rather than passing req.body straight into create or update calls.
