Since Chromium 148, Math.tanh is now fingerprintable to link underlying OS (1 minute read)

Since Chrome 148, a single `Math.tanh` call on the right input leaks the host OS, because V8 commit c1486295ae5 swapped its bundled fdlibm port for `std::tanh`, which routes to the platform libm and returns different last-place bits on glibc, Apple's libsystem_m, and Windows UCRT. The same divergence hits CSS trig functions and the Web Audio compressor, giving anti-bot systems a signal that contradicts a spoofed User-Agent whenever the math bits and the claimed OS disagree.

Unauthenticated RCE in Motorola's MR2600 Router (1 minute read)

Researcher MrBruh disclosed an unauthenticated remote code execution flaw in Motorola's MR2600 Wi-Fi 5 router, chaining two firmware-update endpoints that were meant to require authentication but didn't. The upload handler validates a SEAMA image by checking the multipart body's first four bytes for the magic header, so an attacker who sends the raw firmware without multipart boundaries bypasses the check and flashes malicious firmware. The last firmware, v1.0.22, shipped in mid-2024.

Forget typosquatting; slopsquatting is the software supply chain threat created by AI coding tools (2 minute read)

Slopsquatting is a supply-chain attack that weaponizes LLM hallucinations: AI coding assistants invent plausible-sounding package names, attackers register those names on public registries and populate them with malware. Because these are novel invented names rather than misspellings of real libraries, registry typosquatting defenses like blocking "crossenv" against "cross-env" don't catch them. Defenders should pin and vet AI-suggested dependencies before install, and validate that every package an assistant recommends actually exists and is legitimately maintained.

Why SBOMs, signing, and provenance still don’t tell you if software is safe (1 minute read)

SBOMs, signing, and provenance verify what software contains and where it came from, but not what it will do at execution. The author argues this composition-and-authenticity trust model breaks down as AI generates and mutates code faster than human review, citing University of Toronto research on an AI-powered worm that reasons through new attack paths mid-spread. The defensible gap is behavior: teams need runtime capability analysis, not just provenance.

A hardware security AI assistant that checks chips for hidden backdoors (1 minute read)

Researchers at the University of Florida released VeriChat, a conversational assistant that helps hardware security engineers hunt for hidden circuits in third-party IP blocks. It uses a three-agent retrieval pipeline over 28,221 papers to curb hallucinated answers and can run syntax and synthesis verification on an uploaded design file. In a demo, it walked a user through uncovering a Trojan planted in an AES S-Box that leaked the encryption key on a specific input trigger.

99.9% of fixable AI vulnerabilities remain unpatched (1 minute read)

Orca Security's 2026 State of AI Security Report found that 99.9% of AI vulnerability alerts with an available fix remain unpatched, and 81.2% of companies running AI packages carry at least one known vulnerability. Attackers are targeting five layers of the AI stack: package registries, model hubs, developer tools, agent frameworks, and brand trust. Prioritize patching critical CVEs (present in 74.1% of AI-package deployments) and inventory every production agent as a non-human identity with its own permissions and blast radius.

Cynative: Open-source deep research agent (1 minute read)

Cynative, an open-source security research agent, orchestrates a frontier model against a company's own code, cloud, and runtime, running generated code in a sandbox and classifying every action as read or write before attaching credentials, with writes gated behind explicit opt-in. Read/write mappings resolve from providers' live sources (AWS Service Reference API, iam-dataset fallback) rather than a hand-maintained table, and any unresolvable action is denied outright, so an unmapped write never slips through.
