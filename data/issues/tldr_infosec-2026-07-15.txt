TS-2026-009: Insecure argument handling in Tailscale SSH permitted root access (1 minute read)

Because these usernames were passed to getent(1), a login as `-i` was parsed as a flag that dumped the passwd file and opened an interactive root session; upgrade to version 1.98.9 or newer.

OAuth Client ID Spoofing Lets Attackers Validate Stolen Microsoft Entra Credentials (1 minute read)

Attackers send ROPC-flow requests with syntactically valid but unregistered client IDs to the OAuth token endpoint, reading AADSTS error codes to confirm usernames and passwords while leaving only an application ID and no app name in sign-in logs.

Microsoft’s Secure Boot has been broken for a decade and no one noticed until now (1 minute read)

The unrevoked binaries affect both Windows and Linux devices and enable persistent bootkits that survive OS reinstalls and drive swaps, so defenders should apply the updated UEFI revocation (DBX) lists.

Trust but Verify? Uncovering the Security Debt of Autonomous Coding Agents (1 minute read)

A large-scale study using the AIDev dataset found that 38.9% of agent-generated pull requests contain at least one security smell, with supply-chain integrity issues making up 82.3% and hard-coded credentials accounting for 99.6% of critical-severity findings. Notably, human collaborators introduced 67.6% of genuine leaked secrets, and existing review processes missed 81.1% before merge. The authors argue for context-aware guardrails at the point of human-AI collaboration.

Accelerating EDR Evasion with LLM-Driven Analysis (5 minute read)

SpecterOps researchers detailed using LLMs to accelerate reverse-engineering of the "big 5" EDR products, driving disassembly and debugging to surface on-host evasion techniques far faster than manual analysis. The post argues LLM-driven analysis is systematically breaking on-host detections and urges vendors and defenders to assume their endpoint controls are more evadable than believed. Red teams should expect these bypasses to become public rather than trade-secret, shifting defensive weight toward behavioral and network telemetry that survives host-level evasion.

Antiproof: Synthesizing Vulnerability Detectors and Proofs of Exploitability (1 minute read)

Antiproof is an end-to-end vulnerability discovery system that pairs neuro-symbolic detector synthesis for high recall with proof-of-exploitability oracles that validate candidates by executing concrete attacker capabilities. It detected 64 of 66 vulnerabilities on BountyBench and KEVBench, over 60 points better recall than baselines, and surfaced several hundred unknown bugs across 50 systems. The authors have 12 CVE assignments so far, including RCE in Ray, SGLang, vLLM, and LiteLLM.

AutoTrace: From Patches to Triggers via Agentic Interprocedural Exploration (1 minute read)

AutoTrace is an agentic pipeline that localizes vulnerability triggers by walking a code property graph layer by layer, using LLM agents to decide where to look while deterministic admissibility gates require graph-backed evidence before any trigger is reported. It hit 75.0% VulnHit and 80.8% FuncHit on the InterPVD benchmark, beating prior state of the art, and the authors released SinkTrace-Bench, 1,542 balanced source-to-sink samples on which even frontier LLMs struggle to separate matched vulnerable and patched pairs.

Blocking Slow-Burn Attacks: Contextual Policies in Omnigent (1 minute read)

Databricks demonstrates a contextual policy in Omnigent that blocks indirect prompt-injection "slow-burn" attacks, where a malicious goal is split into individually benign steps (read a doc, read another, summarize, send) that no single-action guardrail catches. The stateful policy tracks accumulated risk across the whole session and blocks the outbound step once the agent has read too much sensitive material. The guard is tamper-resistant: the agent has no tool to disable it, adding policies requires human approval, and any denial wins when policies combine.

MindReader: Using LLMs to Encourage Memorable and Secure Password Replacement (1 minute read)

MindReader is a tool that uses LLMs to help users pick forced-replacement passwords that are harder to guess but still memorable, by inferring the meaning behind an old password's components and suggesting semantically related but syntactically unrelated substitutes. Open-Source Intelligence for Code Provenance and the Security Patterns that Separate Human and Large-Language-Model Implementations of Common Programming Tasks (1 minute read)

This paper builds a fully reproducible, open-source pipeline that collects human and LLM implementations of 31 security-sensitive tasks, including OAuth PKCE, JWT verification, and password hashing, then trains a classifier that recovers human-versus-model provenance at 93% accuracy. The two sources diverge on security patterns in every language tested. A repair case study hit a 77% fix rate but repeatedly showed models stripping the insecure pattern without adding the correct defense, leaving code superficially clean but still exposed.

Cross-Cutting Security Analysis of LLM-Generated Code via Metamorphic Testing and Association Rule Mining (1 minute read)

This paper tests security in LLM-generated code using metamorphic relations plus association rule mining, applying nine relations across major CWE categories to 3,700 snippets from five open models. 68.8% violated at least one relation, led by hard-coded credentials (79.1%) and command injection (74.4%), and violations cluster: XSS plus weak crypto predicts hard-coded credentials with 82.5% confidence. The takeaway for defenders is cluster-aware verification, since database- and auth-related prompts reliably produce broad cross-cutting insecurity.
