New wp2shell WordPress Core Flaw Lets Unauthenticated Attackers Run Code (1 minute read)

WordPress shipped 6.9.5 and 7.0.2 on July 17 to close a pre-auth RCE in core (branded wp2shell by Assetnote's Adam Kues) that an anonymous HTTP request can trigger against a default install with no plugins, affecting 6.9.0 through 6.9.4 and 7.0.0 through 7.0.1. No CVE or CVSS has been assigned yet, so verify your running version directly rather than trusting the forced auto-update to have landed.

Inc Ransomware Exploits SonicWall SMA Zero-Days (1 minute read)

Inc ransomware is exploiting two SonicWall SMA 1000 zero-days, CVE-2026-15409 (unauthenticated SSRF, CVSS 10.0) and CVE-2026-15410 (authenticated command injection, CVSS 7.2), chaining them for unauthenticated RCE and root-level command execution to harvest credentials ahead of ransomware deployment. SonicWall published an advisory on July 14 and CISA added both to its KEV catalog, so patch SMA 1000 appliances immediately and rotate exposed credentials, as a PoC for the SSRF flaw is already public.

HollowByte DDoS flaw bloats OpenSSL server memory with 11-byte payload (1 minute read)

HollowByte, a DoS flaw disclosed by Okta's Red Team, lets an unauthenticated attacker send an 11-byte TLS handshake message with an oversized length field, forcing vulnerable OpenSSL versions to allocate the declared buffer before validating the payload and leaving worker threads blocked on data that never arrives. Repeated connections fragment the heap so RSS stays bloated until restart; OpenSSL silently patched and backported the fix with no identifier, so upgrade the embedded library across NGINX, Apache, and language runtimes.

OpenSSL HollowByte Flaw Could Freeze Server Memory with 11-Byte TLS Requests (1 minute read) [flag]

OpenSSL's HollowByte flaw, disclosed by Okta's Red Team, lets 11-byte TLS handshake messages force an unpatched server to pre-allocate up to 131 KB per connection for a body that never arrives, and on glibc systems heap fragmentation keeps that memory frozen after the attacker disconnects. OpenSSL shipped fixes on June 9 (4.0.1, 3.6.3, 3.5.7, 3.4.6, 3.0.21) with no CVE or advisory, so upgrade to those releases rather than relying on connection limits.

New NadMesh Botnet Hunts Exposed AI Services for Cloud Keys and Kubernetes Tokens (1 minute read)

NadMesh, a Go botnet detailed by QiAnXin's XLab, scans Shodan for exposed AI services like ComfyUI, Ollama, n8n, and Open WebUI, then steals cloud keys from environment variables, k8s service-account tokens, and files like ~/.aws/config and .env. It prioritizes unauthenticated MCP servers, using a JSON-RPC tools/call to execute_command, and distinct source IPs pushing it climbed from near zero in late June to roughly 139 a day.

Seven Malicious Vite npm Packages Use Blockchain C2 to Deliver a RAT (1 minute read)

Checkmarx disclosed ViteVenom, a cluster of seven malicious npm packages using scoped names to impersonate the "@vitejs/*" namespace and deliver a RAT capable of reverse shells, credential harvesting, file exfiltration, and backdoor injection. Attributed to threat actor SuccessKey, the loader executes at import rather than install time to evade endpoint detection and pulls later stages from a four-tier blockchain C2 spanning Tron, Aptos, and Binance Smart Chain that resists takedown.

GoldenEyeDog Subgroup Linked to DigiCert Breach and Code-Signing Certificate Theft (1 minute read)

Expel attributed the April 2026 DigiCert breach to CylindricalCanine, a subgroup of the Chinese cybercrime group GoldenEyeDog (APT-Q-27, Dragon Breath), which used a modified Gh0st RAT variant called Golden Gh0st RAT to compromise a support employee's device and steal code-signing certificates intended for DigiCert customers. Organizations that received DigiCert certificates in that window should treat them as potentially compromised, monitor for binaries signed with stolen certs, and prioritize revocation and reissuance.

Brex built its AI agent policy by watching what agents actually do, not by writing rules first (3 minute read)

Brex open-sourced CrabTrap, an HTTP/HTTPS proxy that intercepts all agent network traffic and uses an LLM-as-a-judge to approve or deny each request against policy, rather than relying on scoped tokens or model guardrails that prompt injection can bypass. The company built its agent policies by observing real in-the-wild behavior instead of writing rules upfront. The takeaway for security teams: treat the network layer as a centralized enforcement point for agent governance, since every outbound request is an interceptable policy decision.

Why Your AI Agent Has an Identity Problem (1 minute read)

AI agents deployed to production typically run under shared service accounts with over-broad permissions, no distinct identity, and no audit trail that can reconstruct what they accessed or why. The author argues traditional IAM breaks because it assumes a human actor, while an agent may make 40 API calls in 30 seconds, touch 500 customer records for one task, and delegate to other agents no human approves. Provisioning per-agent identity early cuts both audit-failure risk and the cost of scaling agentic systems.
