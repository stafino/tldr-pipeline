WordPress Core "wp2shell" RCE flaws get public exploits, patch now (1 minute read)

Public proof-of-concept exploits are out for wp2shell, a chain of CVE-2026-63030 (REST API batch-route confusion) and CVE-2026-60137 (SQL injection in WP_Query's author__not_in) that gives pre-auth RCE on stock WordPress 6.9.0-6.9.4 and 7.0.0-7.0.1. WordPress forced auto-updates to 7.0.2 and 6.9.5; sites with auto-update disabled should patch and hunt for webshells now.

Exploit brokers pay $500k for WordPress RCEs. I found one with GPT5.6 and $25 (1 minute read)

A Searchlight Cyber researcher used GPT5.6 Sol Ultra, running four agents for six hours at roughly $25 in inference, to find a pre-auth RCE chain in WordPress core from source analysis alone. Two other teams independently reproduced the full chain during the disclosure hold and PoCs have since appeared on GitHub, so patch immediately and treat unpatched internet-facing instances as exposed.

Critical NGINX Vulnerability Can Crash Workers and May Allow Remote Code Execution (1 minute read)

F5 patched CVE-2026-42533 (CVSS v4 9.2), a heap buffer overflow in nginx's script engine that a remote, unauthenticated attacker can trigger with crafted HTTP requests to crash workers or, where ASLR is bypassable, achieve RCE. Every build from 0.9.6 through 1.31.2 is affected, but exploitation requires a regex-based map whose output is referenced after an earlier capture, so audit configs before assuming you are clear.

A forensic tool for backdoored code completions in AI assistants (1 minute read)

Researchers at the University of Louisville and the University of North Texas released CodeTracer, a forensic tool that traces a poisoned code completion back to the fine-tuning examples that taught it, without access to training gradients. It summarizes the unsafe behavior, uses UniXcoder to narrow a corpus of over a million Python files to 500 candidates, then has GPT-4.1 judge which ones carry the same pattern. That covers the post-incident gap left by data screening and output scanning, which only catch poisoning before it lands.

Qubes OS Security in the Public Record (1 minute read)

A longitudinal analysis of 109 Qubes Security Bulletins (2011-2025) and the project's Xen Security Advisory tracker found that 87 of 109 QSBs (79.8%) trace to Xen, CPU/microarchitectural, or other upstream components rather than Qubes-core logic, with 113 of 464 XSAs affecting Qubes. Post-2018 annual disclosure rates are statistically flat, and S-shaped vulnerability discovery models did not beat a rolling-mean baseline on short-horizon forecasts. The measurable risk in compartmentalized architectures sits in the upstream trust anchors, not the isolation layer.

The Windows 10 hangover is becoming a security problem (1 minute read)

Windows 10 still runs on 16.9% of Windows devices nine months after end of support, and those machines average 1,903 active CVEs each versus 652 on Windows 11, with 66.6% rated high or critical and 2.4% known-exploited. Healthcare, retail, and manufacturing lag most because embedded and vendor-certified hardware can't take Windows 11. Consumer ESU buys eligible devices until October 2027 but excludes most enterprise, LTSC, and domain-joined configurations, so segment and compensate rather than assume coverage.

Launches & Tools

Capital One Open Sources AI-Powered ‘VulnHunter’ Security Tool (1 minute read)

Capital One open sourced VulnHunter, an agentic code-analysis tool that traces exploitable paths and proposes targeted patches rather than emitting passive scanner findings. It requires Claude Opus 4.8 and a working Claude Code environment, so evaluation costs scale with repo count; Capital One says it ran across thousands of internal repositories, and GitHub ships with quickstart docs and example remediation workflows.

Meet Dusseldorf, Microsoft’s open-source out-of-band security platform (1 minute read)

Microsoft released Dusseldorf, an open-source out-of-band application security testing platform that runs DNS, HTTP, and HTTPS listeners on an operator-controlled domain to catch SSRF, blind XSS, SSTI, and XXE callbacks. Deployment is self-hosted via Docker Compose for lab use or Helm on Azure, which keeps interaction data off third-party collaborator services that most teams currently rely on for the same detection.
