FFmpeg fixes PixelSmash flaw in widely used video decoder (1 minute read)

FFmpeg patched PixelSmash (CVE-2026-8461), a high-severity heap out-of-bounds write in the MagicYUV decoder that JFrog used to achieve remote code execution on a Jellyfin media server through its normal library-scan pipeline. Any application built on libavcodec is exposed, including Kodi, OBS Studio, Nextcloud, PhotoPrism, and Linux desktop thumbnail generators, so IT teams should prioritize updating FFmpeg across self-hosted media and file-preview infrastructure.
