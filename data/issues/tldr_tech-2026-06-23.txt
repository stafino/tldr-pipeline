The new HTTP QUERY method explained (1 minute read)

The new HTTP QUERY method standardizes sending a request body with read requests, solving the long-standing problem of cramming complex filters, deep nesting, and arrays into GET query strings that hit URL length limits and encoding ambiguity. Unlike GET requests with bodies, which proxies and servers handle inconsistently, QUERY is safe and idempotent by design, giving APIs a defined way to express rich queries without resorting to POST.

Beyond Simpson's Paradox: A Cascade of Confounders in AI Agent Pull-Request Co-Authorship (1 minute read)

This paper reanalyzes 33,596 pull requests from five AI coding agents and shows that the apparent merge-rate penalty for human co-authorship (53.8% vs. 79.8%) is a Simpson's Paradox driven by agent mix, since Codex dominates the data and rarely uses co-authorship. Once repository selection and PR structure are controlled, no agent retains a clear co-authorship effect, suggesting prior cross-sectional associations are selection artifacts rather than causal benefits.
